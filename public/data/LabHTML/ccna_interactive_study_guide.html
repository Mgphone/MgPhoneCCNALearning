Basic Device Configuration & Security

This interactive guide covers fundamental steps for initial Cisco device configuration, focusing on securing administrative access. You will learn to assign hostnames, encrypt passwords, and save configurations to NVRAM—critical skills for the CCNA exam and real-world networks.

📊 Overview & Architecture

This section breaks down the core theoretical concepts and visualizes the structure of the commands you will learn. Understanding the "why" behind hostnames and password encryption is critical before jumping into the command line interface.

🏷️ Hostnames

Default settings use `Router` or `Switch`. Hostnames allow engineers to logically identify devices in a network topology, essential for management and troubleshooting.

🔑 Passwords vs Secrets

enable password stores in plain text. enable secret uses a strong hashing algorithm (MD5/SHA) to secure the Privileged EXEC mode password.

Exam Tip: If both are set, the device always requires the enable secret.

🛡️ Service Password-Encryption

Applies a weak Type 7 cipher to plain-text passwords in the config file. Prevents casual "shoulder surfing" but is easily cracked. Not a substitute for secrets.

💾 Running vs Startup Config

The running-config is in active RAM (lost on reboot). The startup-config is saved in NVRAM. Configurations must be explicitly saved.

💻 Interactive Configuration Lab

Explore the step-by-step configuration for routers and switches. Use the interactive tabs below to switch between device types and view the exact Cisco IOS syntax required to secure administrative access. 

! 1. Enter Global Configuration Mode
Router> enable
Router# configure terminal

! 2. Change the Hostname
Router(config)# hostname R1

! 3. Configure Unencrypted Enable Password
R1(config)# enable password CCNA

! 4. Encrypt All Clear-Text Passwords
R1(config)# service password-encryption

! 5. Configure Secure Enable Secret
R1(config)# enable secret Cisco

! 6. Save Configuration
R1(config)# end
R1# copy running-config startup-config
                
! 1. Enter Global Configuration Mode
Switch> enable
Switch# configure terminal

! 2. Change the Hostname
Switch(config)# hostname SW1

! 3. Configure Unencrypted Enable Password
SW1(config)# enable password CCNA

! 4. Encrypt All Clear-Text Passwords
SW1(config)# service password-encryption

! 5. Configure Secure Enable Secret
SW1(config)# enable secret Cisco

! 6. Save Configuration
SW1(config)# end
SW1# write memory

🔍 Verification Step

To verify your password security and device settings, view the active configuration file in RAM:

R1# show running-config
  • Look near the top of the output to find the hostname.
  • Check for enable secret 5 [hash] and enable password 7 [hash].
  • In Global Config mode, prepend with do: R1(config)# do show running-config
