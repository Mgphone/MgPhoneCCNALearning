Basic Configuration

Basic Device Configuration & Security

This module covers the fundamental steps for initial Cisco device configuration, focusing on securing administrative access. You will learn how to assign logical hostnames, configure and encrypt privileged EXEC mode passwords, and safely save your configurations to NVRAM.

Theory & Mechanics

When you first boot up a Cisco router or switch, it uses default settings (e.g., the hostname Router or Switch) and has no security for administrative access. Anyone with physical console access can manipulate the device.

🔑 Enable Password vs. Enable Secret

  • Enable Password: Secures Privileged EXEC mode but stores the password in plain text within the configuration file by default.
  • Enable Secret: Serves the same purpose but uses a strong hashing algorithm (MD5, SHA-256) to securely obscure the password in the config file.

🛡️ Service Password-Encryption

This command applies a weak proprietary cipher (Type 7) to any plain-text passwords stored in the configuration file. It prevents casual "shoulder surfing" but is easily cracked. It should not be a substitute for the enable secret.

⚠️ CCNA Exam Tip

If both enable password and enable secret are configured on a Cisco device, the device will always require the enable secret. The enable password effectively becomes invalid.

Command Distribution by Mode

Analysis of where the core security configuration commands are executed.

Most configuration changes occur in Global Config mode, while viewing and saving occur in Privileged EXEC mode.

Device Configuration Lab

Review the exact Cisco IOS commands required to secure administrative access. The logic is identical for both routers and switches.

! 1. Enter Global Configuration Mode
Router> enable
Router# configure terminal

! 2. Change the Hostname
Router(config)# hostname R1

! 3. Configure Unencrypted Enable Password
R1(config)# enable password CCNA

! 4. Encrypt All Clear-Text Passwords
R1(config)# service password-encryption

! 5. Configure Secure Enable Secret
R1(config)# enable secret Cisco

! 6. Save Configuration to NVRAM
R1(config)# end
R1# copy running-config startup-config
! 1. Enter Global Configuration Mode
Switch> enable
Switch# configure terminal

! 2. Change the Hostname
Switch(config)# hostname SW1

! 3. Configure Unencrypted Enable Password
SW1(config)# enable password CCNA

! 4. Encrypt All Clear-Text Passwords
SW1(config)# service password-encryption

! 5. Configure Secure Enable Secret
SW1(config)# enable secret Cisco

! 6. Save Configuration to NVRAM
SW1(config)# end
SW1# write memory

🔍 Verification Step

To verify your password security and device settings, you should view the active configuration file in RAM:

R1# show running-config
  • Look near the top of the output to find the hostname.
  • Check for enable secret 5 [hash] and enable password 7 [hash].
  • If you are in Global Configuration mode, prepend the command with do (e.g., do show running-config).

Lab Task Q&A

Below are the expert solutions to the lab tasks from the video. Check off the steps as you complete them to save your progress! Click on a question to expand its answer.

To change the hostname, use the 'hostname' command in global configuration mode.

Router> enable
Router# configure terminal
Router(config)# hostname R1

Switch> enable
Switch# configure terminal
Switch(config)# hostname SW1

This command protects Privileged EXEC mode but stores the password in plain text.

R1(config)# enable password CCNA
SW1(config)# enable password CCNA

You can use exit successively or disable to drop down out of Privileged EXEC mode.

R1(config)# exit
R1# exit
R1> enable
Password: CCNA

You must be in Privileged EXEC mode to issue standard show commands.

R1# show running-config

*You will see the line enable password CCNA stored in plain, readable text near the top of the output.

This applies a proprietary cipher to any plain-text passwords currently in the config and automatically encrypts any new plain-text passwords you create later.

R1# configure terminal
R1(config)# service password-encryption

You can drop back to Privileged EXEC mode, or use the do command to run the check directly from Global Config mode.

R1(config)# do show running-config

*The password will now read something like enable password 7 0822455D0A16, masking the text "CCNA".

The enable secret command uses a strong hashing algorithm (MD5/SHA) and is the Cisco standard best practice.

R1(config)# enable secret Cisco
SW1(config)# enable secret Cisco
Answer: You MUST use the newly configured enable secret password, which is Cisco. The enable password ("CCNA") is effectively invalid when a secret exists.

When you run show running-config, observe the numbers immediately following the password commands.

  • Encrypted 'enable password': Type 7 (Indicated by enable password 7 .... This is the weak proprietary cipher applied by service password-encryption).
  • Encrypted 'enable secret': Type 5 (Indicated by enable secret 5 .... This represents a stronger MD5 hash algorithm).

You must do this from Privileged EXEC mode. If you reboot without doing this, all changes are lost.

R1(config)# end
R1# copy running-config startup-config

*Alternatively, you can use the legacy command write memory or just wr.
