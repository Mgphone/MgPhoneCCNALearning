VLSM Subnetting

Variable Length Subnet Masking (VLSM)

Learn to take a single Class C network block (192.168.5.0 /24) and divide it efficiently into multiple, differently-sized subnets. This satisfies specific host requirements without wasting valuable IP addresses.

🧠 The Golden Rule of VLSM

Always allocate your subnets starting from the largest host requirement down to the smallest. If you do not allocate the largest subnets first, you risk fragmenting your IP space and being unable to fit large subnets later.

Address Allocation Share

Allocation Breakdown

  • LAN 2: 64 hosts needed → /25 (128 IPs)
  • LAN 1: 45 hosts needed → /26 (64 IPs)
  • LAN 3: 14 hosts needed → /28 (16 IPs)
  • LAN 4: 9 hosts needed → /28 (16 IPs)
  • R1-R2 P2P: 2 hosts needed → /30 (4 IPs)
  • Unused Space: Available for future growth.
Note: A point-to-point router link only requires 2 usable IPs. A /30 mask provides exactly 2 usable addresses (4 total IPs), which is the standard CCNA safe answer.

🖥️ Device Configuration Lab

Goal: Assign the calculated VLSM IP addresses to R1's interfaces, and create static routes pointing towards R2's networks.

R1> enable
R1# configure terminal
R1(config)# hostname R1

! LAN 2 Interface (Last usable address in /25)
R1(config)# interface g0/1
R1(config-if)# ip address 192.168.5.126 255.255.255.128
R1(config-if)# no shutdown

! LAN 1 Interface (Last usable address in /26)
R1(config-if)# interface g0/0
R1(config-if)# ip address 192.168.5.190 255.255.255.192
R1(config-if)# no shutdown

! Point-to-Point Link to R2 (First usable address in /30)
R1(config-if)# interface g0/0/0
R1(config-if)# ip address 192.168.5.225 255.255.255.252
R1(config-if)# no shutdown
R1(config-if)# exit

! Static Routes pointing to R2's LANs via R2's P2P IP
R1(config)# ip route 192.168.5.192 255.255.255.240 192.168.5.226
R1(config)# ip route 192.168.5.208 255.255.255.240 192.168.5.226
R1(config)# end
R1# write memory

Goal: Assign the calculated VLSM IP addresses to R2's interfaces, and create static routes pointing towards R1's networks.

R2> enable
R2# configure terminal
R2(config)# hostname R2

! LAN 3 Interface (Last usable address in /28)
R2(config)# interface g0/0
R2(config-if)# ip address 192.168.5.206 255.255.255.240
R2(config-if)# no shutdown

! LAN 4 Interface (Last usable address in /28)
R2(config-if)# interface g0/1
R2(config-if)# ip address 192.168.5.222 255.255.255.240
R2(config-if)# no shutdown

! Point-to-Point Link to R1 (Last usable address in /30)
R2(config-if)# interface g0/0/0
R2(config-if)# ip address 192.168.5.226 255.255.255.252
R2(config-if)# no shutdown
R2(config-if)# exit

! Static Routes pointing to R1's LANs via R1's P2P IP
R2(config)# ip route 192.168.5.0 255.255.255.128 192.168.5.225
R2(config)# ip route 192.168.5.128 255.255.255.192 192.168.5.225
R2(config)# end
R2# write memory

You must allocate from largest to smallest to avoid overlapping blocks.

  • LAN 2 (64 hosts): Needs /25 (126 usable). Net: 192.168.5.0 /25
  • LAN 1 (45 hosts): Needs /26 (62 usable). Net: 192.168.5.128 /26
  • LAN 3 (14 hosts): Needs /28 (14 usable). Net: 192.168.5.192 /28
  • LAN 4 (9 hosts): Needs /28 (14 usable). Net: 192.168.5.208 /28
  • R1-R2 P2P (2 hosts): Needs /30 (2 usable). Net: 192.168.5.224 /30

Mask: 255.255.255.192

  • R1 (LAN 1 - G0/0): 192.168.5.190
  • R1 (LAN 2 - G0/1): 192.168.5.126
  • R2 (LAN 3 - G0/0): 192.168.5.206
  • R2 (LAN 4 - G0/1): 192.168.5.222

For the P2P connection, we use the only two available addresses: R1 gets .225, R2 gets .226.

See the CLI Configuration tab for exact syntax. Conceptually:

  • R1 must route 192.168.5.192/28 and 192.168.5.208/28 to R2's interface (192.168.5.226).
  • R2 must route 192.168.5.0/25 and 192.168.5.128/26 to R1's interface (192.168.5.225).
