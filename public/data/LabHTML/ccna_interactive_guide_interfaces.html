Advanced Interface Configuration

Move beyond basic IP addressing. Learn how to manually define speed and duplex settings for stable uplinks, use the interface range command for bulk port management, and secure your network by administratively disabling unused switch ports.

🧠 Core Concepts

Default Port States

  • Routers: All interfaces are administratively down by default. You must explicitly issue no shutdown.
  • Switches: All interfaces are up by default. For security, explicitly issue shutdown on unused ports.

Speed & Duplex Tuning

Interfaces dynamically auto-negotiate settings. However, for critical infrastructure links (Router-to-Switch, Switch-to-Switch), hardcoding speed 1000 and duplex full prevents negotiation failures that cause network flapping.

Packet Tracer Quirk: If you manually configure these settings, Packet Tracer's show interfaces status command may still show "a-1000" and "a-full" (indicating auto). Real Cisco hardware drops the "a-". Always trust your running-config.

🖥️ Device CLI Configurations

Goal: Set IP, configure manual speed/duplex for the switch uplink, label it, and bring it online.

Router> enable
Router# configure terminal
Router(config)# hostname R1

! Configure the active interface to SW1
R1(config)# interface g0/0
R1(config-if)# ip address 172.16.255.254 255.255.0.0
R1(config-if)# speed 1000
R1(config-if)# duplex full
R1(config-if)# description to switch 1
R1(config-if)# no shutdown
R1(config-if)# exit

! Add descriptions to unused ports
R1(config)# interface range g0/1 - 2
R1(config-if-range)# description not in use
R1(config-if-range)# end
R1# copy running-config startup-config

Goal: Configure hardcoded uplinks to R1 and SW2, label edge ports, and securely shut down all unused interfaces using ranges.

Switch> enable
Switch# configure terminal
Switch(config)# hostname SW1

! Configure uplink to R1
SW1(config)# interface g0/1
SW1(config-if)# speed 1000
SW1(config-if)# duplex full
SW1(config-if)# description to r1
SW1(config-if)# exit

! Configure uplink to SW2
SW1(config)# interface g0/2
SW1(config-if)# speed 1000
SW1(config-if)# duplex full
SW1(config-if)# description to switch 2
SW1(config-if)# exit

! Configure edge ports for PCs
SW1(config)# interface range f0/1 - 2
SW1(config-if-range)# description to end hosts
SW1(config-if-range)# exit

! Secure unused ports
SW1(config)# interface range f0/3 - 24
SW1(config-if-range)# description not in use
SW1(config-if-range)# shutdown
SW1(config-if-range)# end
SW1# write memory

Goal: Hardcode uplink to SW1, label edge ports, and use a comma-separated range to disable Gigabit and FastEthernet ports simultaneously.

Switch> enable
Switch# configure terminal
Switch(config)# hostname SW2

! Configure uplink to SW1
SW2(config)# interface g0/1
SW2(config-if)# speed 1000
SW2(config-if)# duplex full
SW2(config-if)# description to switch 1
SW2(config-if)# exit

! Configure edge ports for PCs
SW2(config)# interface range f0/1 - 2
SW2(config-if-range)# description to end hosts
SW2(config-if-range)# exit

! Secure unused ports (combining different port types)
SW2(config)# interface range g0/2 , f0/3 - 24
SW2(config-if-range)# description not in use
SW2(config-if-range)# shutdown
SW2(config-if-range)# end
SW2# write

Enter Global Configuration mode on each respective device and apply the hostname.

Router(config)# hostname R1
Switch(config)# hostname SW1
Switch(config)# hostname SW2

The network used in the lab is 172.16.0.0 /16.

  • R1 (G0/0): 172.16.255.254 255.255.0.0
  • PC1: 172.16.0.1 255.255.0.0 (GW: 172.16.255.254)
  • PC2: 172.16.0.2 255.255.0.0 (GW: 172.16.255.254)
  • PC3: 172.16.0.3 255.255.0.0 (GW: 172.16.255.254)
  • PC4: 172.16.0.4 255.255.0.0 (GW: 172.16.255.254)

Apply these commands ONLY to the Router-to-Switch and Switch-to-Switch uplinks (the Gigabit Ethernet ports).

R1(config-if)# interface g0/0
R1(config-if)# speed 1000
R1(config-if)# duplex full

SW1(config-if)# interface g0/1
SW1(config-if)# speed 1000
SW1(config-if)# duplex full
SW1(config-if)# interface g0/2
SW1(config-if)# speed 1000
SW1(config-if)# duplex full

SW2(config-if)# interface g0/1
SW2(config-if)# speed 1000
SW2(config-if)# duplex full

You should label uplinks based on their destination, label edge ports as end hosts, and add "not in use" to unconnected ports.

! Examples:
R1(config-if)# description to switch 1
SW1(config-if)# description to switch 2
SW2(config)# interface range f0/1 - 2
SW2(config-if-range)# description to end hosts

For routers, interfaces are administratively down by default, so we only use the command to document them. For switches, unused interfaces remain active and pose a security risk. Use the interface range command to shut them down efficiently.

! On SW1
SW1(config)# interface range f0/3 - 24
SW1(config-if-range)# shutdown

! On SW2 (Note the comma to group different port types)
SW2(config)# interface range g0/2 , f0/3 - 24
SW2(config-if-range)# shutdown
