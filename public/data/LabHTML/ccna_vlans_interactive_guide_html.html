Virtual LANs (VLANs) - Part 1

VLANs logically segment a single physical switch into multiple independent broadcast domains. In this lab, we configure access ports, assign them to distinct VLANs (Engineering, HR, Sales), and utilize a router with multiple physical interfaces to route traffic between them.

Addressing the Departments

Each VLAN acts as its own distinct subnet. We are dividing the 10.0.0.0/24 network into four /26 subnets to accommodate our three departments.

Subnet Allocation

  • VLAN 10 (Engineering)
    10.0.0.0 /26
    Gateway: .62
    PCs: .1, .2
  • VLAN 20 (HR)
    10.0.0.64 /26
    Gateway: .126
    PCs: .65, .66
  • VLAN 30 (Sales)
    10.0.0.128 /26
    Gateway: .190
    PCs: .129, .130
  • Unused Space
    10.0.0.192 /26
    Available
Design Note: Legacy Inter-VLAN Routing
In this lab, we use a separate physical cable from SW1 to R1 for every VLAN. This works but does not scale well. In the next module, we will replace these multiple links with a single "Trunk" link (Router-on-a-Stick).

Device Configuration Lab

Review the specific commands to map interfaces to VLANs and route them.

Goal: Provide a default gateway for each VLAN using separate physical interfaces.

R1> enable
R1# configure terminal

! VLAN 10 Gateway
R1(config)# interface g0/0
R1(config-if)# ip address 10.0.0.62 255.255.255.192
R1(config-if)# no shutdown

! VLAN 20 Gateway
R1(config-if)# interface g0/1
R1(config-if)# ip address 10.0.0.126 255.255.255.192
R1(config-if)# no shutdown

! VLAN 30 Gateway
R1(config-if)# interface g0/2
R1(config-if)# ip address 10.0.0.190 255.255.255.192
R1(config-if)# no shutdown

Goal: Name VLANs and map the physical access ports (both host and router links) into their respective VLAN domains.

SW1> enable
SW1# configure terminal

! Create & Name VLANs
SW1(config)# vlan 10
SW1(config-vlan)# name Engineering
SW1(config-vlan)# vlan 20
SW1(config-vlan)# name HR
SW1(config-vlan)# vlan 30
SW1(config-vlan)# name Sales
SW1(config-vlan)# exit

! Assign ports to VLAN 10 (Includes G0/1 uplink to R1)
SW1(config)# interface range g0/1, f3/1, f4/1
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 10

! Assign ports to VLAN 20 (Includes G1/1 uplink to R1)
SW1(config-if-range)# interface range g1/1, f5/1, f6/1
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 20

! Assign ports to VLAN 30 (Includes G2/1 uplink to R1)
SW1(config-if-range)# interface range g2/1, f7/1, f8/1
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 30

All subnets use a /26 mask (255.255.255.192). Configured via Packet Tracer GUI.

  • VLAN 10 (PC1, PC2): IPs: 10.0.0.1, 10.0.0.2 | GW: 10.0.0.62
  • VLAN 20 (PC3, PC4): IPs: 10.0.0.65, 10.0.0.66 | GW: 10.0.0.126
  • VLAN 30 (PC5, PC6): IPs: 10.0.0.129, 10.0.0.130 | GW: 10.0.0.190

We establish 3 physical cables and assign the previously calculated Gateways to R1:

R1(config)# int g0/0
R1(config-if)# ip addr 10.0.0.62 255.255.255.192
R1(config-if)# no shut
R1(config)# int g0/1
R1(config-if)# ip addr 10.0.0.126 255.255.255.192
R1(config-if)# no shut
R1(config)# int g0/2
R1(config-if)# ip addr 10.0.0.190 255.255.255.192
R1(config-if)# no shut

We use the interface range command, explicitly including the Gigabit uplinks facing R1.

SW1(config)# vlan 10
SW1(config-vlan)# name Engineering
SW1(config)# int range g0/1, f3/1, f4/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10
... (Repeat mapping pattern for VLANs 20 and 30)
  • Unicast Ping (PC1 to PC3): PC1 sends ICMP Echo. It goes to SW1, then is routed up to R1's G0/0 interface. R1 routes it out G0/1 back to SW1 on VLAN 20, which delivers it to PC3.
  • Broadcast Ping (PC1 to 10.0.0.63): Sent from PC1. It hits SW1. SW1 floods it out all active ports assigned to VLAN 10. It is received by PC2 and R1 (G0/0). Crucially: It does NOT cross into VLAN 20 or 30, proving the broadcast domain is segmented.
