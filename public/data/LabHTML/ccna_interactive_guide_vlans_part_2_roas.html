VLANs Part 2 (ROAS)

Router on a Stick (ROAS) & Trunking

Scale your network by learning how to route multiple VLANs over a single physical link. This module covers 802.1Q trunking, Native VLANs, and deploying virtual Subinterfaces on a Cisco router.

🧠 The Problem with Legacy Routing

In VLANs Part 1, we used a separate physical cable from the switch to the router for *every* VLAN. If a company scales to 50 VLANs, a router would need 50 physical interfaces, which is impossible and expensive. The solution is 802.1Q Trunking and Router-on-a-Stick.

How ROAS Works

  • Subinterfaces: Instead of physical ports, we divide one router port (e.g., G0/0) into multiple virtual interfaces (e.g., G0/0.10).
  • 802.1Q Encapsulation: The switch tags frames with their VLAN ID. The router reads the tag, routes the packet internally, re-tags it for the destination VLAN, and sends it back down the "stick".

Trunking Security

By default, trunks allow all VLANs. Best practice is to manually specify an allowed list to prune unnecessary traffic.

You should also change the Native VLAN (untagged traffic) from the default VLAN 1 to a completely unused, "dead" VLAN (e.g., VLAN 1001) to prevent VLAN hopping attacks.

💻 Device Configuration Lab

Review the specific commands to configure Trunks, Native VLANs, and Router Subinterfaces.

Goal: Implement Router on a Stick by configuring dot1q subinterfaces.

R1> enable
R1# configure terminal

! Enable the physical interface (No IP needed here)
R1(config)# interface g0/0
R1(config-if)# no shutdown
R1(config-if)# exit

! VLAN 10 Subinterface
R1(config)# interface g0/0.10
R1(config-subif)# encapsulation dot1q 10
R1(config-subif)# ip address 10.0.0.62 255.255.255.192

! VLAN 20 Subinterface
R1(config-subif)# interface g0/0.20
R1(config-subif)# encapsulation dot1q 20
R1(config-subif)# ip address 10.0.0.126 255.255.255.192

! VLAN 30 Subinterface
R1(config-subif)# interface g0/0.30
R1(config-subif)# encapsulation dot1q 30
R1(config-subif)# ip address 10.0.0.190 255.255.255.192

Goal: Set access ports and configure a highly secure trunk to SW2.

SW1> enable
SW1# configure terminal

! Assign Access Ports
SW1(config)# interface range f0/1 - 2
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 10

SW1(config-if-range)# interface range f0/3 - 4
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 30
SW1(config-if-range)# exit

! Configure secure Trunk to SW2
SW1(config)# interface g0/1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,30
! ^ We prune VLAN 20 because SW1 has no VLAN 20 hosts
SW1(config-if)# switchport trunk native vlan 1001

Goal: Configure the trunk to SW1, manually create VLAN 30, and configure the ROAS trunk to R1.

SW2> enable
SW2# configure terminal

! Ensure all required VLANs exist in the database manually
SW2(config)# vlan 30
SW2(config-vlan)# exit

! Assign Access Ports
SW2(config)# interface f0/1
SW2(config-if)# switchport mode access
SW2(config-if)# switchport access vlan 20

SW2(config-if)# interface range f0/2 - 3
SW2(config-if-range)# switchport mode access
SW2(config-if-range)# switchport access vlan 10
SW2(config-if-range)# exit

! Configure Trunk to SW1
SW2(config)# interface g0/1
SW2(config-if)# switchport mode trunk
SW2(config-if)# switchport trunk allowed vlan 10,30
SW2(config-if)# switchport trunk native vlan 1001
SW2(config-if)# exit

! Configure ROAS Trunk to R1 (Allowing all active VLANs)
SW2(config)# interface g0/2
SW2(config-if)# switchport mode trunk
SW2(config-if)# switchport trunk allowed vlan 10,20,30
SW2(config-if)# switchport trunk native vlan 1001

📝 Lab Tasks & Q&A

Check off the steps as you complete them to save your progress! Click on a question to expand its formal answer based on the video.

Assign edge ports to specific VLANs. This automatically creates the VLAN in the switch's database if it does not already exist.

SW1(config)# interface range F0/1 - 2
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 10

SW1 does not have any VLAN 20 hosts connected to it. Therefore, we prune VLAN 20 from the allowed list to save bandwidth.

SW1(config)# interface g0/1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,30
SW1(config-if)# switchport trunk native vlan 1001
Important Troubleshooting Step: Even if a trunk allows VLAN 30, SW2 will drop the traffic if VLAN 30 is not in its local VLAN database. Because SW2 has no VLAN 30 access ports, it wasn't auto-created. You MUST manually type vlan 30 in Global Config on SW2.

Enable the physical interface, then configure logical subinterfaces. The encapsulation command tells the router which 802.1Q VLAN tag to listen for.

R1(config)# interface g0/0
R1(config-if)# no shutdown
R1(config)# interface g0/0.10
R1(config-subif)# encapsulation dot1q 10
R1(config-subif)# ip address 10.0.0.62 255.255.255.192

From PC1 (VLAN 10), open the command prompt and ping a PC in VLAN 20 and VLAN 30.

PC1> ping 10.0.0.65
Reply from 10.0.0.65...

PC1> ping 10.0.0.129
Reply from 10.0.0.129...

Success! The traffic travels to SW1, over the trunk to SW2, up the trunk to R1, routes to the different subinterface, and goes back down the trunk to its destination.

📋 Command Reference

🔍
Command Mode Purpose
switchport mode trunk Interface Config Forces the port to act as an 802.1Q trunk port, allowing multiple VLANs.
switchport trunk allowed vlan [x,y] Interface Config Restricts the trunk to only forward the explicitly listed VLANs.
switchport trunk native vlan [id] Interface Config Changes the untagged Native VLAN from default (1) to the specified ID.
interface [type] [number].[sub_id] Global Config Creates/enters a virtual subinterface on a router (e.g. g0/0.10).
encapsulation dot1q [vlan_id] Subinterface Config Tells the router subinterface to listen for and tag packets with this VLAN ID.
show interfaces trunk Privileged EXEC Displays trunk ports, their allowed VLANs, and currently active VLANs.
