FTP & TFTP File Management

Learn how to maintain and upgrade Cisco devices by transferring IOS image files and configuration backups across the network using the Trivial File Transfer Protocol (TFTP) and the File Transfer Protocol (FTP).

🧠 Core Operations & Memory Types

TFTP vs. FTP

  • TFTP (Trivial FTP) Uses UDP Port 69. Very fast, lightweight, and simple. No authentication required. Excellent for quick local transfers, but insecure.
  • FTP (File Transfer Protocol) Uses TCP Ports 20 & 21. More reliable due to TCP overhead, and supports basic username/password authentication. (Note: Still sends credentials in plain-text).
  • SCP / SFTP (Not in this lab) Modern best practice uses Secure Copy or Secure FTP, which run over SSH (TCP 22) to fully encrypt file transfers and credentials.

The Restore "Gotcha": Merge vs. Replace

When pulling a configuration backup from a server, the destination dictates the behavior:

  • To NVRAM (startup-config): The incoming file completely replaces the old configuration.
  • To DRAM (running-config): The incoming file merges with the currently active configuration, potentially leaving old, unwanted commands behind.

Cisco Router Memory Components

Understanding where files are saved during transfers.

IOS images are stored in Flash. Active configurations run in DRAM. Saved configurations persist in NVRAM.

💻 Device Configuration Lab

Review the commands to transfer files, configure boot variables, and backup configurations.

Goal: Download a new IOS image via TFTP, set it as the boot image, reload the router, and delete the old file.

R1> enable
! 1. Download the new image from the TFTP server into local flash memory
R1# copy tftp flash
Address or name of remote host []? 10.0.0.1
Source filename []? c1900-universalk9-mz.SPA.155-3.M4a.bin
Destination filename [c1900-universalk9-mz.SPA.155-3.M4a.bin]? <Enter>
Accessing tftp://10.0.0.1/...
Loading c1900-universalk9... !!!!!!!!!!!!!!!!!!!!!!!!!!
[OK]

! 2. Tell the router to boot using the new image file
R1# configure terminal
R1(config)# boot system flash c1900-universalk9-mz.SPA.155-3.M4a.bin
R1(config)# exit

! 3. Save config and reload to apply the new IOS
R1# copy running-config startup-config
R1# reload
Proceed with reload? [confirm] <Enter>

! 4. (After boot) Verify version and delete the old IOS file to free up space
R1# show version
R1# delete flash:c1900-universalk9-mz.SPA.151-4.M4.bin

Goal: Backup the active configuration to a remote server, and safely restore it later.

R4> enable

! 1. Backup the active running configuration to a TFTP Server
R4# copy running-config tftp
Address or name of remote host []? 192.168.1.2
Destination filename [r4-confg]? r4config

! 2. (Sometime later) Restore the configuration from the server
! BEST PRACTICE: Restore to startup-config (NVRAM) so it completely REPLACES the old file.
R4# copy tftp startup-config
Address or name of remote host []? 192.168.1.2
Source filename []? r4config
Destination filename [startup-config]? <Enter>

! 3. Load the newly restored configuration into active memory
R4# copy startup-config running-config

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

First, verify the current version using show version and check space using show flash:. Then execute the copy command from Privileged EXEC mode.

R1# copy tftp flash
Address: 10.0.0.1
Source: c1900-universalk9-mz.SPA.155-3.M4a.bin

You must instruct the router which file in flash it should load on startup.

R1(config)# boot system flash c1900-universalk9-mz.SPA.155-3.M4a.bin
R1(config)# end
R1# write memory
R1# reload

Crucial: If you forget to save the configuration before reloading, the `boot system` command is lost, and the router will boot the old image again.

After verifying the new image loaded correctly with show version, free up memory by removing the old `.bin` file.

R1# delete flash:c1900-universalk9-mz.SPA.151-4.M4.bin

Because FTP requires authentication, you must configure the router with credentials before issuing the copy command.

R2(config)# ip ftp username jeremy
R2(config)# ip ftp password ccna
R2(config)# end
R2# copy ftp flash

📋 Command Reference

🔍
Command Mode Purpose
copy [source] [destination] Privileged EXEC The universal command to move files. E.g., `copy tftp flash`, `copy run tftp`, `copy start run`.
show version Privileged EXEC Displays the currently loaded IOS image version and system uptime.
show flash: Privileged EXEC Lists the files stored in Flash memory, including their exact filenames and sizes.
boot system flash [filename] Global Config Instructs the router which IOS image to load upon the next reboot.
delete flash:[filename] Privileged EXEC Permanently removes a file from flash memory to free up space.
ip ftp username/password Global Config Provides credentials for the router to use when connecting to an FTP server.
