Domain Name System

Domain Name System (DNS)

Computers route traffic using IP addresses, but humans prefer memorable names. Learn how DNS (UDP Port 53) translates Fully Qualified Domain Names (FQDNs) like youtube.com into IP addresses, and how to configure routers to resolve names locally or query external servers.

🧠 Name Resolution Hierarchy

How Routers Resolve Names

When you type a command like ping PC1 on a Cisco router, it must figure out the IP address associated with "PC1" before it can send the ICMP packet. It checks sources in a specific order:

  • 1
    Local Host Table: The router first checks its own manually configured host table (configured via the ip host command).
  • 2
    External DNS Server: If the name isn't in the local table, the router sends a DNS Query to configured external DNS servers (configured via ip name-server).
  • 3
    Broadcast: (Legacy) If no DNS server is configured, older IOS versions might attempt a local broadcast. Modern networks rely strictly on DNS.

You cannot ping a name without first resolving its IP via DNS. You cannot send an IP packet without first resolving the next-hop MAC address via ARP.

💻 Device Configuration Lab

Review the commands to configure local host tables, DNS servers, and routing.

Goal: Configure a default route to the internet, point R1 to a public DNS server, and build a local host table for the LAN PCs.

R1> enable
R1# configure terminal

! Create a default route to the Internet
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.2

! Tell R1 to use Cloudflare's public DNS server for name resolution
! Note: "ip domain-lookup" is enabled by default
R1(config)# ip name-server 1.1.1.1

! Build a Local Host Table so R1 can ping local devices by name
R1(config)# ip host R1 192.168.0.254
R1(config)# ip host PC1 192.168.0.1
R1(config)# ip host PC2 192.168.0.2
R1(config)# ip host PC3 192.168.0.3

! Test local resolution
R1(config)# do ping PC1
Translating "PC1"...domain server (1.1.1.1) [OK]
Type escape sequence to abort.
Sending 5, 100-byte ICMP Echos to 192.168.0.1...
!!!!!

Goal: (From Boson Preview) Configure an IP address on a Layer 2 switch so it can be managed remotely, and set its DNS server.

Switch1> enable
Switch1# configure terminal

! Layer 2 switches don't assign IPs to physical interfaces.
! Assign the management IP to the Switch Virtual Interface (SVI) for VLAN 1.
Switch1(config)# interface vlan 1
Switch1(config-if)# ip address 10.0.0.2 255.255.255.0
Switch1(config-if)# no shutdown
Switch1(config-if)# exit

! Configure DNS lookup so the switch can ping names
Switch1(config)# ip domain-lookup
Switch1(config)# ip name-server 10.0.0.1
Note on Cisco Routers as DNS Servers: You can configure a Cisco router to act as a DNS server for the local network using the global command ip dns server. This allows PCs to point to the router's IP for DNS. However, this command is not supported in Packet Tracer.

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

To reach cloudflare's DNS and YouTube, R1 needs a route indicating how to reach unknown networks.

R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.2

Done via the Packet Tracer GUI.

  • Open PC -> Desktop -> IP Configuration.
  • Ensure 'Static' is selected.
  • Enter 1.1.1.1 in the DNS Server field.

In real networks, this is usually handled automatically via DHCP.

R1(config)# ip name-server 1.1.1.1
R1(config)# ip host PC1 192.168.0.1
R1(config)# ip host PC2 192.168.0.2
R1(config)# ip host PC3 192.168.0.3

Verify the local entries using show hosts.

When you type ping youtube.com on PC1, the following happens:

  1. PC1 creates a DNS Query targeting 1.1.1.1.
  2. The DNS query travels through R1 and the Internet router to Cloudflare.
  3. Cloudflare sends a DNS Reply back containing YouTube's IP (e.g., 172.217.6.78).
  4. Only NOW does PC1 generate the ICMP Echo Request destined for the IP it just learned.

First Ping Timeout:

If the Internet router hasn't talked to YouTube yet, it must pause the ICMP packet to send an ARP Request to resolve YouTube's MAC address. During this delay, PC1's first ping will likely timeout.

📋 Command Reference

Command Mode Purpose
ip name-server [ip] Global Config Configures the external DNS server the router uses to resolve names.
ip domain-lookup Global Config Enables hostname-to-IP translation. Enabled by default on Cisco routers.
no ip domain-lookup Global Config Disables DNS. Useful in labs to prevent the router from hanging when you mistype a command.
