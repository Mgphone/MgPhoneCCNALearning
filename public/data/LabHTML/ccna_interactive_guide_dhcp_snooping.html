DHCP Snooping & Switch Security

Prevent rogue DHCP servers and mitigate exhaustion attacks. By establishing a perimeter of Trusted and Untrusted ports, DHCP Snooping ensures that authoritative IP assignments only come from approved interfaces.

Trust Boundaries & Option 82

Once DHCP Snooping is enabled globally and for specific VLANs, all ports default to an Untrusted state.

  • Untrusted Ports (Client-Facing) Can only send client messages (DISCOVER, REQUEST). If the switch receives server messages (OFFER, ACK) on these ports, it drops them instantly, blocking rogue servers.
  • Trusted Ports (Uplinks/Servers) Must be manually configured. They are allowed to forward authoritative DHCP server messages like OFFER and ACK.
  • The Option 82 Bug Switches running DHCP Snooping insert Option 82 data into client requests. However, if a downstream switch receives Option 82 on an untrusted port, it drops the packet! You must disable this using no ip dhcp snooping information option.

Device Configurations

Goal: Configure the router to act as the legitimate DHCP server for the subnet.

R1(config)# ip dhcp excluded-address 192.168.1.1 192.168.1.9

R1(config)# ip dhcp pool POOL1
R1(dhcp-config)# network 192.168.1.0 255.255.255.0
R1(dhcp-config)# default-router 192.168.1.1
R1(dhcp-config)# exit

Goal: Enable DHCP Snooping, trust the port facing the router, and fix the Option 82 bug.

! 1. Enable Snooping globally, then per-vlan
SW1(config)# ip dhcp snooping
SW1(config)# ip dhcp snooping vlan 1

! 2. Trust the uplink port facing the DHCP server
SW1(config)# interface g0/2
SW1(config-if)# ip dhcp snooping trust
SW1(config-if)# exit

! 3. Prevent the switch from dropping packets due to Option 82 on untrusted ports
SW1(config)# no ip dhcp snooping information option

Goal: Implement basic Boson NetSim security measures (Passwords, Banners, Port Shutdowns).

! Create user and encrypt all plaintext passwords
SW1(config)# username boson password cisco
SW1(config)# enable secret cisco
SW1(config)# service password-encryption

! Force local authentication for console and VTY lines
SW1(config)# line console 0
SW1(config-line)# login local
SW1(config-line)# line vty 0 15
SW1(config-line)# login local
SW1(config-line)# exit

! Configure a Login Warning Banner (using $ as delimiter)
SW1(config)# banner login $Unauthorized access is prohibited$

! Secure unused ports to prevent unauthorized physical access
SW1(config)# interface range f0/3 - 12
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# shutdown

📝 Lab Walkthrough Checklist

Check off the steps as you complete the configuration in your lab.

Configure exclusions first, then create the pool, assign the network subnet, and configure the default gateway.

Use ip dhcp snooping in global config to turn the feature on, and then explicitly activate it for the active VLANs using ip dhcp snooping vlan 1.

Identify all ports that face the legitimate DHCP server (or uplinks to other switches). Enter interface config and run ip dhcp snooping trust.

If PC's are failing to get IP addresses across multiple switches, check the syslog for drops due to "Option 82". Fix it globally on the switches using no ip dhcp snooping information option.

Command Matrix

Command Device/Mode Purpose
ip dhcp snooping Global Config Enables DHCP Snooping globally on the switch.
ip dhcp snooping vlan 1 Global Config Activates DHCP Snooping specifically for VLAN 1.
ip dhcp snooping trust Interface Config Configures an interface as Trusted, allowing it to forward DHCP Offers.
no ip dhcp snooping information option Global Config Prevents switch from dropping valid requests due to Option 82 mismatches.
service password-encryption Global Config Encrypts all plaintext passwords in the running configuration (Type 7).
