Dynamic NAT & PAT

Dynamic NAT & Port Address Translation

Static NAT is inefficient for user access. Learn how Dynamic NAT allocates public IPs from a pool, and how Port Address Translation (PAT / NAT Overload) allows thousands of users to securely share a single public IP address.

🧠 Evolution of NAT

NAT Varieties

  • Static NAT A strict 1-to-1 mapping. Used for servers that must be permanently reachable from the outside. Requires one dedicated public IP per internal server.
  • Dynamic NAT (Pools) Translates internal IPs dynamically to a pool of available public IPs. Still a 1-to-1 mapping temporarily. If the pool runs out of addresses, subsequent internal hosts are blocked from accessing the internet.
  • PAT / NAT Overload Translates thousands of internal IPs to a single public IP. The router tracks individual connections by assigning a unique Layer 4 Source Port (e.g., TCP 1025) to each session in the translation table.

The "Overload" Keyword

Without the overload keyword, an ip nat inside source command performs standard Dynamic NAT. By appending overload, you unlock PAT, saving massive amounts of public IPv4 space. This is how your home router works!

Translation Efficiency: Static vs PAT

Comparing the maximum number of internal hosts that can share a single Public IP address.

Dynamic NAT maps 1 private IP to 1 public IP from the pool. Port Address Translation (PAT) uses port multiplexing to map up to ~65,000 hosts to a single public IP.

💻 Device Configuration Lab

Review the commands to define ACLs, configure NAT pools, and apply PAT (Overload).

Goal: Translate traffic from 172.16.0.0/24 into a small public pool of exactly two IP addresses (100.0.0.1 and 100.0.0.2).

R1> enable
R1# configure terminal

! 1. Define the NAT interfaces
R1(config)# interface g0/1
R1(config-if)# ip nat inside
R1(config-if)# exit
R1(config)# interface g0/0
R1(config-if)# ip nat outside
R1(config-if)# exit

! 2. Create an ACL identifying internal traffic allowed to be translated
R1(config)# access-list 1 permit 172.16.0.0 0.0.0.255

! 3. Create the NAT Pool of public IPs
R1(config)# ip nat pool POOL1 100.0.0.1 100.0.0.2 netmask 255.255.255.0

! 4. Bind the ACL to the Pool (Dynamic NAT)
R1(config)# ip nat inside source list 1 pool POOL1

Goal: Replace the limited pool with Port Address Translation (Overload) so all internal PCs share R1's single public G0/0 IP address (203.0.113.1).

R1> enable
R1# clear ip nat translation *
R1# configure terminal

! The interface settings and ACL 1 remain the same from the previous step.

! 1. Overwrite the NAT command using the 'interface' and 'overload' keywords
! (This maps all traffic matching ACL 1 to the IP of G0/0, using port multiplexing)
R1(config)# ip nat inside source list 1 interface g0/0 overload

! 2. Verify the configuration is active
R1(config)# do show ip nat translations

Answer: PC1 and PC2 succeed, but PC3 fails. Because Dynamic NAT relies on a 1-to-1 mapping and the pool only contained two public IPs, the pool becomes exhausted. PC3's traffic is dropped by R1 because it has no available public IP to translate it to.

Clear the old translations to unlock the NAT engine, then overwrite the command using the `overload` keyword.

R1# clear ip nat translation *
R1# configure terminal
R1(config)# ip nat inside source list 1 interface g0/0 overload

*Note: Entering the new command overwrites the old one. You don't necessarily have to type 'no ip nat inside...' first.
