Packet Lifecycle

The Life of a Packet

Analyze how network devices handle traffic as it moves from source to destination. This module focuses on understanding how Layer 2 (MAC) addresses are rewritten at every router hop, while Layer 3 (IP) addresses remain unchanged end-to-end.

The Routing Rule

  • L3
    IP Addresses (End-to-End): The Source and Destination IP addresses in the packet header do not change as the packet travels across routers. They define the original sender and final receiver.
  • L2
    MAC Addresses (Hop-by-Hop): The Source and Destination MAC addresses in the frame header change at every router. A router strips the old Layer 2 frame and builds a completely new one for the next physical segment.
  • SW
    Switches are Transparent: Layer 2 switches forward frames based on the Destination MAC address but they do not alter the MAC addresses in the header.
💡 BIA vs. Configured MAC Address

When you check a router's interface using show interface g0/0, you will see two MAC addresses:

  • BIA (Burnt-In Address): The physical MAC address permanently assigned to the interface by the manufacturer (Cisco).
  • Configured MAC: The active MAC address. You can override the BIA by using the mac-address command in interface config mode to make it easier to read (e.g., `aaaa.aaaa.aaaa`).

Note: Routers rewrite MAC addresses. Switches do not.

  • A. PC1 → SW1:
    Src: ....1111 (PC1) | Dst: ....aaaa (R1 G0/0)
  • B. SW1 → R1:
    Src: ....1111 (PC1) | Dst: ....aaaa (R1 G0/0) (Unchanged by SW1)
  • C. R1 → R2:
    Src: ....bbbb (R1 G0/1) | Dst: ....cccc (R2 G0/0) (Rewritten by R1)
  • D. R2 → R3:
    Src: ....dddd (R2 G0/1) | Dst: ....eeee (R3 G0/0) (Rewritten by R2)
  • E. R3 → SW2:
    Src: ....ffff (R3 G0/1) | Dst: ....4444 (PC4) (Rewritten by R3)
  • F. SW2 → PC4:
    Src: ....ffff (R3 G0/1) | Dst: ....4444 (PC4) (Unchanged by SW2)

Note: Because PC1 and PC3 are on the same local subnet, the traffic never hits a router. ARP resolves PC3's MAC directly.

  • A. PC1 → SW1:
    Src: ....1111 (PC1) | Dst: ....3333 (PC3)
  • B. SW1 → PC3:
    Src: ....1111 (PC1) | Dst: ....3333 (PC3) (Unchanged by SW1)

Note: This is the exact inverse of Question 1.

  • A. PC4 → SW2:
    Src: ....4444 (PC4) | Dst: ....ffff (R3 G0/1)
  • B. SW2 → R3:
    Src: ....4444 (PC4) | Dst: ....ffff (R3 G0/1) (Unchanged by SW2)
  • C. R3 → R2:
    Src: ....eeee (R3 G0/0) | Dst: ....dddd (R2 G0/1) (Rewritten by R3)
  • D. R2 → R1:
    Src: ....cccc (R2 G0/0) | Dst: ....bbbb (R1 G0/1) (Rewritten by R2)
  • E. R1 → SW1:
    Src: ....aaaa (R1 G0/0) | Dst: ....1111 (PC1) (Rewritten by R1)
  • F. SW1 → PC1:
    Src: ....aaaa (R1 G0/0) | Dst: ....1111 (PC1) (Unchanged by SW1)
