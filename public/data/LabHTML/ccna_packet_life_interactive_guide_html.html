The Life of a Packet

Analyze how network devices handle traffic as it moves from source to destination. This module focuses on understanding how Layer 2 (MAC) addresses are rewritten at every router hop, while Layer 3 (IP) addresses generally remain unchanged end-to-end.

The Routing Rule

  • L3
    IP Addresses (End-to-End): The Source and Destination IP addresses in the packet header do not change as the packet travels across routers (excluding NAT). They define the original sender and final receiver.
  • L2
    MAC Addresses (Hop-by-Hop): The Source and Destination MAC addresses in the frame header change at every router. A router strips the old Layer 2 frame and builds a completely new one for the next physical segment.
  • SW
    Switches are Transparent: Layer 2 switches forward frames based on the Destination MAC address but they do not alter the MAC addresses in the header.
💡 ARP Dependency

Before PC1 can send a ping to a remote network (like PC4), it must know the MAC address of its Default Gateway (R1). It uses ARP (Address Resolution Protocol) to resolve the gateway's IP to its MAC. The first ping often times out while ARP completes.

Note: Routers rewrite MAC addresses. Switches do not.

  • A. PC1 → SW1:
    Src: PC1 | Dst: R1 G0/0 (Default Gateway)
  • B. SW1 → R1:
    Src: PC1 | Dst: R1 G0/0 (Unchanged by SW1)
  • C. R1 → R2:
    Src: R1 G0/1 | Dst: R2 G0/0 (Rewritten by R1)
  • D. R2 → R3:
    Src: R2 G0/1 | Dst: R3 G0/0 (Rewritten by R2)
  • E. R3 → SW2:
    Src: R3 G0/1 | Dst: PC4 (Rewritten by R3)
  • F. SW2 → PC4:
    Src: R3 G0/1 | Dst: PC4 (Unchanged by SW2)

Note: Because PC1 and PC3 are on the same local subnet, the traffic never hits a router. ARP resolves PC3's MAC directly.

  • A. PC1 → SW1:
    Src: PC1 | Dst: PC3
  • B. SW1 → PC3:
    Src: PC1 | Dst: PC3 (Unchanged by SW1)

Note: This is the exact inverse of Question 1.

  • A. PC4 → SW2:
    Src: PC4 | Dst: R3 G0/1 (Default Gateway)
  • B. SW2 → R3:
    Src: PC4 | Dst: R3 G0/1

    Src: R3 G0/0 | Dst: R2 G0/1

    Src: R2 G0/0 | Dst: R1 G0/1

    Src: R1 G0/0 | Dst: PC1

Command Matrix

Commands used to verify MAC addresses and routing paths across devices.

Command Mode/Device Purpose
show mac address-table Switch Displays the MAC table mapping learned MACs to physical switch ports.
show arp Router Displays the router's ARP cache mapping IP addresses to MAC addresses.
show interfaces [id] Router/Switch Detailed interface info. Look for "Hardware is..." to find the built-in MAC address (BIA) of the interface.
arp -a Windows PC Displays the PC's ARP table (IP to MAC mapping).
ipconfig /all Windows PC Displays PC network info including its own Physical Address (MAC).
