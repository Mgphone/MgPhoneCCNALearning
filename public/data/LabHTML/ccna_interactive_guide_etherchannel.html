EtherChannel (LAG)

EtherChannel (Link Aggregation)

Combine multiple physical links into a single logical interface. Increase bandwidth and provide redundancy without Spanning Tree Protocol (STP) blocking your redundant links. Learn LACP, PAgP, and Layer 3 EtherChannels.

🧠 Core Concepts: LACP & PAgP

Negotiation Protocols

  • LACP
    IEEE 802.3ad (Open Standard): Uses modes active (initiates) and passive (waits). Supported by almost all vendors.
  • PAgP
    Cisco Proprietary: Uses modes desirable (initiates) and auto (waits). Only supported on Cisco hardware.
  • Static
    Unconditional (On): Uses mode on. No negotiation packets are sent. Risky if the other side is misconfigured.

The EtherChannel Rule

All physical ports bundled in an EtherChannel must have identical configurations. This includes Speed, Duplex, Native VLAN, Allowed VLANs, and Switchport Mode.

Once the port-channel is created, always apply configuration changes to the logical interface port-channel [number], not the physical interfaces.

STP vs. EtherChannel (4x 1Gbps Links)

How STP treats redundant links vs. how EtherChannel treats them.

Without EtherChannel, STP blocks 3 out of 4 links to prevent loops, yielding only 1 Gbps. EtherChannel bundles them logically so STP sees one 4 Gbps link.

💻 Device Configuration Lab

Review the commands to configure LACP, PAgP, Static, and Layer 3 EtherChannels.

Goal: Configure a Layer 2 LACP EtherChannel to SW2, initiating the negotiation (Active), and a Static EtherChannel to SW3.

SW1> enable
SW1# configure terminal

! Group 1: LACP Active (To SW2)
SW1(config)# interface range f0/1 - 2
SW1(config-if-range)# channel-group 1 mode active
Creating a port-channel interface Port-channel 1
SW1(config-if-range)# exit

! Configure the logical Port-Channel as a Trunk
SW1(config)# interface port-channel 1
SW1(config-if)# switchport mode trunk
SW1(config-if)# exit

! Group 2: Static / On (To SW3)
SW1(config)# interface range f0/3 - 4
SW1(config-if-range)# channel-group 2 mode on
SW1(config-if-range)# exit
SW1(config)# interface port-channel 2
SW1(config-if)# switchport mode trunk

Goal: Configure an LACP EtherChannel (Passive) to SW1, and a PAgP EtherChannel (Desirable) to SW3.

SW2> enable
SW2# configure terminal

! Group 1: LACP Passive (Responding to SW1)
SW2(config)# interface range f0/1 - 2
SW2(config-if-range)# channel-group 1 mode passive
SW2(config-if-range)# exit
SW2(config)# interface port-channel 1
SW2(config-if)# switchport mode trunk
SW2(config-if)# exit

! Group 3: PAgP Desirable (Initiating to SW3)
SW2(config)# interface range g0/1 - 2
SW2(config-if-range)# channel-group 3 mode desirable
SW2(config-if-range)# exit
SW2(config)# interface port-channel 3
SW2(config-if)# switchport mode trunk

Goal: Configure a Layer 3 EtherChannel using a Multilayer Switch. IPs are assigned directly to the logical port-channel.

MLS1> enable
MLS1# configure terminal

! 1. Convert physical ports to Layer 3 before bundling
MLS1(config)# interface range g1/0/1 - 2
MLS1(config-if-range)# no switchport
MLS1(config-if-range)# channel-group 5 mode active
MLS1(config-if-range)# exit

! 2. Assign IP address directly to the Port-Channel
MLS1(config)# interface port-channel 5
MLS1(config-if)# ip address 10.0.0.1 255.255.255.252
MLS1(config-if)# no shutdown

One side must be desirable. The other can be desirable or auto.

! On SW2:

! On SW3:
SW3(config)# interface range g0/1 - 2
SW3(config-if-range)# channel-group 3 mode auto

Use show etherchannel summary to ensure the bundle is formed correctly.

SW1# show etherchannel summary
Flags: D - down P - bundled in port-channel
S - Layer2 U - in use
...
Group Port-channel Protocol Ports
------+-------------+-----------+-----------------------
1 Po1(SU) LACP Fa0/1(P) Fa0/2(P)

Tip: Look for "SU". S = Switched (Layer 2), U = Up/In Use. Look for "P" next to ports, meaning bundled in Port-channel.

📋 Command Reference

🔍
Command Mode Purpose
channel-group [X] mode [active|passive] Interface Config Assigns physical interfaces to an LACP EtherChannel group.
channel-group [X] mode [desirable|auto] Interface Config Assigns physical interfaces to a PAgP EtherChannel group.
channel-group [X] mode on Interface Config Assigns physical interfaces to a Static (un-negotiated) EtherChannel.
interface port-channel [X] Global Config Enters the logical interface to apply global settings (like switchport mode trunk).
show etherchannel summary Privileged EXEC Displays the status, protocol, and bundled ports for all EtherChannels.
port-channel load-balance [method] Global Config Changes the hashing algorithm (e.g. src-mac, dst-ip) used to distribute traffic across links.
