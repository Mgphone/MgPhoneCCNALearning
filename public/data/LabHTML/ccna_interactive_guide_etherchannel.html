Layer 2 & Layer 3 Resilience Jeremy's IT Lab Day 23

Configuring EtherChannel

Combine multiple physical links into one logical connection! In this comprehensive lab, we will configure Layer 2 EtherChannels using LACP and PAgP, transition to Layer 3 EtherChannels for routed links, and manipulate the load-balancing hashing algorithm.

EtherChannel Protocols & Modes

You can manually force interfaces to bundle together (Static), or you can use a negotiation protocol to establish the bundle dynamically.

  • LACP (IEEE 802.3ad) The open standard protocol. Uses modes: Active (actively asks to form a channel) and Passive (waits to be asked).
  • PAgP (Cisco Proprietary) Cisco's proprietary protocol. Uses modes: Desirable (actively asks) and Auto (waits to be asked).
  • Static / Layer 3 Forces the interface into an EtherChannel without negotiation (Mode: On). For a Layer 3 channel, you must use no switchport on the physical interfaces before bundling them.

Load-Balancing Hashing

Traffic isn't round-robin. It's pinned to a specific link based on a math hash (e.g., src-dst-ip).

Device Configurations

Task 1: Configure Layer 2 EtherChannel between ASW1 and DSW1 using LACP, configured as a trunk.

! On ASW1 (Active)
ASW1(config)# interface range g0/1 - 2
ASW1(config-if-range)# channel-protocol lacp
ASW1(config-if-range)# channel-group 1 mode active

! On DSW1 (Active or Passive)
DSW1(config)# interface range g1/0/3 - 4
DSW1(config-if-range)# channel-protocol lacp
DSW1(config-if-range)# channel-group 1 mode active

! Configure the logical Port-Channel as a Trunk on both switches
ASW1(config)# interface port-channel 1
ASW1(config-if)# switchport trunk encapsulation dot1q
ASW1(config-if)# switchport mode trunk

Task 2: Configure Layer 2 EtherChannel between ASW2 and DSW2 using PAgP, configured as a trunk.

! On ASW2 (Desirable)
ASW2(config)# interface range g0/1 - 2
ASW2(config-if-range)# channel-protocol pagp
ASW2(config-if-range)# channel-group 2 mode desirable

! On DSW2 (Desirable or Auto)
DSW2(config)# interface range g1/0/3 - 4
DSW2(config-if-range)# channel-protocol pagp
DSW2(config-if-range)# channel-group 2 mode desirable

! Configure the logical Port-Channel as a Trunk on both switches
ASW2(config)# interface port-channel 2
ASW2(config-if)# switchport trunk encapsulation dot1q
ASW2(config-if)# switchport mode trunk

Task 3: Configure Layer 3 EtherChannel between DSW1 and DSW2 using static EtherChannel.

! On DSW1 (Convert physical ports to Layer 3 BEFORE bundling)
DSW1(config)# interface range g1/0/1 - 2
DSW1(config-if-range)# no switchport
DSW1(config-if-range)# channel-group 3 mode on

! Assign an IP address directly to the logical port-channel
DSW1(config)# interface port-channel 3
DSW1(config-if)# ip address 10.0.0.1 255.255.255.252

! Repeat on DSW2
DSW2(config)# interface range g1/0/1 - 2
DSW2(config-if-range)# no switchport
DSW2(config-if-range)# channel-group 3 mode on
DSW2(config)# interface port-channel 3
DSW2(config-if)# ip address 10.0.0.2 255.255.255.252

Tasks 4, 5, 6: Enable IP Routing, add static routes, and modify EtherChannel Load-Balancing.

! Task 4: Enable routing and configure static routes to reach remote LANs
DSW1(config)# ip routing
DSW1(config)# ip route 172.16.2.0 255.255.255.0 10.0.0.2

DSW2(config)# ip routing
DSW2(config)# ip route 172.16.1.0 255.255.255.0 10.0.0.1

! Task 5: Check default load-balancing method
DSW1# show etherchannel load-balance
EtherChannel Load-Balancing Configuration:
src-mac

! Task 6: Configure load-balancing based on source and destination IPs on ALL switches
ASW1(config)# port-channel load-balance src-dst-ip
ASW2(config)# port-channel load-balance src-dst-ip
DSW1(config)# port-channel load-balance src-dst-ip
DSW2(config)# port-channel load-balance src-dst-ip

📝 Lab Walkthrough Checklist

Complete these 6 specific lab tasks requested in the scenario.

Enter the interface range connecting ASW1 to DSW1 (g0/1 - 2 on ASW1, and g1/0/3 - 4 on DSW1). Use channel-group 1 mode active to initiate LACP. Then, enter interface port-channel 1 and configure it as a trunk.

Enter the interface range connecting ASW2 to DSW2 (g0/1 - 2 on ASW2, and g1/0/3 - 4 on DSW2). Use channel-group 2 mode desirable to initiate PAgP. Enter interface port-channel 2 and configure it as a trunk.

You MUST use no switchport on the physical interfaces first (interface range g1/0/1 - 2). Then use channel-group 3 mode on. Finally, assign an IP address directly to interface port-channel 3.

Enable ip routing on both DSW1 and DSW2. Then, configure static routes so DSW1 routes to 172.16.2.0/24 via 10.0.0.2, and DSW2 routes to 172.16.1.0/24 via 10.0.0.1.

Use the command show etherchannel load-balance. On most Cisco switches, the default method is src-mac (Source MAC address).

In global configuration mode on ALL switches (ASW1, ASW2, DSW1, and DSW2), enter port-channel load-balance src-dst-ip to apply this algorithm across all EtherChannels on the switches.

Command Matrix

Command Mode Purpose
channel-group [id] mode [mode] Interface Config Assigns physical interfaces to an EtherChannel bundle (active/passive for LACP, desirable/auto for PAgP, on for static).
no switchport Interface Config Converts a Layer 2 switchport into a Layer 3 routed port. Mandatory before creating a L3 EtherChannel.
interface port-channel [id] Global Config Enters the logical interface. IP addresses or trunking commands must be applied here.
port-channel load-balance src-dst-ip Global Config Changes the mathematical hash used to distribute traffic across the physical links.
show etherchannel summary Priv EXEC Shows the state of the port-channels and the status of bundled physical links (e.g., 'SU' or 'RU').
show etherchannel load-balance Priv EXEC Displays the currently active load balancing hash algorithm.
