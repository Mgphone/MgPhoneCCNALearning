Static NAT

Network Address Translation (NAT)

Private IP addresses (RFC 1918) are strictly forbidden from routing across the public Internet. Learn how to configure Static NAT on your edge routers to translate private internal IPs into globally routable public IP addresses.

🧠 Core NAT Terminology

The 4 NAT Terms

Cisco defines IP addresses from the perspective of the router performing the translation.

  • Inside Local The actual private IP address assigned to a host on the internal network (e.g., PC1's 172.16.0.1).
  • Inside Global The public IP address that represents the internal host to the outside world (e.g., 100.0.0.1). This is what the Internet sees.
  • Outside Local The IP address of the destination as it appears to the internal host. (Usually identical to Outside Global).
  • Outside Global The actual public IP address of the external destination (e.g., Google's 8.8.8.8).

Static Source NAT

In this lab, we configure a 1-to-1 Static Mapping. Every single internal PC requires its own dedicated public IP address. While inefficient for large networks (which use PAT), it is vital for hosting internal servers that must be reachable from the internet.

Translation Efficiency: Static vs PAT

Comparing the maximum number of internal hosts that can share a single Public IP address.

Static NAT maps exactly 1 private IP to 1 public IP. Port Address Translation (PAT) uses source port numbers to map up to ~65,000 internal hosts to a single public IP.

💻 Device Configuration Lab

Review the commands to define NAT interfaces and build 1-to-1 static mappings.

Goal: Designate the trusted LAN as "inside", the untrusted internet as "outside", and map three private IPs to three public IPs.

R1> enable
R1# configure terminal

! 1. Define the NAT interfaces
R1(config)# interface g0/1
R1(config-if)# ip nat inside
R1(config-if)# exit
R1(config)# interface g0/0
R1(config-if)# ip nat outside
R1(config-if)# exit

! 2. Create the Static Mappings (Inside Local -> Inside Global)
! Syntax: ip nat inside source static [Private IP] [Public IP]
R1(config)# ip nat inside source static 172.16.0.1 100.0.0.1
R1(config)# ip nat inside source static 172.16.0.2 100.0.0.2
R1(config)# ip nat inside source static 172.16.0.3 100.0.0.3

! 3. Verify the mappings were installed
R1(config)# do show ip nat translations
Pro Inside global Inside local Outside local Outside global
--- 100.0.0.1 172.16.0.1 --- ---
--- 100.0.0.2 172.16.0.2 --- ---
--- 100.0.0.3 172.16.0.3 --- ---

📋 Command Reference

🔍
Command Mode Purpose
ip nat inside Interface Config Designates the interface as facing the trusted, private internal network.
ip nat outside Interface Config Designates the interface as facing the untrusted, public external network.
ip nat inside source static [loc] [glb] Global Config Creates a permanent 1-to-1 mapping between a private IP (local) and a public IP (global).
show ip nat translations Privileged EXEC Displays the active NAT table, showing inside/outside local and global mappings.
show ip nat statistics Privileged EXEC Displays hit counts, configured inside/outside interfaces, and total active translations.
