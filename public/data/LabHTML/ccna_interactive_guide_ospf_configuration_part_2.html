🧠 Modern OSPF Tuning

Configuration Upgrades

  • Interface-Level OSPF Instead of using complex wildcard masks with the network command, modern IOS allows you to simply enter an interface and type ip ospf 1 area 0. This is much less prone to human error.
  • Passive-Interface Default Instead of manually making 50 LAN interfaces passive, you can make every interface passive by default, and then selectively use no passive-interface on the few uplinks connected to other routers.
  • The Reference Bandwidth Problem By default, OSPF's formula (100 Mbps / Bandwidth) gives both 100M and 1000M links a cost of 1. You must increase the reference bandwidth globally so OSPF can differentiate between fast and ultra-fast links.

OSPF Cost Re-calculation

How changing Reference Bandwidth to 10,000 fixes the metric flaw.

New Formula: Cost = 10,000 Mbps / Interface Bandwidth.
Now, FastEthernet (100M) costs 100, while Gigabit (1000M) costs 10. OSPF can now accurately choose the fastest path!

Advanced Topology Detail: OSPF External Routes (O*E2)

When you inject a default route into OSPF, it usually appears as an External Type 2 (E2) route. E2 routes have a quirky behavior: their metric does not increase as they cross the network. If R1 injects a route with cost 1, R4 sees the cost as 1, regardless of how many slow links are in between. This often causes unexpected load-balancing in simulations like Packet Tracer.

💻 Device Configuration Lab

Review the commands to enable interface-level OSPF and manipulate the auto-cost reference bandwidth.

Goal: Configure OSPF directly on interfaces using comma-separated ranges. Adjust the reference bandwidth and inject a default route.

R1> enable
R1# configure terminal

! Use a comma to configure multiple dissimilar interfaces at once
R1(config)# interface range g0/0 , f1/0 , l0
! Enable OSPF Process 1 in Area 0 directly on the interface
R1(config-if-range)# ip ospf 1 area 0
R1(config-if-range)# exit

R1(config)# router ospf 1
R1(config-router)# passive-interface l0

! Fix the metric calculation so Gigabit is preferred over FastEthernet
R1(config-router)# auto-cost reference-bandwidth 10000
% OSPF: Reference bandwidth is changed.
        Please ensure reference bandwidth is consistent across all routers.

! Inject static default route into OSPF
R1(config-router)# default-information originate

Goal: Use passive-interface default for strict security, then manually re-enable OSPF Hellos on uplinks.

R4> enable
R4# configure terminal
R4(config)# interface range g0/0 , f1/0 , f2/0 , l0
R4(config-if-range)# ip ospf 1 area 0
R4(config-if-range)# exit

R4(config)# router ospf 1

! Lock down the router by making EVERY interface passive
R4(config-router)# passive-interface default
%OSPF-5-ADJCHG: Process 1, Nbr 2.2.2.2 on FastEthernet1/0 from FULL to DOWN
%OSPF-5-ADJCHG: Process 1, Nbr 3.3.3.3 on FastEthernet2/0 from FULL to DOWN

! Bring neighbor adjacencies back up securely
R4(config-router)# no passive-interface f1/0
R4(config-router)# no passive-interface f2/0

! Update metric calculation
R4(config-router)# auto-cost reference-bandwidth 10000

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

R1(config)# interface range g0/0 , f1/0 , l0
R1(config-if-range)# ip ospf 1 area 0

Explanation: This is a newer, preferred method for enabling OSPF. It eliminates the risk of miscalculating wildcard masks. Notice the use of commas in interface range to select completely different interface types simultaneously.

R4(config)# router ospf 1
R4(config-router)# passive-interface default
R4(config-router)# no passive-interface f1/0
R4(config-router)# no passive-interface f2/0

Explanation: In a production network with 50 local LAN segments and only 2 uplinks to other routers, it is much faster and safer to set everything to passive, and explicitly un-passive the 2 uplinks. This guarantees you won't accidentally leak OSPF hellos onto a new user VLAN.

To make a FastEthernet interface calculate to a cost of 100, we must change the global numerator to 10,000 Mbps.

R1(config-router)# auto-cost reference-bandwidth 10000

Crucial: You must type this exact command on every single OSPF router in the domain. If routers have different reference bandwidths, routing loops can occur!
