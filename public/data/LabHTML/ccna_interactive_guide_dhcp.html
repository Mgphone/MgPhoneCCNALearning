DHCP Configuration

Dynamic Host Configuration Protocol (DHCP)

Automate IP address distribution on your network. Learn how to configure Cisco routers as DHCP servers, set up address pools, exclude static IPs, configure router interfaces as DHCP clients, and utilize DHCP Relay Agents to bridge broadcast domains.

🧠 The DORA Process & Relay Mechanics

The D.O.R.A. Handshake

  • 1. Discover (Broadcast) The client sends a broadcast message to find an available DHCP server on the local subnet.
  • 2. Offer (Unicast/Broadcast) The server replies with an IP address lease offer, including subnet mask, gateway, and DNS info.
  • 3. Request (Broadcast) The client broadcasts an acceptance of the offered IP address. (Broadcast so other servers know their offers were rejected).
  • 4. Acknowledgment (Unicast/Broadcast) The server formally finalizes the lease and the client begins using the IP address.
The Problem with Broadcasts

Routers drop broadcast packets by default. If a PC is on VLAN 10, but the DHCP server is on a remote subnet, the Discover broadcast will die at the local router. To fix this, we use the ip helper-address command to turn the local router into a DHCP Relay Agent.

💻 Device Configuration Lab

Review the commands to exclude addresses, build DHCP pools, configure relay agents, and assign DHCP to interfaces.

Goal: Configure R2 as the centralized DHCP server for the entire topology. Exclude static IPs first, then build the pools.

R2> enable
R2# configure terminal

! Crucial: Exclude addresses BEFORE creating pools to prevent IP conflicts
R2(config)# ip dhcp excluded-address 192.168.1.1 192.168.1.10
R2(config)# ip dhcp excluded-address 192.168.2.1 192.168.2.10
R2(config)# ip dhcp excluded-address 203.0.113.1

! Pool 1 for 192.168.1.0/24 Subnet
R2(config)# ip dhcp pool POOL1
R2(dhcp-config)# network 192.168.1.0 255.255.255.0
R2(dhcp-config)# dns-server 8.8.8.8
R2(dhcp-config)# domain-name jeremysitlab.com
R2(dhcp-config)# default-router 192.168.1.1
R2(dhcp-config)# exit

! Pool 3 for Router Link (Assigns an IP to R1)
R2(config)# ip dhcp pool POOL3
R2(dhcp-config)# network 203.0.113.0 255.255.255.252
R2(dhcp-config)# exit

Goal: Obtain a dynamic IP for the uplink to R2, and configure the LAN interface to relay DHCP broadcasts to R2.

R1> enable
R1# configure terminal

! Configure R1 as a DHCP Client on the WAN link
R1(config)# interface g0/0
R1(config-if)# ip address dhcp
R1(config-if)# no shutdown
*Mar 2 00:05:12.331: %DHCP-6-ADDRESS_ASSIGN: Interface GigabitEthernet0/0 assigned DHCP address 203.0.113.2, mask 255.255.255.252

! Configure R1 as a DHCP Relay Agent for the local LAN
! Applied to the interface facing the client PCs
R1(config-if)# interface g0/1
R1(config-if)# ip helper-address 203.0.113.1
R1(config-if)# exit

📝 Lab Tasks & Solutions

Exclusions are placed in Global Config mode. Pools are defined using sub-commands for network, default-router, and dns-server.

R2(config)# ip dhcp excluded-address 192.168.1.1 192.168.1.10
R2(config)# ip dhcp pool POOL1
R2(dhcp-config)# network 192.168.1.0 255.255.255.0
R2(dhcp-config)# default-router 192.168.1.1
R2(dhcp-config)# dns-server 8.8.8.8

Instead of manually typing an IP, instruct the interface to request one.

R1(config)# interface g0/0
R1(config-if)# ip address dhcp
R1(config-if)# no shutdown

Apply the ip helper-address command to the interface facing the local clients (G0/1), pointing to the remote DHCP server's IP.

R1(config)# interface g0/1
R1(config-if)# ip helper-address 203.0.113.1

From the Windows Command Prompt on the PC:

C:\> ipconfig /renew
IP Address......................: 192.168.1.11
Subnet Mask.....................: 255.255.255.0
Default Gateway.................: 192.168.1.1

C:\> ipconfig /all

Check ipconfig /all to verify the DNS server and Domain Name were applied successfully.

As demonstrated in the Boson NetSim troubleshooting section, Cisco routers have a global service command that must be enabled. It is enabled by default, but if someone typed no service dhcp, the router will silently drop all DHCP messages.

! To fix:
R2(config)# service dhcp

📋 Command Reference

🔍
Command Mode Purpose
ip dhcp excluded-address [ip] [end-ip] Global Config Prevents specific IPs (like gateways) from being handed out by the server.
ip dhcp pool [name] Global Config Creates a DHCP pool and enters DHCP configuration mode.
network [ip] [mask] DHCP Config Defines the range of addresses the server is allowed to lease out.
default-router [ip] DHCP Config Specifies the default gateway IP to give to the clients.
ip address dhcp Interface Config Configures the router interface to act as a DHCP client and request an IP.
ip helper-address [server-ip] Interface Config Turns the interface into a Relay Agent, converting broadcasts to unicast toward the server.
show ip dhcp binding Privileged EXEC Displays a table mapping assigned IP addresses to client MAC addresses.
