Cisco Discovery Protocol & LLDP

Learn how to use Layer 2 discovery protocols to map out an unknown network. You will master both Cisco's proprietary CDP and the vendor-neutral LLDP (802.1AB), including how to enable, disable, and tune their timers for optimal network awareness and security.

🧠 Neighbor Discovery Concepts

CDP vs. LLDP

  • Cisco Discovery Protocol (CDP) Cisco proprietary. Enabled globally on Cisco devices by default. It sends multicast advertisements to 01:00:0C:CC:CC:CC to discover directly connected Cisco neighbors.
  • Link Layer Discovery Protocol (LLDP) IEEE 802.1AB standard (Vendor-neutral). Often disabled globally by default on Cisco equipment. It is essential when your network incorporates devices from multiple manufacturers (e.g., Juniper, HP, Arista).
  • Security Implications Discovery protocols leak detailed information (device model, IP address, OS version). It is a major security best practice to disable them on edge interfaces connected to PCs, servers, or untrusted networks (the Internet).

Default Protocol Timers

Comparing the default Hello (Message) and Hold timers.

The Hold Timer dictates how long a device will remember a neighbor after it stops receiving Hello messages before flushing it from the table.

💻 Device Configuration Lab

Review the commands to migrate from CDP to LLDP, and how to securely disable them on edge interfaces.

Goal: Disable the proprietary CDP protocol globally and switch the router to the open-standard LLDP. Explicitly enable LLDP Tx/Rx on the connecting interfaces.

R1> enable
R1# configure terminal

! Disable CDP globally
R1(config)# no cdp run

! Enable LLDP globally
R1(config)# lldp run

! Enable LLDP transmission and reception on the uplink interfaces
! (Note: Depending on IOS version, 'lldp run' may auto-enable this)
R1(config)# interface range g0/0 - 2
R1(config-if-range)# lldp transmit
R1(config-if-range)# lldp receive
R1(config-if-range)# end

! Verify LLDP Neighbors
R1# show lldp neighbors

Goal: Disable discovery protocols on the FastEthernet port connected to PC1 to prevent information leakage.

SW1> enable
SW1# configure terminal

! Navigate to the interface connected to PC1
SW1(config)# interface f0/10

! Disable CDP on this specific interface ONLY
SW1(config-if)# no cdp enable

! Verify CDP is disabled (Output should be blank)
SW1(config-if)# do show cdp interface f0/10

! If migrating to LLDP, apply the same logic to prevent LLDP transmission
SW1(config-if)# no lldp transmit

Goal: Tune the LLDP timers to speed up discovery updates. (As seen in the Boson NetSim portion of the video).

R1> enable
R1# configure terminal

! View current timers (Default: Timer=30s, Holdtime=120s)
R1(config)# do show lldp

! Cut the timers in half for faster updates
R1(config)# lldp timer 15
R1(config)# lldp holdtime 60

R1(config)# do show lldp
Global LLDP Information:
    Status: ACTIVE
    LLDP advertisements are sent every 15 seconds
    LLDP hold time advertised is 60 seconds

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

Open the Command Prompt on the PC and use the Windows IP command.

C:\> ipconfig

Result: You will discover PC1's IP is 192.168.1.1 and its Default Gateway is 192.168.1.254 (which implies R1's IP address).

Use the standard neighbor command in Privileged EXEC mode.

R1# show cdp neighbors

This command reveals the Device ID (Hostname), Local Interface, and Port ID (Remote Interface) of adjacent Cisco equipment.

Important: CDP only works between Cisco devices. It will NOT show PC1. To find which port PC1 connects to on SW1, you must use show interfaces status and look for the 'connected' edge port.

If show cdp neighbors reveals R3 is attached to G0/0, use the detail command or query the specific entry.

R1# show cdp entry R3
! OR
R1# show cdp neighbors detail

This will display detailed Layer 3 information, including R3's configured IP address.

In Global Configuration mode, disable the proprietary protocol and enable the IEEE standard.

R1(config)# no cdp run
R1(config)# lldp run

To verify, use show lldp neighbors.

📋 Command Reference

🔍
Command Mode Purpose
show cdp neighbors Privileged EXEC Displays basic info about directly connected Cisco devices (Hostname, Ports).
show cdp entry [name] Privileged EXEC Displays detailed info for a specific neighbor, including its IP address and iOS version.
no cdp run Global Config Completely disables CDP globally on the entire device.
no cdp enable Interface Config Disables CDP on a specific interface only (e.g., ports facing PCs).
lldp run Global Config Enables LLDP globally on the device.
lldp transmit / lldp receive Interface Config Explicitly enables sending/receiving LLDP frames on an interface.
