Interfaces & Security

Configuring Interfaces & Port Security

This module focuses on practical configuration of router and switch interfaces, expanding beyond basic IP addressing to include physical layer tuning. You will learn to manually set speed/duplex for stable uplinks, use bulk configuration commands, and secure network infrastructure by disabling unused switch ports.

Interface Default States

Understanding out-of-the-box behavior is critical for troubleshooting:

  • ▪
    Routers: All interfaces are administratively down by default. You must explicitly issue the no shutdown command.
  • ▪
    Switches: All interfaces are up by default. For security, explicitly issue the shutdown command on unused ports.

Auto-Negotiation vs. Manual

By default, Ethernet interfaces negotiate speed (10/100/1000) and duplex (Half/Full) dynamically.

Uplinks (Router-to-Switch):

Engineers hardcode speed and duplex settings to prevent negotiation failures that cause flapping.

Edge Ports (to PCs):

Typically left to auto-negotiate, as end-user hardware varies wildly.

⚠️ CCNA Exam Tip & Packet Tracer Quirk

In real Cisco IOS, manually configuring speed 1000 and duplex full displays exactly that in show interfaces status. In Packet Tracer, it sometimes incorrectly displays a-1000 and a-full (auto). Always trust the running-config.

Step-by-Step Configuration Guide

Select a device to view its specific CLI configuration sequence.

Goal: Configure hostname, active uplink to SW1 (with speed/duplex), and document unused ports.

R1> enable
R1# configure terminal
R1(config)# hostname R1

! Configure the active interface to SW1
R1(config)# interface g0/0
R1(config-if)# ip address 172.16.0.254 255.255.0.0
R1(config-if)# speed 1000
R1(config-if)# duplex full
R1(config-if)# description to switch 1
R1(config-if)# no shutdown
R1(config-if)# exit

! Add descriptions to unused ports
R1(config)# interface range g0/1 - 2
R1(config-if-range)# description not in use
R1(config-if-range)# end
R1# copy running-config startup-config

Goal: Set hostname, configure uplinks to R1/SW2, label edge ports, and securely shut down all unused ports using ranges.

Switch> enable
Switch# configure terminal
Switch(config)# hostname SW1

! Configure uplink to R1
SW1(config)# interface g0/1
SW1(config-if)# speed 1000
SW1(config-if)# duplex full
SW1(config-if)# description to R1

! Configure uplink to SW2
SW1(config-if)# interface g0/2
SW1(config-if)# speed 1000
SW1(config-if)# duplex full
SW1(config-if)# description to switch 2

! Configure edge ports
SW1(config-if)# interface range f0/1 - 2
SW1(config-if-range)# description to end hosts
SW1(config-if-range)# exit

! Secure unused ports
SW1(config)# interface range f0/3 - 24
SW1(config-if-range)# description not in use
SW1(config-if-range)# shutdown
SW1(config-if-range)# end
SW1# write memory

Goal: Similar to SW1, but demonstrates combining different interface types (Gigabit and FastEthernet) in a single range command.

Switch> enable
Switch# configure terminal
Switch(config)# hostname SW2

! Configure uplink to SW1
SW2(config)# interface g0/1
SW2(config-if)# speed 1000
SW2(config-if)# duplex full
SW2(config-if)# description to switch 1

! Configure edge ports
SW2(config-if)# interface range f0/1 - 2
SW2(config-if-range)# description to end hosts
SW2(config-if-range)# exit

! Secure unused ports (combining Gigabit and FastEthernet)
SW2(config)# interface range g0/2 , f0/3 - 24
SW2(config-if-range)# description not in use
SW2(config-if-range)# shutdown
SW2(config-if-range)# end
SW2# write

Enter Global Configuration mode on each respective device and apply the hostname.

Router(config)# hostname R1
Switch(config)# hostname SW1
Switch(config)# hostname SW2

The network used in the lab is 172.16.0.0 /16.

  • R1 (G0/0): 172.16.0.254 255.255.0.0
  • PC1: 172.16.0.1 255.255.0.0 (Gateway: 172.16.0.254)
  • PC2: 172.16.0.2 255.255.0.0 (Gateway: 172.16.0.254)
  • PC3: 172.16.0.3 255.255.0.0 (Gateway: 172.16.0.254)
  • PC4: 172.16.0.4 255.255.0.0 (Gateway: 172.16.0.254)

We only apply these commands to the Router-to-Switch and Switch-to-Switch uplinks (the Gigabit Ethernet ports).

R1(config-if)# interface g0/0
R1(config-if)# speed 1000
R1(config-if)# duplex full

SW1(config-if)# interface g0/1
SW1(config-if)# speed 1000
SW1(config-if)# duplex full
SW1(config-if)# interface g0/2
SW1(config-if)# speed 1000
SW1(config-if)# duplex full

SW2(config-if)# interface g0/1
SW2(config-if)# speed 1000
SW2(config-if)# duplex full

You should label uplinks based on their destination, and label edge ports as end hosts. Add "not in use" to unconnected ports.

! Examples:
R1(config-if)# description to switch 1
SW1(config-if)# description to switch 2
SW2(config-if)# interface range f0/1 - 2
SW2(config-if-range)# description to end hosts

For routers, interfaces are off by default. However, for switches, unused interfaces remain active and pose a security risk. Use the interface range command to efficiently shut them down.

! On SW1
SW1(config)# interface range f0/3 - 24
SW1(config-if-range)# shutdown

! On SW2 (Note the comma to group different port types)
SW2(config)# interface range g0/2 , f0/3 - 24
SW2(config-if-range)# shutdown

Verification Checklist

To verify the interface states on your switches, use show interfaces status.

  • Look at the Name column to verify your descriptions.
  • Look at the Status column. Unused ports should say disabled. Active ports should say connected.
  • Verify the Speed and Duplex columns match your manual configs.
