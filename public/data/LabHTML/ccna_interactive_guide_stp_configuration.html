STP Configuration & Timers

Learn how to manipulate the Spanning Tree Protocol to force your core switches to become the Root Bridge. We will also analyze the 50-second convergence delay introduced by classic 802.1D STP transitional states.

🧠 Core Concepts: Timers & Configuration

The STP Timers

  • 2 sec
    Hello Timer: How often the Root Bridge sends Configuration BPDUs to maintain the topology.
  • 15 sec
    Forward Delay: The time a port spends in the Listening state, and then again in the Learning state (30 seconds total).
  • 20 sec
    Max Age: How long a switch will hold onto a BPDU before discarding it. If a blocking port stops receiving BPDUs for 20s, it assumes the topology changed and begins transitioning to Forwarding.

Manipulating the Root Election

You should never leave the Root Bridge election up to chance (lowest MAC address). You can manually set the priority in increments of 4096.

spanning-tree vlan 1 root primary

*This macro command automatically lowers the switch's priority to 24576 (or lower) to ensure it beats the current root.

802.1D Convergence Time (50 Seconds)

Why it takes a disconnected port 50s to start forwarding data.

This massive delay is the primary reason classic 802.1D Spanning Tree was eventually replaced by Rapid Spanning Tree (802.1w).

💻 Device Configuration Lab

Review the commands to force Root Bridge elections and manipulate port costs to alter traffic paths.

Goal: Ensure SW1 becomes the Root Bridge for VLAN 1 without needing to manually calculate priority numbers.

SW1> enable
SW1# configure terminal

! Method 1: Using the macro command (Best Practice)
SW1(config)# spanning-tree vlan 1 root primary

! Method 2: Manually setting the priority to a lower number
! (Must be in increments of 4096: 0, 4096, 8192, etc.)
SW1(config)# spanning-tree vlan 1 priority 4096

SW1(config)# exit
SW1# show spanning-tree
...
This bridge is the root

Goal: Ensure SW2 takes over as Root Bridge if SW1 fails.

SW2> enable
SW2# configure terminal

! Method 1: Using the macro command
SW2(config)# spanning-tree vlan 1 root secondary
! This automatically sets priority to 28672 (better than default 32768, worse than primary 24576)

! Method 2: Manually setting the priority
SW2(config)# spanning-tree vlan 1 priority 28672

SW2(config)# end

Goal: SW3 currently uses G0/1 as its Root Port. Manually increase the cost of G0/1 to force SW3 to use G0/2 instead.

SW3> enable
SW3# configure terminal

! Verify current state (Assume G0/1 is Root, G0/2 is Altn)
SW3(config)# do show spanning-tree

! Increase the STP cost of the current Root Port to make it less desirable
SW3(config)# interface g0/1
SW3(config-if)# spanning-tree vlan 1 cost 100
SW3(config-if)# end

! Verify the change. G0/2 should now be the Root Port.
SW3# show spanning-tree
...
Gi0/1 Altn BLK 100 128.1 P2p
Gi0/2 Root FWD 4 128.2 P2p

You can use the macro command which dynamically lowers the priority. It usually drops it to 24576.

SW1(config)# spanning-tree vlan 1 root primary

This command drops the priority to 28672. If SW1 goes offline, SW2's priority will be better than the default (32768) of the other switches, allowing it to gracefully take over.

SW2(config)# spanning-tree vlan 1 root secondary

By default, Gigabit links have a cost of 4. By raising the cost of an interface, spanning tree will see that path as "more expensive" and choose an alternate path if one is available.

SW3(config)# interface g0/1
SW3(config-if)# spanning-tree vlan 1 cost 100

📋 Command Reference

🔍
Command Mode Purpose
spanning-tree vlan [id] root primary Global Config Forces the switch to become the root bridge by automatically lowering its priority (usually to 24576).
spanning-tree vlan [id] root secondary Global Config Sets the priority to 28672, making it the backup root bridge.
spanning-tree vlan [id] priority [value] Global Config Manually sets bridge priority. Must be in increments of 4096.
spanning-tree vlan [id] cost [value] Interface Config Manually overrides the default STP port cost to manipulate path selection.
spanning-tree vlan [id] port-priority [val] Interface Config Alters port priority (default 128) to win tiebreakers when costs are equal.
