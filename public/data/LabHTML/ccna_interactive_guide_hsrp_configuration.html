HSRP Configuration

Hot Standby Router Protocol (HSRP)

Guarantee high availability for your end hosts. HSRP is a Cisco-proprietary First Hop Redundancy Protocol (FHRP) that allows multiple physical routers to share a single Virtual IP (VIP) and Virtual MAC address, ensuring that if the active default gateway fails, a standby router immediately takes over.

🧠 HSRP Mechanics & Terminology

Core Components

  • Active & Standby Roles The router with the highest priority (default 100) becomes the Active router. If priorities tie, the highest IP address wins. The Active router responds to ARP requests for the VIP.
  • Virtual IP (VIP) & Virtual MAC PCs are configured to use the VIP as their gateway. The Active router generates a Virtual MAC (e.g., 0000.0c9f.f001 for v2 group 1) linked to this VIP. If a failover occurs, the Standby router assumes control of this exact same MAC and IP.
  • Preemption By default, if a failed Active router comes back online, it does not take its job back. You must explicitly configure standby preempt to force it to reclaim the Active role based on its higher priority.

HSRP Version 1 vs. Version 2

Why we use `standby version 2` in modern networks.

Important: HSRP versions must match across routers! If R1 is v2 and R2 is v1, they cannot read each other's Hello packets. Both will assume the other is dead and claim the "Active" role, causing an IP conflict.

💻 Device Configuration Lab

Review the commands to configure HSRP version 2, set priorities, and establish preemption.

Goal: Assign the Virtual IP (VIP), ensure R1 is elected as the Active router by raising its priority to 200, and ensure it can reclaim the role if it reboots.

R1> enable
R1# configure terminal
R1(config)# interface g0/0

! Enable HSRP Version 2 (Must match on all routers)
R1(config-if)# standby version 2

! Configure the Virtual IP (VIP) for Group 1
R1(config-if)# standby 1 ip 10.0.1.254

! Set priority higher than the default (100) to guarantee it becomes Active
R1(config-if)# standby 1 priority 200

! Enable preemption so R1 takes over again if it crashes and recovers
R1(config-if)# standby 1 preempt

%HSRP-5-STATECHANGE: GigabitEthernet0/0 Grp 1 state Standby -> Active

Goal: Configure R2 to listen for the VIP and remain in standby mode unless R1 fails.

R2> enable
R2# configure terminal
R2(config)# interface g0/0

! Must match R1's version
R2(config-if)# standby version 2

! Must match R1's Virtual IP and Group number
R2(config-if)# standby 1 ip 10.0.1.254

! Lower the priority (Optional, default 100 already loses to R1's 200)
R2(config-if)# standby 1 priority 50

%HSRP-5-STATECHANGE: GigabitEthernet0/0 Grp 1 state Speak -> Standby

Scenario: You forgot to type standby version 2 on R2. R1 is running v2, but R2 is running v1.

! Because the versions don't match, they cannot read each other's Hello packets.
! Both routers assume the other is dead, and BOTH try to become the Active router.

! The console will flood with duplicate IP address warnings:
%IP-4-DUPADDR: Duplicate address 10.0.1.254 on GigabitEthernet0/0, sourced by 0000.0c9f.f001
%IP-4-DUPADDR: Duplicate address 10.0.1.254 on GigabitEthernet0/0, sourced by 0000.0c9f.f001

! To fix it, simply align the versions on R2:
R2(config)# interface g0/0
R2(config-if)# standby version 2
%HSRP-5-STATECHANGE: GigabitEthernet0/0 Grp 1 state Active -> Standby

📝 Lab Tasks & Solutions

Check off the steps as you complete the troubleshooting tasks.

Before any failure, PC1's default gateway is the VIP (10.0.1.254), which is currently held by R1.

C:\> tracert 8.8.8.8
1 1 ms 1 ms 1 ms 10.0.1.253

Notice:

Even though the PC sent the packet to the Virtual IP (.254), the traceroute response comes back from the physical interface IP address of the active router (.253 for R1). This is incredibly useful for network engineers to verify exactly which physical router is currently processing the HSRP traffic.

Answer: Yes.

When R1 is powered off, it stops sending HSRP Hello messages. After the Hold timer expires (default 10 seconds), R2 promotes itself from Standby to Active. It takes over the Virtual MAC and Virtual IP. The ping from the PC drops 1 or 2 packets during this brief window, then resumes successfully.

If you run a traceroute now, the first hop will show 10.0.1.252 (R2's physical IP).

Answer: Run a traceroute from the PC.

Because we explicitly configured standby 1 preempt on R1, the moment it finishes booting and sees R2's priority is lower (50 vs 200), it forces R2 back into Standby mode and reclaims the Active role.

A new tracert 8.8.8.8 from PC1 will immediately show 10.0.1.253 as the first hop again, proving R1 is back in charge.

Answer: Yes, on a Layer 3 Multilayer Switch.

In modern enterprise networks, the default gateway for PCs is often a Switch Virtual Interface (SVI) located on a core or distribution switch, rather than a traditional router. You can configure HSRP directly on the SVIs.

DSwitch1(config)# interface vlan 10
DSwitch1(config-if)# ip address 10.0.10.253 255.255.255.0
DSwitch1(config-if)# standby 10 ip 10.0.10.254

📋 Command Reference

🔍
Command Mode Purpose
standby version 2 Interface Config Enables HSRPv2. Must be typed before other standby commands to ensure compatibility.
standby [grp] ip [address] Interface Config Defines the Virtual IP (VIP) for the specified HSRP group.
standby [grp] priority [0-255] Interface Config Sets election priority. Default is 100. Highest priority becomes Active.
standby [grp] preempt Interface Config Allows a router with a higher priority to immediately overthrow the current Active router.
show standby Privileged EXEC Displays state (Active/Standby), VIP, Virtual MAC, and preemption settings.
tracert [ip] Windows CMD Traces path. Useful for identifying which physical router is currently acting as the VIP gateway.
