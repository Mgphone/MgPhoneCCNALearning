OSI Model Analysis

The OSI Model & Simulation Mode

Network traffic is encapsulated layer by layer. This module explores how to use Packet Tracer's powerful "Simulation Mode" to intercept packets in transit and inspect their Protocol Data Units (PDUs) across the 7 layers of the OSI model.

Encapsulation & The OSI Model

When a device sends data, it is encapsulated with headers as it moves down the OSI model. Different protocols operate at different layers, containing specific addressing information required to move the data across the network.

L2

Layer 2: Data Link

Protocols like STP (Spanning Tree Protocol) operate here. The PDU is a Frame, containing MAC addresses and Ethernet headers (IEEE 802.3).

L3

Layer 3: Network

Protocols like OSPF and IPv4/IPv6 operate here. The PDU is a Packet, containing logical Source and Destination IP addresses.

L7

Layer 7: Application

Protocols like DHCP or HTTP operate here. They rely on the lower layers (like UDP/TCP at Layer 4) for transport.

Protocol Stack Depth Comparison

Visualizing how many OSI layers different protocols utilize when inspected in Simulation Mode.

Notice how Application layer protocols (like DHCP) require a full stack of headers (L1-L4, L7) to successfully transmit data, whereas L2 protocols (STP) only need L1 and L2.

💡 Packet Tracer Quirk: Layers 5 & 6

When inspecting Layer 7 traffic (like DHCP) in Packet Tracer, you will notice that Layers 5 (Session) and 6 (Presentation) are empty. This is because modern networking uses the TCP/IP Model, which combines layers 5, 6, and 7 into a single unified "Application Layer".

Protocol: STP

Spanning Tree Protocol (STP) operates purely at Layer 2 to prevent switching loops.

  • Layer 2: Encapsulates into an IEEE 802.3 Ethernet Frame header with a destination MAC address.
  • Layer 1: Sent out physical switch ports (e.g., FastEthernet0/1).

Note: It does not have an IP header (L3), as it is strictly switch-to-switch communication.

To analyze traffic, click the Simulation tab in the bottom right corner of Packet Tracer, then click "Play" or use the step-forward button. Click on the colored envelopes (PDUs) to inspect them.

  • STP (Spanning Tree Protocol) Uses Layers 1 & 2. It encapsulates data into an IEEE 802.3 Ethernet frame to prevent Layer 2 loops. It does not use IP addresses.
  • OSPF (Open Shortest Path First) Uses Layers 1, 2, & 3. It is a routing protocol, so it encapsulates data into an IP packet with Source and Destination IP addresses to map network paths.

To generate this traffic, open PC1's Desktop tab, open the Command Prompt, and issue the following commands:

C:\> ipconfig /release
C:\> ipconfig /renew

This generates DHCP (Dynamic Host Configuration Protocol) traffic.

DHCP Analysis:
  • Layer 7 (Application): Contains the actual DHCP request data.
  • Layers 5 & 6: Empty (Merged into L7 by the TCP/IP model).
  • Layer 4 (Transport): Encapsulated into a UDP Segment.
  • Layer 3 (Network): Encapsulated into an IP Packet.
  • Layer 2 (Data Link): Encapsulated into an Ethernet Frame.
  • Layer 1 (Physical): Sent out the FastEthernet port.
