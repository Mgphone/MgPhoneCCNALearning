Wireless LAN Controllers (WLC)

Move away from autonomous Access Points! Learn how to manage enterprise wireless networks centrally using a Wireless LAN Controller (WLC). We will configure dynamic interfaces, map them to VLANs, and secure our SSIDs with WPA2-PSK through the WLC GUI.

The Split-MAC Architecture

In a WLC deployment, the responsibilities of managing the wireless network are split between the physical Access Point (Lightweight AP) and the central Controller (WLC).

  • WLC Duties (Control Plane) Handles authentication, security policies, roaming management, and mapping wireless traffic to the correct wired VLANs via Dynamic Interfaces.
  • Access Point Duties (Data Plane) Handles real-time RF (Radio Frequency) duties: transmitting beacons, encrypting/decrypting wireless frames, and managing physical radio connections.
  • CAPWAP Tunnels The protocol that binds the AP to the WLC. It uses UDP ports 5246 (Control traffic) and 5247 (Data traffic). All client data travels through the CAPWAP data tunnel to the WLC before entering the wired network.

📝 Lab Walkthrough Checklist

Check off the steps as you complete them to save your progress.

Configure the Switch as a DHCP server with pools for Management (VLAN 10), Internal (VLAN 100), and Guest (VLAN 200). Use option 43 hex [WLC-IP] to tell booting APs where the controller is located.

From PC1's web browser, navigate to https://172.16.1.10. Ensure you use HTTPS, as HTTP is disabled by default on modern WLCs.

Under the Controller > Interfaces tab, create logical interfaces representing your client VLANs. Assign them an IP address and specify the physical switch port (Port 1).

Under the WLANs tab, create the SSIDs. Map each SSID to the correct Dynamic Interface. Enable WPA2 and enter a Pre-Shared Key (PSK). Crucially: Check the "Enabled" box on the General tab.

Wireless Terminology

Term Category Definition
LAP Hardware Lightweight Access Point. Relies entirely on a WLC for configuration and operation.
CAPWAP Protocol The UDP-based tunneling protocol that connects the LAP to the WLC (Ports 5246/5247).
Dynamic Interface WLC Config A logical interface created on the WLC to map a wireless network (WLAN) to a wired VLAN.
WLAN WLC Config Wireless Local Area Network. This is where you configure the actual SSID that clients connect to.
Option 43 DHCP Parameter A DHCP option configured on the router/switch to tell booting LAPs the IP address of the WLC.
WPA2-PSK Security Wi-Fi Protected Access 2 with a Pre-Shared Key. The standard password-based security for basic networks.
