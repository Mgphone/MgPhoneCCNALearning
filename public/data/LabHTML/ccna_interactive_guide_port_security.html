Port Security

Port Security (Layer 2)

Prevent rogue devices and DHCP exhaustion attacks. Learn how to configure port security to restrict switch access based on MAC addresses, understand learning methods (Static, Dynamic, Sticky), and manage security violation modes.

Violation Modes Compared

When an unauthorized MAC address attempts to access a secured port (or the maximum allowed MACs is exceeded), the switch takes action based on the configured Violation Mode.

  • Shutdown (Default) The port is placed into an err-disabled state. Traffic drops, syslog is sent, and the port LED turns off. Requires manual intervention to recover.
  • Restrict The port stays UP, but traffic from the offending MAC is dropped. A syslog message is generated and the violation counter increments.
  • Protect The port stays UP. Traffic from the offending MAC is silently dropped. No syslog is sent and counters do not increment.

Device Configuration

Goal: Enable default port security on user-facing ports and set an aging time of 60 minutes.

SW1> enable
SW1# configure terminal

! Interfaces must not be dynamic auto. Hardcode as access ports first!
SW1(config)# interface range fa0/1 - 3
SW1(config-if-range)# switchport mode access

! Turn the feature on (Default: Max 1 MAC, Shutdown Mode)
SW1(config-if-range)# switchport port-security

! Force learned MACs to expire after 60 minutes
SW1(config-if-range)# switchport port-security aging time 60
SW1(config-if-range)# end

! Verification
SW1# show port-security interface fa0/1

Goal: Connect to another switch. Allow up to 4 MACs, drop excess traffic without shutting down, and save learned MACs to config.

SW2# configure terminal
SW2(config)# interface gig0/1
SW2(config-if)# switchport mode access
SW2(config-if)# switchport port-security

! Switch 2 learns the PCs AND Switch 1's CDP MAC address.
SW2(config-if)# switchport port-security maximum 4

! Change violation mode from Shutdown to Restrict
SW2(config-if)# switchport port-security violation restrict

! Automatically write dynamically learned MACs into the running-config
SW2(config-if)# switchport port-security mac-address sticky

By default, switch ports are in dynamic auto mode. Port security requires a static access or trunk port.

SW1(config-if)# switchport mode access
SW1(config-if)# switchport port-security

Sticky learning tells the switch to permanently memorize the first X MAC addresses it sees by injecting them into the running-config.

SW2(config-if)# switchport port-security maximum 4
SW2(config-if)# switchport port-security mac-address sticky

If a violation occurs in the default "Shutdown" mode, the port goes down. You must manually bounce the port to restore it.

! After removing the rogue device:
SW1(config-if)# shutdown
SW1(config-if)# no shutdown

Command Matrix

Command Mode Purpose
switchport port-security Interface Enables port security on the interface.
switchport mode access Interface Forces the port out of dynamic auto mode so security can be applied.
switchport port-security max <1-8192> Interface Sets the maximum number of secure MAC addresses (Default 1).
switchport port-security violation restrict Interface Changes action to drop unauthorized traffic while keeping the port UP.
switchport port-security mac-address sticky Interface Converts dynamically learned MACs into sticky MACs in the config.
show port-security interface fa0/1 Priv EXEC Shows detailed security status, violation counts, and the last offending MAC.
