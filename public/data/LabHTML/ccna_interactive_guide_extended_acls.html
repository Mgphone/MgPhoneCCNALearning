Extended ACLs

Extended Access Control Lists (ACLs)

Control network traffic with high granularity. While Standard ACLs only filter by source IP, Extended ACLs can filter by Source, Destination, Protocol, and Port Number. Learn how to securely permit web traffic while blocking pings or DNS.

🧠 Core Extended ACL Mechanics

Extended Rules of Engagement

  • Granular Filtering Extended ACLs inspect Layer 3 (Source/Dest IP) AND Layer 4 (TCP/UDP Source/Dest Ports). You can block HTTP to a specific server while allowing Ping.
  • Placement: Closest to Source Because Extended ACLs are so precise (specifying the exact destination), you should place them as close to the Source as possible. This drops unwanted traffic immediately, saving network bandwidth.
  • The Implicit Deny Just like Standard ACLs, every Extended ACL ends with an invisible deny ip any any rule. You must explicitly permit ip any any at the end if you want unmentioned traffic to pass.
Port Knowledge is Mandatory

You must know your Layer 4 ports to configure Extended ACLs effectively:

  • HTTP: TCP Port 80
  • HTTPS: TCP Port 443
  • DNS: UDP & TCP Port 53
  • Telnet: TCP Port 23

💻 Device Configuration Lab

Review the commands to configure Extended Numbered and Extended Named ACLs, matching heavily on ports and protocols.

Goal: Block Subnet 1 from accessing DNS on Server 1. Permit all other traffic. Apply inbound on G0/0.

R1> enable
R1# configure terminal

! Extended Numbered ACL (100-199)
! DNS uses both UDP and TCP port 53. We must deny both.
R1(config)# access-list 100 deny udp 172.16.1.0 0.0.0.255 host 192.168.1.100 eq 53
R1(config)# access-list 100 deny tcp 172.16.1.0 0.0.0.255 host 192.168.1.100 eq 53

! Explicitly permit all other IP traffic to override the implicit deny
R1(config)# access-list 100 permit ip any any

! Apply INBOUND as close to the source (Subnet 1) as possible
R1(config)# interface g0/0
R1(config-if)# ip access-group 100 in

Goal: Block Subnet 2 from communicating with PC1, and from accessing HTTP/HTTPS on Server 2. Apply inbound on G0/1.

R1> enable
R1# configure terminal

! Enter Extended Named ACL config mode. (We can use a number here too!)
R1(config)# ip access-list extended 101

! Deny IP traffic from Subnet 2 to the specific host PC1
R1(config-ext-nacl)# deny ip 172.16.2.0 0.0.0.255 host 172.16.1.1

! Deny web traffic (TCP 80 & 443) from Subnet 2 to Server 2
R1(config-ext-nacl)# deny tcp 172.16.2.0 0.0.0.255 host 192.168.2.100 eq 80
R1(config-ext-nacl)# deny tcp 172.16.2.0 0.0.0.255 host 192.168.2.100 eq 443

! Permit everything else
R1(config-ext-nacl)# permit ip any any
R1(config-ext-nacl)# exit

! Apply INBOUND as close to the source (Subnet 2) as possible
R1(config)# interface g0/1
R1(config-if)# ip access-group 101 in

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

We configure ACL 100 on R1 to block port 53 (DNS) for both UDP and TCP from the 172.16.1.0/24 subnet to the host IP of Server 1.

R1(config)# access-list 100 deny udp 172.16.1.0 0.0.0.255 host 192.168.1.100 eq 53
R1(config)# access-list 100 deny tcp 172.16.1.0 0.0.0.255 host 192.168.1.100 eq 53
R1(config)# access-list 100 permit ip any any

We configure ACL 101 to block all IP traffic to PC1's host IP, and block TCP ports 80/443 to Server 2's host IP.

R1(config)# ip access-list extended 101
R1(config-ext-nacl)# deny ip 172.16.2.0 0.0.0.255 host 172.16.1.1
R1(config-ext-nacl)# deny tcp 172.16.2.0 0.0.0.255 host 192.168.2.100 eq 80
R1(config-ext-nacl)# deny tcp 172.16.2.0 0.0.0.255 host 192.168.2.100 eq 443
R1(config-ext-nacl)# permit ip any any

For Extended ACLs, applying them closest to the source means dropping the traffic as it enters the router from the local subnets (inbound).

R1(config)# interface g0/0 ! Facing Subnet 1
R1(config-if)# ip access-group 100 in

R1(config)# interface g0/1 ! Facing Subnet 2
R1(config-if)# ip access-group 101 in
