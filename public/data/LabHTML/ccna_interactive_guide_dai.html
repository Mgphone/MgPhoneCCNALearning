Dynamic ARP Inspection (DAI)

Stop Man-in-the-Middle (MitM) attacks caused by ARP Spoofing/Poisoning. DAI relies on the DHCP Snooping binding database to inspect all ARP packets on untrusted ports, ensuring the Sender IP matches the corresponding Sender MAC address.

DAI Validation & Rate Limiting

Dynamic ARP Inspection works hand-in-hand with DHCP Snooping. Like snooping, it uses the concept of Trusted and Untrusted ports.

  • How it validates (The Database) When an ARP packet enters an untrusted port, DAI checks the Sender MAC and Sender IP against the DHCP Snooping binding table. If there is no match, the packet is dropped and logged.
  • Advanced Validations By default, only the IP-to-MAC mapping is checked. You can enable deeper checks to validate the Destination MAC, Source MAC, and IP fields within the ARP body against the Ethernet header.
  • Rate Limiting (15 PPS) To prevent CPU overload, untrusted ports are rate-limited to 15 ARP packets per second by default. If exceeded, the port is placed into an err-disabled state. (Trusted ports have no limit).

*Note: In Packet Tracer, trusted ports sometimes incorrectly show a 15 PPS limit due to a simulator bug. On real Cisco gear, trusted ports are unrestricted.

Device Configurations

Goal: Set up DHCP Snooping (DAI depends completely on the DHCP Snooping binding table).

! Enable DHCP Snooping globally and per-vlan
SW1(config)# ip dhcp snooping
SW1(config)# ip dhcp snooping vlan 1

! Disable Option 82 to prevent dropped packets (per Jeremy's recommendation)
SW1(config)# no ip dhcp snooping information option

! Trust the port facing the DHCP server
SW1(config)# interface g0/2
SW1(config-if)# ip dhcp snooping trust

Goal: Enable DAI for VLAN 1, enable advanced validation checks, and trust uplinks.

! Enable DAI for the target VLAN
SW1(config)# ip arp inspection vlan 1

! Enable advanced checks in a single line (Dest MAC, IP, Src MAC)
SW1(config)# ip arp inspection validate dst-mac ip src-mac

! Trust uplink ports connecting to other switches or the router
SW1(config)# interface range g0/1 - 2
SW1(config-if-range)# ip arp inspection trust

Goal: Verify the operational state of DAI and review interface trust levels.

📝 Lab Walkthrough Checklist

Check off the steps as you complete the DAI configuration in your lab.

DAI relies completely on the DHCP binding table to know which IP belongs to which MAC. You must enable `ip dhcp snooping` and trust your uplinks before proceeding.

Like snooping, DAI must be explicitly enabled for the VLANs operating on the switch using the command: ip arp inspection vlan 1.

Enter the interfaces connected to other switches or the router, and run ip arp inspection trust. Untrusted ports are for end-devices (PCs) only.

To ensure total ARP integrity, use ip arp inspection validate dst-mac ip src-mac to force the switch to check the ARP payload against the Ethernet header fields.

Command Matrix

Command Device/Mode Purpose
ip arp inspection vlan [id] Global Config Enables Dynamic ARP Inspection for the specified VLAN(s).
ip arp inspection trust Interface Config Configures the interface to bypass ARP inspection. (For uplinks)
ip arp inspection validate src-mac dst-mac ip Global Config Enables advanced payload checking. All three must be entered on the same line to activate all three.
ip arp inspection limit rate [pps] Interface Config Changes the default rate limit of ARP packets allowed per second (Default: 15).
show ip arp inspection Priv EXEC Shows global DAI configuration and which validation checks are active.
