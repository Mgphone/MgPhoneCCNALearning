🧠 The IPv6 Transition

Core Concepts

  • IPv4 Exhaustion IPv4 uses 32-bit addresses, yielding about 4.3 billion unique IPs. This is no longer enough. IPv6 uses 128-bit addresses, offering 3.4 x 1038 addresses (an essentially limitless supply).
  • Dual Stack A migration strategy where devices run both IPv4 and IPv6 simultaneously on the same interfaces. This allows legacy IPv4 communication while enabling modern IPv6 capabilities.
  • The Routing "Gotcha" Cisco routers forward IPv4 packets by default out-of-the-box. However, IPv6 routing is disabled by default. You can configure IPv6 addresses on interfaces, but the router will drop traffic crossing subnets unless you globally enable it.

💻 Device Configuration Lab

Review the commands to configure IPv6 addresses on interfaces and enable global routing.

Goal: Assign IPv6 addresses to the interfaces, implementing a Dual Stack (IPv4 is already running). Enable IPv6 routing.

R1> enable
R1# configure terminal

! Assign IPv6 address to G0/0 (Subnet 1)
! Notice the double colon '::' compresses consecutive zeros
R1(config)# interface g0/0
R1(config-if)# ipv6 address 2001:db8:0:1::1/64
! No 'no shutdown' needed; interface is already up from IPv4 config
R1(config-if)# exit

! Assign IPv6 address to G0/1 (Subnet 2)
R1(config)# interface g0/1
R1(config-if)# ipv6 address 2001:db8:0:2::1/64
R1(config-if)# exit

! Assign IPv6 address to G0/2 (Subnet 3)
R1(config)# interface g0/2
R1(config-if)# ipv6 address 2001:db8:0:3::1/64
R1(config-if)# exit

! Crucial: Enable global IPv6 packet forwarding
R1(config)# ipv6 unicast-routing

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

We use the ipv6 address command. This runs concurrently with the existing IPv4 addresses, establishing a Dual Stack.

R1(config)# interface g0/0
R1(config-if)# ipv6 address 2001:db8:0:1::1/64
R1# show ipv6 interface brief
GigabitEthernet0/0 [up/up]
    FE80::201:C9FF:FE01:0101
    2001:DB8:0:1::1

Note: As soon as you enable IPv6 on an interface, the router automatically generates a Link-Local address (starting with FE80). These are used for internal protocol communications on the local segment.

PC1> ping 2001:db8:0:2::2
Request timed out.

Explanation: PC1 can ping its local gateway (R1 G0/0), but it cannot ping PC2. This is because IPv6 routing is disabled by default on Cisco routers. R1 receives the IPv6 packet, sees it is destined for a different subnet, and drops it because it isn't allowed to route IPv6 yet.

R1(config)# ipv6 unicast-routing

PC1> ping 2001:db8:0:2::2
Reply from 2001:db8:0:2::2: bytes=32 time=1ms

Success! R1 now acts as an IPv6 gateway, forwarding packets across subnets.
