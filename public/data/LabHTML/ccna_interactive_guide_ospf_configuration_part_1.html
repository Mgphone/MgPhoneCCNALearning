OSPF Config (1)

Open Shortest Path First (OSPF)

Learn to configure single-area OSPF (Area 0). This lab covers essential configurations including loopback interfaces, wildcard mask variations, passive interfaces, and injecting default routes into the OSPF domain.

🧠 OSPF Configuration Mechanics

Core Components

  • Process ID Unlike EIGRP's AS number, the OSPF Process ID (e.g., router ospf 1) is locally significant. Two routers can have different Process IDs and still become neighbors.
  • Wildcard Masks OSPF requires wildcard masks to enable routing on interfaces. You can be broad (0.0.0.0 255.255.255.255 matches everything) or exact (10.1.1.1 0.0.0.0 matches only that specific IP).
  • Passive Interfaces OSPF sends Hello packets to dynamically find neighbors. You should configure interfaces connected to PCs, Loopbacks, or ISPs as passive-interface to stop wasting CPU and bandwidth.
  • Default Information Originate Used on edge routers (ASBRs) to inject a static default route (0.0.0.0/0) into the OSPF domain, telling all internal routers how to reach the Internet.

The OSPF Default Cost "Gotcha"

Why OSPF treats FastEthernet and GigabitEthernet identically by default.

Formula: Cost = Reference Bandwidth / Interface Bandwidth.
By default, the reference bandwidth is 100 Mbps. Since cost cannot be less than 1, both FastEthernet (100M) and Gigabit (1000M) calculate to a cost of 1!

💻 Device Configuration Lab

Review the commands to configure OSPF. Notice the different variations of the network command used across the routers.

Goal: Configure OSPF Process ID 4, use the 0.0.0.0 wildcard shortcut to enable OSPF everywhere, and secure the loopback.

R4> enable
R4# configure terminal

! Create the loopback interface
R4(config)# interface l0
R4(config-if)# ip address 4.4.4.4 255.255.255.255
R4(config-if)# exit

! Initialize OSPF Process (ID is locally significant)
R4(config)# router ospf 4

! Enable OSPF on all active interfaces simultaneously
R4(config-router)# network 0.0.0.0 255.255.255.255 area 0

! Stop sending OSPF Hello packets out the loopback
R4(config-router)# passive-interface l0

Goal: Configure OSPF Process ID 3, use precise host matching (0.0.0.0 wildcard) to enable OSPF, and secure the loopback.

R3> enable
R3# configure terminal
R3(config)# interface l0
R3(config-if)# ip address 3.3.3.3 255.255.255.255
R3(config-if)# exit

R3(config)# router ospf 3

! Precise matching: Only enable OSPF on interfaces with these exact IPs
R3(config-router)# network 10.0.13.2 0.0.0.0 area 0
R3(config-router)# network 10.0.34.1 0.0.0.0 area 0
R3(config-router)# network 3.3.3.3 0.0.0.0 area 0

R3(config-router)# passive-interface l0

Goal: Configure OSPF Process ID 1, use subnet wildcard masks, secure the internet link, and inject a default route into the OSPF domain.

R1> enable
R1# configure terminal
R1(config)# interface l0
R1(config-if)# ip address 1.1.1.1 255.255.255.255
R1(config-if)# exit

! Configure Static Default Route to the ISP
R1(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.2

R1(config)# router ospf 1

! Subnet matching (0.0.0.3 represents a /30 subnet)
R1(config-router)# network 10.0.12.0 0.0.0.3 area 0
R1(config-router)# network 10.0.13.0 0.0.0.3 area 0
R1(config-router)# network 1.1.1.1 0.0.0.0 area 0

! Inject the default route into OSPF so other routers can reach the internet
R1(config-router)# default-information originate

R1(config-router)# passive-interface l0

In OSPF, the highest active IPv4 address automatically becomes the Router ID (the name it uses to identify itself to neighbors). Because physical interfaces can fail, network engineers use stable loopback addresses to ensure the OSPF Router ID never unexpectedly changes.

R1# show ip ospf database

! View ONLY OSPF Routes in the Routing Table:
R1# show ip route ospf
R4# show ip route ospf
O*E2 0.0.0.0/0 [110/1] via 10.0.24.2, FastEthernet1/0
                       via 10.0.34.3, FastEthernet0/0

The "Gotcha":

Even though the path through R2 includes a fast Gigabit link, R4 calculates both paths as completely equal. This is because OSPF's default Reference Bandwidth is 100 Mbps.

Cost = (100 Mbps / Interface Bandwidth). Since a cost cannot be less than 1, FastEthernet (100M) = Cost 1, and GigabitEthernet (1000M) = Cost 1. Thus, both paths have a total cost of 2, and R4 blindly load-balances across both.

To fix this in real life, you must manually change the `auto-cost reference-bandwidth` on all routers to something higher (like 1000 or 100000).
