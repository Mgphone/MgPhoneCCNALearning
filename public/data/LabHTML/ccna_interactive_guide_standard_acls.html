Standard ACLs

Standard Access Control Lists (ACLs)

Control network traffic by selectively permitting or denying packets. Standard ACLs operate purely on the Source IP address. Master configuring Numbered vs. Named ACLs, the Implicit Deny rule, and proper ACL placement.

🧠 Core ACL Rules & Mechanics

The Golden Rules of Standard ACLs

  • Filter By Source IP Only Standard ACLs cannot filter by destination IP, protocol, or port numbers. They only look at where the packet came from.
  • Placement: Closest to Destination Because they only filter by source, placing a Standard ACL near the source router would block that host from reaching any network. Place them on the router closest to the target destination.
  • The Implicit Deny Every ACL ends with an invisible deny any rule. If a packet doesn't match any of your permit statements, it will be automatically dropped. You must have at least one permit statement for an ACL to allow any traffic.

Standard vs. Extended ACL Features

Comparing the filtering capabilities of Standard and Extended Access Lists.

Standard ACL ranges: 1-99 & 1300-1999.
Extended ACL ranges: 100-199 & 2000-2699.

💻 Device Configuration Lab

Review the commands to configure Standard Numbered and Named ACLs, apply them to interfaces, and set up OSPF.

Goal: Restrict traffic to Server 1 (permit only PC1/PC3) and Server 2 (deny 172.16.2.0/24 subnet). Uses Standard Named ACLs.

R2> enable
R2# configure terminal

! Setup Named ACL for Server 1's LAN
R2(config)# ip access-list standard 2_192.168.1.0/24
R2(config-std-nacl)# permit 172.16.1.1
R2(config-std-nacl)# permit 172.16.2.1
! Good practice: explicitly write the implicit deny for readability
R2(config-std-nacl)# deny any
R2(config-std-nacl)# exit

! Apply outbound on the interface facing Server 1
R2(config)# interface g0/0
R2(config-if)# ip access-group 2_192.168.1.0/24 out
R2(config-if)# exit

! Setup Named ACL for Server 2's LAN
R2(config)# ip access-list standard 2_192.168.2.0/24
! Deny PC3/PC4's entire subnet using a wildcard mask
R2(config-std-nacl)# deny 172.16.2.0 0.0.0.255
R2(config-std-nacl)# permit any
R2(config-std-nacl)# exit

! Apply outbound on the interface facing Server 2
R2(config)# interface g0/1
R2(config-if)# ip access-group 2_192.168.2.0/24 out

Goal: Stop the 172.16.1.0/24 and 172.16.2.0/24 subnets from communicating. Uses Standard Numbered ACLs.

R1> enable
R1# configure terminal

! Numbered ACLs (1-99) use a single-line command structure
! Deny Subnet 1, permit everything else
R1(config)# access-list 1 deny 172.16.1.0 0.0.0.255
R1(config)# access-list 1 permit any

! Deny Subnet 2, permit everything else
R1(config)# access-list 2 deny 172.16.2.0 0.0.0.255
R1(config)# access-list 2 permit any

! Apply ACL 1 outbound facing Subnet 2 (G0/1)
R1(config)# interface g0/1
R1(config-if)# ip access-group 1 out
R1(config-if)# exit

! Apply ACL 2 outbound facing Subnet 1 (G0/0)
R1(config)# interface g0/0
R1(config-if)# ip access-group 2 out

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

Before restricting traffic, you must ensure the routers have complete routing tables.

R1(config)# router ospf 1
R1(config-router)# network 172.16.0.0 0.0.255.255 area 0
R1(config-router)# network 203.0.113.0 0.0.0.3 area 0

We use a Standard Named ACL on R2, applied outbound on G0/0 facing Server 1.

R2(config)# ip access-list standard 2_192.168.1.0/24
R2(config-std-nacl)# permit 172.16.1.1
R2(config-std-nacl)# permit 172.16.2.1
R2(config-std-nacl)# deny any
R2(config-std-nacl)# interface g0/0
R2(config-if)# ip access-group 2_192.168.1.0/24 out

We apply another Standard Named ACL outbound on R2's G0/1 interface. This time, we deny the entire subnet using a wildcard mask, but we MUST permit all other traffic, or the implicit deny will break everything else.

R2(config-std-nacl)# deny 172.16.2.0 0.0.0.255
R2(config-std-nacl)# permit any

After running pings from the PCs, execute the following command to see exactly how many packets matched each permit/deny statement:

R2# show access-lists
Standard IP access list 2_192.168.1.0/24
    10 permit 172.16.1.1 (4 matches)
    20 permit 172.16.2.1
    30 deny any (8 matches)
