SNMP Operations

Simple Network Management Protocol (SNMP)

Learn how to monitor and remotely configure network devices. This module covers SNMP Agents, the Network Management Station (NMS), configuring Read-Only and Read-Write community strings, and querying variables in the Management Information Base (MIB).

🧠 SNMP Components & Operations

The SNMP Framework

  • NMS (Network Management Station) The central server (or PC) running management software. It polls devices for data and receives alerts.
  • SNMP Agent The software component running on the managed device (like a router or switch) that collects and stores data.
  • MIB & OID The Management Information Base (MIB) is a hierarchical database of variables on the device. An Object Identifier (OID) is a specific numerical path (e.g., 1.3.6.1.2.1.1.5) that points to a specific variable, like the router's hostname.
  • Polling vs. Traps Get/Set: The NMS queries the agent (Polling) to read or write data.
    Traps/Informs: The agent autonomously sends an alert to the NMS when a specific event occurs (e.g., a link goes down).

SNMP Version Comparison

Security features across the three main versions of SNMP.

SNMPv1 and v2c use plain-text "Community Strings" as passwords. SNMPv3 is highly recommended in production because it supports hashing (Auth) and encryption (Priv).

💻 Device Configuration Lab

Review the commands to configure SNMP community strings, contact info, and traps.

Goal: Configure the router to respond to NMS polling by setting up Read-Only (RO) and Read-Write (RW) passwords (community strings).

R1> enable
R1# configure terminal

! Create a Read-Only string (NMS can view data but not change it)
R1(config)# snmp-server community cisco1 ro

! Create a Read-Write string (NMS can view and alter data, e.g., Hostname)
R1(config)# snmp-server community cisco2 rw

! (After using the MIB Browser to SET the hostname)
R1(config)# exit
R11# ! Notice the prompt changed automatically because the NMS modified sysName

Goal: Add administrative details (Contact, Location) to the MIB, and configure the router to send autonomous Traps to the NMS.

R1> enable
R1# configure terminal

! Administrative MIB objects (Useful for inventory management)
R1(config)# snmp-server contact snmp@boson.com
R1(config)# snmp-server location R1_SNMP

! Tell the router where to send autonomous alerts (Traps)
! Syntax: snmp-server host [IP] version [ver] [community_string]
R1(config)# snmp-server host 10.10.0.2 version 2c snmp_logs

! Verification
R1(config)# do show snmp
R1(config)# do show snmp host
Note: SNMP configuration is generally beyond the scope of the current CCNA 200-301 exam objectives, which is why Jeremy utilized a CCNP-level lab environment in the video to demonstrate these features.

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

Enter global configuration mode to set the community strings. These act as plain-text passwords for SNMPv1/v2c.

R1(config)# snmp-server community cisco1 ro
R1(config)# snmp-server community cisco2 rw

In Packet Tracer, open PC1's Desktop and launch the MIB Browser.

  • Enter R1's IP (192.168.1.254).
  • Click Advanced and set Read Community to cisco1 and Write Community to cisco2.
  • Navigate the tree to .iso.org.dod.internet.mgmt.mib-2.system.
  • Select sysUpTime and click Go (Get operation). It will return a TimeTicks value representing how long R1 has been booted.
  • Select sysName and click Go. It will return R1.

In the MIB Browser:

  • Select sysName in the tree.
  • Change the Operation dropdown from Get to Set.
  • Ensure the Data Type is set to OctetString.
  • Type R11 in the Value field and click Go.

Verification:

If you switch back to R1's CLI and press enter, you will see the prompt has magically changed from R1# to R11#. The NMS successfully altered the router's configuration over the network.

Use the specific show commands to verify the administrative variables you configured via snmp-server contact and snmp-server location.

R1# show snmp contact
snmp@boson.com

R1# show snmp location
R1_SNMP

R1# show snmp host
Notification host: 10.10.0.2 udp-port: 162 type: trap
user: snmp_logs security model: v2c

Notice that Traps are sent via UDP port 162, while standard Polling (Get/Set) uses UDP port 161.

📋 Command Reference

🔍
Command Mode Purpose
snmp-server community [string] [ro|rw] Global Config Sets up a plain-text password for an NMS to poll or modify the device (SNMP v1/v2c).
snmp-server contact [text] Global Config Defines the sysContact MIB variable (e.g., admin email).
snmp-server location [text] Global Config Defines the sysLocation MIB variable (e.g., Server Room Rack 2).
snmp-server host [ip] version [ver] [string] Global Config Tells the router where to proactively send autonomous SNMP Traps.
show snmp Privileged EXEC Displays general SNMP statistics, logging data, and chassis configuration info.
show snmp host Privileged EXEC Verifies which Network Management Station (NMS) IP addresses are configured to receive traps.
