Learn the inner mechanics of IPv6 Interface IDs. In this lab, you will configure addresses dynamically using the EUI-64 MAC conversion method, utilize Link-Local addresses via ipv6 enable, and master the specific syntax required for IPv6 static routing over link-local networks.

🧠 EUI-64 & Link-Local Mechanics

The EUI-64 Conversion Process

EUI-64 automatically generates a 64-bit IPv6 Interface ID using the device's 48-bit MAC address. Here is how the router calculates it:

  1. Split the MAC: Take the 48-bit MAC (e.g., 0000.1111.2222) and split it in half.
  2. Insert FFFE: Inject FFFE into the exact middle → 0000.11FF.FE11.2222.
  3. Invert the 7th Bit: Flip the 7th binary bit of the first byte. (e.g., 00 becomes 02).
  4. Result: 0200:11FF:FE11:2222

The Static Route "Gotcha"

When creating an IPv4 static route, providing the Next-Hop IP is usually enough. But in IPv6, if you use a Link-Local address (FE80::) as the Next-Hop, the router's routing table throws an error: % Interface has to be specified.

Why? Because Link-Local addresses are unique ONLY to a specific physical link. The router literally doesn't know which port to send the packet out of unless you explicitly tell it the exit interface.

Structure of a /64 Global Unicast Address

Visualizing how the 128 bits are allocated when using EUI-64.

The Global Routing Prefix and Subnet ID are provided by the administrator (or ISP). The final 64 bits are automatically filled by the EUI-64 MAC conversion logic.

💻 Device Configuration Lab

Review the commands to configure EUI-64 addresses, Link-Local routing, and specific static routes.

Goal: Enable global routing, configure the edge port using EUI-64, enable the transit port without a global address, and construct the static route.

R1> enable
R1# configure terminal

! Crucial: Enable global IPv6 packet forwarding
R1(config)# ipv6 unicast-routing

! Edge Port: Configure Network Prefix and instruct router to generate the Interface ID from its MAC
R1(config)# interface g0/1
R1(config-if)# ipv6 address 2001:db8::/64 eui-64
R1(config-if)# exit

! Transit Port: We don't want to waste a Global IP here. Just enable IPv6 to auto-generate the Link-Local.
R1(config)# interface g0/0
R1(config-if)# ipv6 enable
R1(config-if)# exit

! Static Route to R2's Subnet.
! MUST specify the exit interface (g0/0) before the Link-Local Next-Hop IP!
R1(config)# ipv6 route 2001:db8:0:1::/64 g0/0 fe80::202:16ff:fe17:1645

Goal: Enable global routing, configure the edge port using EUI-64, enable the transit port, and configure the return static route.

R2> enable
R2# configure terminal

R2(config)# ipv6 unicast-routing

! Edge Port
R2(config)# interface g0/1
R2(config-if)# ipv6 address 2001:db8:0:1::/64 eui-64
R2(config-if)# exit

! Transit Port: Enable Link-Local only
R2(config)# interface g0/0
R2(config-if)# ipv6 enable
R2(config-if)# exit

! Static Route back to R1's Subnet
R2(config)# ipv6 route 2001:db8::/64 g0/0 fe80::201:97ff:fe7d:6601

We use the eui-64 keyword at the end of the address assignment. Remember to enable ipv6 unicast-routing first!

R1(config)# ipv6 unicast-routing
R1(config)# interface g0/1
R1(config-if)# ipv6 address 2001:db8::/64 eui-64

You must first check the dynamically generated EUI-64 address on the routers using show ipv6 interface brief, copy it, and paste it into the Gateway field of the PCs in the Packet Tracer GUI.

  • PC2 Gateway: (Paste R2's dynamic G0/1 address here)

Use the ipv6 enable command. This automatically generates a Link-Local address using the FE80::/10 prefix and the EUI-64 conversion of the MAC address, keeping the interface completely hidden from the global internet.

R1(config)# interface g0/0
R1(config-if)# ipv6 enable

Find the generated link-local addresses using show ipv6 interface brief on both routers. Then configure the routes. Crucially: You must specify the exit interface before the link-local IP.

R1(config)# ipv6 route 2001:db8:0:1::/64 g0/0 fe80::202:16ff:fe17:1645

After configuring this on both routers, the ping from PC1 to PC2 will succeed!
