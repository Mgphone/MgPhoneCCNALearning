Step away from the router CLI! Today we explore Type 2 Hypervisors by installing Oracle VirtualBox on a Host OS (Windows) and provisioning a virtual machine running a Guest OS (Ubuntu Linux). This provides the foundation for running advanced network emulators like GNS3 or CML later on.

Type 2 Hypervisors & Resource Allocation

A hypervisor abstracts physical hardware (CPU, RAM, Storage) so it can be shared among multiple isolated Virtual Machines (VMs).

  • Type 2 (Hosted) Hypervisor Software like VirtualBox or VMware Workstation runs as an application on top of a pre-existing Host OS (like Windows or macOS). It relies on the Host OS to manage the hardware.
  • The Guest OS The operating system installed inside the virtual machine (e.g., Ubuntu Linux). It believes it is running on a dedicated physical computer.
  • Resource Provisioning When creating a VM, you carve out chunks of your physical hardware (e.g., 1GB of RAM, 10GB of Disk Space) to be exclusively dedicated to the Guest OS while it is running.

📝 Lab Walkthrough Checklist

Check off the steps to track the chronological flow of creating a VM.

Download and install VirtualBox (for Windows hosts) from virtualbox.org. Also, download the Ubuntu LTS ISO file from ubuntu.com (An ISO is an image of the installation CD/DVD).

Open VirtualBox, click "New". Name it "Ubuntu 20.04". Allocate 1GB (1024MB) of RAM. Create a 10GB Dynamically Allocated Virtual Hard Disk (VDI).

Before booting, go to Settings -> Storage. Click the "Empty" optical drive, click the disk icon, and choose the Ubuntu ISO file you downloaded. This acts as the bootable installation media.

Click "Start" to turn the VM on. The VM boots from the ISO. Follow the Ubuntu GUI installation wizard (Choose Minimal Installation, Erase Disk to install Ubuntu, and create a user account).

Terminology Matrix

Term Category Definition
Type 1 Hypervisor Architecture A "bare-metal" hypervisor that runs directly on physical hardware without a Host OS (e.g., VMware ESXi).
Type 2 Hypervisor Architecture A "hosted" hypervisor that runs as an application on top of an existing Host OS (e.g., Oracle VirtualBox).
Host OS Component The primary operating system installed directly on the physical computer (e.g., Windows 10/11 on your laptop).
Guest OS Component The operating system running inside the virtual machine environment (e.g., the Ubuntu Linux installed in the lab).
ISO File File Type A disk image file that contains the entire contents of a CD/DVD, commonly used to distribute OS installers.
VDI File Type VirtualBox Disk Image. The file format VirtualBox uses to simulate a physical hard drive for the VM.
