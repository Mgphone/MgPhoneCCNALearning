OSPF Network Types & Troubleshooting

Dive deeper into OSPF topology mechanics. Learn the critical differences between Broadcast and Point-to-Point networks, how mismatched timers prevent adjacencies, and explore the core LSA types that build the OSPF database.

🧠 Advanced OSPF Mechanics

OSPF Network Types

  • Broadcast (Default for Ethernet) Designed for multi-access segments (like switches). It requires the election of a Designated Router (DR) and Backup Designated Router (BDR) to reduce LSA flooding. Default Timers: 10s Hello, 40s Dead.
  • Point-to-Point (Default for Serial/HDLC) Designed for direct router-to-router links. It does not elect a DR/BDR because there are only two devices on the link. Default Timers: 10s Hello, 40s Dead.

⚠️ Adjacency Killers

Two OSPF routers will refuse to become neighbors if their Hello and Dead timers do not match exactly. Furthermore, if the Network Types mismatch (e.g., Broadcast vs. P2P), the routers might form an adjacency but will fail to successfully install routes in the routing table due to database inconsistencies.

Link State Database (LSDB) Overview

The core LSA types found in a standard CCNA Single-Area OSPF configuration.

Use show ip ospf database to view these. Type 1 represents routers, Type 2 represents the DR's multi-access network, and Type 5 represents external routes (like an injected default route).

💻 Troubleshooting CLI Lab

Review the commands used to repair the broken network adjacencies.

Goal: Establish IP connectivity over the serial link, set the DCE clock rate, and enable OSPF.

! On R1 (Verified as the DCE end via `show controllers s0/0/0`)
R1(config)# interface s0/0/0
R1(config-if)# ip address 192.168.12.1 255.255.255.252
R1(config-if)# clock rate 128000
R1(config-if)# no shutdown
R1(config-if)# ip ospf 1 area 0

! On R2 (DTE end, no clock rate required)
R2(config)# interface s0/0/0
R2(config-if)# ip address 192.168.12.2 255.255.255.252
R2(config-if)# no shutdown
R2(config-if)# ip ospf 1 area 0
Note: Serial connections use HDLC encapsulation by default. OSPF automatically detects this and assigns a network type of POINT_TO_POINT.

Goal: Fix the network type mismatch. R3's Gigabit interface was manually (and incorrectly) set to point-to-point, conflicting with R4's default broadcast type.

! Verify the problem on R3
R3# show ip ospf interface g0/1
GigabitEthernet0/1 is up, line protocol is up
  Internet Address 10.0.34.3/24, Area 0
  Process ID 3, Router ID 3.3.3.3, Network Type POINT_TO_POINT

! Remove the bad configuration to revert to default Broadcast
R3(config)# interface g0/1
R3(config-if)# no ip ospf network point-to-point

! Verify R4 now receives the routes successfully
R4# show ip route
...
O 10.0.2.0/24 [110/2] via 10.0.34.3, 00:00:12, GigabitEthernet0/1

Goal: Fix mismatched timers preventing adjacency, and add the missing default route required for the ASBR injection.

! 1. Fix the mismatched timers on G0/0
R5# show ip ospf interface g0/0
  Timer intervals configured, Hello 5, Dead 20

R5(config)# interface g0/0
R5(config-if)# no ip ospf hello-interval
R5(config-if)# no ip ospf dead-interval
%OSPF-5-ADJCHG: Process 1, Nbr 2.2.2.2 on GigabitEthernet0/0 from LOADING to FULL

! 2. R5 has 'default-information originate' but lacks a static route. Add it.
R5(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.2

! Now R5 will generate a Type 5 LSA and share it with the OSPF domain!

📝 Lab Tasks & Solutions

Check off the steps as you complete the troubleshooting tasks.

Verify which side is DCE using show controllers s0/0/0. Set the clock rate 128000 on the DCE side (R1). Apply the IP addresses and use ip ospf 1 area 0 directly on the interface to enable routing.

OSPF Note: Serial links automatically use the Point-to-Point network type, meaning no DR or BDR is elected.

Diagnosis: Network Type Mismatch.

By running show ip ospf interface on both routers, we see R4 is using the default BROADCAST type for Ethernet, while R3 was manually forced to POINT_TO_POINT.

Because their Hello timers happen to match (10s), the adjacency forms. However, because their network types differ, they expect different LSA structures (Broadcast expects a Type 2 Network LSA from a DR; P2P does not). This inconsistency prevents routes from being installed in the routing table. Removing the P2P command on R3 fixes it.

Diagnosis: Timer Mismatch.

Checking R5 with show ip ospf interface g0/0 reveals its timers were manually changed to Hello 5, Dead 20. OSPF has a strict rule: Hello and Dead intervals MUST match for an adjacency to form. Using no ip ospf hello-interval and no ip ospf dead-interval on R5 reverts them to the default 10/40, instantly bringing the neighbors up.

Diagnosis: Missing actual default route.

R5 has the default-information originate command configured under its OSPF process. However, this command only injects a default route if the router actually has a default route in its own routing table. Because R5 lacked a static default route to the ISP, it had nothing to advertise.

Adding ip route 0.0.0.0 0.0.0.0 [isp-ip] to R5 triggers OSPF to generate a Type 5 AS-External LSA and flood it to the rest of the network.
