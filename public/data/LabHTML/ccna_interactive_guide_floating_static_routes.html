Floating Routes

Floating Static Routes

Ensure high availability in your network. Learn how to configure backup static routes that remain hidden in the background, only "floating" into the routing table when the primary dynamic routing protocol (like OSPF) fails.

🧠 Administrative Distance (AD)

The Trustworthiness Value

If a router learns about the same destination network from two different sources (e.g., OSPF and a Static Route), it uses Administrative Distance (AD) to decide which path is more trustworthy. Lower AD wins.

  • Connected Interface AD: 0
  • Standard Static Route AD: 1
  • OSPF (Dynamic) AD: 110

The "Floating" Mechanism

By default, a static route (AD 1) will overwrite an OSPF route (AD 110). If we want the static route to act as a backup, we must manually configure it with an AD higher than OSPF (e.g., 111).

The router will hide the backup route. It only "floats" to the surface if the OSPF route disappears due to a link failure.

Administrative Distance Scale

Lower numbers are more preferred by the routing engine.

We configure the floating static route to AD 111 so that OSPF (110) wins the election during normal operations.

💻 Device Configuration Lab

Review the commands to configure floating static routes and verify their status in the routing table.

Goal: Add a static route to R2's network (10.0.2.0/24) pointing through ISPA, with an AD of 111.

R1> enable
R1# configure terminal

! View current routing table
R1(config)# do show ip route
...
O 10.0.2.0/24 [110/2] via 192.168.12.2, GigabitEthernet0/2

! Add the Floating Static Route (Notice the "111" at the end)
R1(config)# ip route 10.0.2.0 255.255.255.0 203.0.113.1 111

! Verify it does NOT overwrite OSPF during normal operation
R1(config)# do show ip route
O 10.0.2.0/24 [110/2] via 192.168.12.2, GigabitEthernet0/2

Goal: Add a return static route to R1's network (10.0.1.0/24) pointing through ISPA, with an AD of 111.

R2> enable
R2# configure terminal

! Add the Floating Static Route pointing to ISPA
R2(config)# ip route 10.0.1.0 255.255.255.0 203.0.113.5 111

! Simulate a primary link failure to test the backup
R2(config)# interface g0/2
R2(config-if)# shutdown
%LINK-5-CHANGED: Interface GigabitEthernet0/2, changed state to administratively down
%OSPF-5-ADJCHG: Process 1, Nbr 1.1.1.1 on GigabitEthernet0/2 from FULL to DOWN

! Verify the floating route has now been injected into the table
R2(config-if)# do show ip route
...
S 10.0.1.0/24 [111/0] via 203.0.113.5

Goal: Use PC commands to verify the traffic path during failover.

! On Windows PC1 (Command Prompt)
! Use traceroute to see the hop-by-hop path

📝 Lab Tasks & Solutions

Using show ip route, we look for codes next to the learned routes.

O 10.0.2.0/24 [110/2] via 192.168.12.2, GigabitEthernet0/2

Answer: The `O` code indicates that OSPF (Open Shortest Path First) is the dynamic routing protocol in use. The `110` in brackets is OSPF's default Administrative Distance.

We must configure a static route but override the default AD of 1 to be higher than OSPF's 110.

! On R1 (Pointing to Server 1's network via ISPA)
R1(config)# ip route 10.0.2.0 255.255.255.0 203.0.113.1 111

! On R2 (Return path to PC1's network via ISPA)
R2(config)# ip route 10.0.1.0 255.255.255.0 203.0.113.5 111

By bringing down the primary interface, the OSPF adjacency drops, and the OSPF route is removed. The router then elevates the hidden floating static route into the active routing table.

R2(config)# interface g0/2
R2(config-if)# shutdown
R2(config-if)# do show ip route
...
S 10.0.1.0/24 [111/0] via 203.0.113.5

Open the command prompt on PC1 and run the trace command to the server IP.

C:\> tracert 10.0.2.1

The output will show hops passing through the 203.0.113.x network, proving the traffic is successfully navigating the ISPA fallback infrastructure instead of dropping.

📋 Command Reference

🔍
Command Mode Purpose
ip route [net] [mask] [hop] [AD] Global Config Configures a static route. Appending a number at the end overrides the default Administrative Distance of 1.
show ip route Privileged EXEC Displays the active routing table. Inactive floating routes will NOT be visible here.
tracert [ip] Windows CMD Traces the path a packet takes hop-by-hop to reach the destination (Windows).
traceroute [ip] Privileged EXEC Cisco IOS version of the trace tool. Sends UDP probes to track hops.
