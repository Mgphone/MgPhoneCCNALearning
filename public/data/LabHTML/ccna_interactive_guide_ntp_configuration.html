Network Time Protocol

Network Time Protocol (NTP)

Accurate timekeeping is critical for logging, troubleshooting, and certificate validation. Learn how to synchronize network devices to a central time source using NTP (UDP Port 123), configure Stratum hierarchies, and secure time updates with MD5 authentication.

🧠 Core NTP Mechanics

How NTP Operates

  • The Stratum Hierarchy NTP uses a tiered system called "Stratum" to indicate the distance from a reference clock. Stratum 0 represents highly precise atomic clocks or GPS clocks. Stratum 1 servers are directly connected to Stratum 0 devices. A router syncing to a Stratum 1 server becomes Stratum 2, and so on. Stratum 15 is the maximum valid level; 16 means unsynchronized.
  • NTP Master vs. Client A router configured as an ntp server syncs its clock to that specific IP. A router configured with ntp master acts as a fallback authoritative time source for the network, using its own internal hardware clock if external servers become unreachable.
  • NTP Authentication Time manipulation is a serious security risk. Attackers could alter device times to invalidate security certificates or hide traces in log files. NTP authentication uses MD5 keys to verify that time updates are coming from trusted sources.

💻 Device Configuration Lab

Review the commands to configure manual clocks, NTP servers, and MD5 authentication.

Goal: Manually seed the clock, configure R1 to sync to the Internet Server (1.1.1.1), designate R1 as a fallback NTP Master, and set up the MD5 authentication key for downstream routers.

R1> enable

! Manually set the clock close to the real time (Speeds up NTP Sync)
R1# clock set 12:00:00 30 dec 2020
R1# configure terminal
R1(config)# clock timezone JST 9

! Sync to the Internet Server
R1(config)# ntp server 1.1.1.1

! Act as a backup Master (Stratum 8 default) if Internet fails
R1(config)# ntp master

! Setup NTP Authentication for downstream clients (R2, R3)
R1(config)# ntp authenticate
R1(config)# ntp authentication-key 1 md5 Jeremy'sITLab
R1(config)# ntp trusted-key 1

! Force the hardware calendar to update alongside the software clock
R1(config)# ntp update-calendar

Goal: Configure R2 and R3 to securely synchronize their time from R1 using the MD5 authentication key.

R2> enable
R2# clock set 12:00:00 30 dec 2020
R2# configure terminal
R2(config)# clock timezone JST 9

! Setup the exact same Authentication key as R1
R2(config)# ntp authenticate
R2(config)# ntp authentication-key 1 md5 Jeremy'sITLab
R2(config)# ntp trusted-key 1

! Sync to R1, and mandate the use of key 1
R2(config)# ntp server 192.168.12.1 key 1

! (Configuration for R3 is identical, using R1's IP: ntp server 192.168.13.1 key 1)

Manual clock setting speeds up the initial NTP synchronization process.

! Exec mode (Not config)
R1# clock set 12:00:00 30 dec 2020
! Global config
R1# configure terminal
R1(config)# clock timezone JST 9
R1# show ntp associations

address ref clock st when poll reach delay offset disp
*~1.1.1.1 .LOCL. 1 21 64 17 2.0 15.5 3.4

Answer: The asterisk (*) next to the IP address signifies that the router has successfully synchronized to this peer. (Note: In Packet Tracer, you may need to fast-forward time repeatedly for the asterisk to appear).

R1# show ntp status
Clock is synchronized, stratum 2, reference is 1.1.1.1
...

Answer: Stratum 2. Because Server 1 is acting as a primary reference clock (Stratum 1), R1 adds 1 hop to the stratum distance when it syncs.

After configuring the authentication keys on both routers, you must append the key parameter to the server command.

R2(config)# ntp authenticate
R2(config)# ntp authentication-key 1 md5 Jeremy'sITLab
R2(config)# ntp trusted-key 1
R2(config)# ntp server 192.168.12.1 key 1

📋 Command Reference

🔍
Command Mode Purpose
clock set [hh:mm:ss] [date] Privileged EXEC Manually sets the software clock. Often used to speed up initial NTP synchronization.
clock timezone [name] [offset] Global Config Configures the router's local time zone offset from UTC.
ntp server [ip] Global Config Configures the router to act as an NTP client and sync its time to the specified server IP.
ntp master [stratum] Global Config Configures the router to act as an authoritative time source using its internal clock. Default stratum is 8.
show ntp status Privileged EXEC Displays the current Stratum level and whether the clock is synchronized.
show ntp associations Privileged EXEC Lists known NTP peers. An asterisk (*) indicates an active, successful synchronization.
show clock detail Privileged EXEC Displays the current time and confirms the 'Time source' (e.g., NTP or user configuration).
