Layer 3 Switches & SVIs

Upgrade your network's core. Learn how to replace Router-on-a-Stick (ROAS) bottlenecks with wire-speed routing using a Multilayer Switch. You will configure Switch Virtual Interfaces (SVIs), routed ports, and default routes.

🧠 Evolving from ROAS

While Router-on-a-Stick works, all inter-VLAN traffic must travel up to the router and back down over a single physical link, creating a severe bottleneck. Layer 3 Switches (Multilayer Switches) solve this by routing packets internally via hardware ASICs at wire speed.

Key L3 Switch Concepts

  • SVIs (Switch Virtual Interfaces): Logical interfaces mapped to specific VLANs (e.g., interface vlan 10). These act as the default gateways for the PCs in those VLANs.
  • Routed Ports: By default, L3 switch ports act as Layer 2 access/trunk ports. Using the no switchport command converts a port into a standard Layer 3 router port, allowing direct IP assignment.

The "Gotcha" Command

Even if you assign IP addresses to SVIs, a Layer 3 switch will not route packets out of the box. You must explicitly tell the switch to behave like a router globally by issuing the ip routing command.

💻 Device Configuration Lab

Review the specific commands to strip ROAS, build SVIs, and configure routed ports.

Goal: Remove the old ROAS subinterfaces and convert the physical interface back to a standard Layer 3 point-to-point link.

R1> enable
R1# configure terminal

! Remove the old ROAS subinterfaces
R1(config)# no interface g0/0.10
R1(config)# no interface g0/0.20
R1(config)# no interface g0/0.30

! Configure physical interface for Point-to-Point routing
R1(config)# interface g0/0
R1(config-if)# ip address 10.0.0.194 255.255.255.192
R1(config-if)# no shutdown
R1(config-if)# exit
Note: R1 already has default/static routes configured pointing out to the internet and back to SW2's subnets as part of the pre-lab state.

Goal: Convert the uplink port to a routed port, enable global routing, configure default route to R1, and build SVIs for inter-VLAN routing.

SW2> enable
SW2# configure terminal

! 1. Convert uplink (to R1) to a Layer 3 Routed Port
SW2(config)# interface g1/0/2
! (Wipe any residual trunk configs if necessary: default interface g1/0/2)
SW2(config-if)# no switchport
SW2(config-if)# ip address 10.0.0.193 255.255.255.192
SW2(config-if)# exit

! 2. Enable Global IP Routing and set Default Route to R1
SW2(config)# ip routing
SW2(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.194

! 3. Configure SVIs for inter-VLAN routing
! VLAN 10 (Engineering)
SW2(config)# interface vlan 10
SW2(config-if)# ip address 10.0.0.62 255.255.255.192
SW2(config-if)# no shutdown
SW2(config-if)# exit

! VLAN 20 (HR)
SW2(config)# interface vlan 20
SW2(config-if)# ip address 10.0.0.126 255.255.255.192
SW2(config-if)# no shutdown
SW2(config-if)# exit

! VLAN 30 (Sales)
SW2(config)# interface vlan 30
SW2(config-if)# ip address 10.0.0.190 255.255.255.192
SW2(config-if)# no shutdown

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

You must strip the subinterfaces off R1 and convert SW2's port to Layer 3 using no switchport.

! On R1:
R1(config)# no interface g0/0.10
R1(config)# no interface g0/0.20
R1(config)# interface g0/0
R1(config-if)# ip address 10.0.0.194 255.255.255.192

! On SW2:
SW2(config)# interface g1/0/2
SW2(config-if)# no switchport
SW2(config-if)# ip address 10.0.0.193 255.255.255.192
SW2(config)# ip routing
SW2(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.194

We use the interface vlan [id] command to create the SVI. Before it goes up/up, the VLAN must exist in the switch database, and at least one active physical port (access or trunk) must be passing traffic for that VLAN.

SW2(config)# interface vlan 10
SW2(config-if)# ip address 10.0.0.62 255.255.255.192
SW2(config-if)# no shutdown
... repeat for interface vlan 20 (.126) and 30 (.190)

Open PC1 (VLAN 10 on SW1) command prompt and ping PC3 (VLAN 20 on SW2) at `10.0.0.65`.

PC1> ping 10.0.0.65
Reply from 10.0.0.65: bytes=32 time<1ms TTL=127

Notice the traffic never hits R1. SW2 routes it directly between SVIs and sends it down its local access port to PC3.

From PC1, ping the simulated internet address.

PC1> ping 1.1.1.1
Reply from 1.1.1.1: bytes=32 time=2ms TTL=126

Success! PC1 hits its gateway (SW2 SVI 10). SW2 uses its default route to send the packet out the Layer 3 routed port to R1. R1 forwards it to the Internet.
