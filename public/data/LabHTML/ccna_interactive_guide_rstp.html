Rapid STP (802.1w)

Rapid Spanning Tree Protocol (RSTP)

Classic 802.1D STP takes up to 50 seconds to converge, which is unacceptable for modern networks. Learn how Rapid STP (802.1w) redesigns port roles, states, and the BPDU sync mechanism to achieve sub-second convergence.

🧠 The Evolution to 802.1w

New Port States

RSTP simplifies the 5 classic STP states down to just 3 highly efficient states:

  • Discarding
    Combines Disabled, Blocking, and Listening. Port drops data but processes BPDUs.
  • Learning
    Same as classic. Silently populates the MAC table without forwarding user data.
  • Forwarding
    Fully operational. Sends and receives user data frames.

New Port Roles

RSTP splits the classic "Non-Designated" role into two distinct roles to drastically speed up recovery:

  • Alternate Port: A backup path to the Root Bridge. If the Root Port fails, the Alternate port immediately takes over.
  • Backup Port: A backup path for a Designated Port on a shared segment (like a hub). Rare in modern switched networks.

How long it takes a network to recover from a link failure.

By actively negotiating transitions via Proposal/Agreement BPDUs, RSTP bypasses timer-based delays entirely.

💻 Device Configuration Lab

Review the commands to globally enable Rapid PVST+ and define link types.

Goal: Change the operating mode of the switch from classic IEEE 802.1D (PVST+) to Rapid 802.1w (Rapid-PVST+).

SW1> enable
SW1# configure terminal

! View the current mode (defaults to pvst)
SW1(config)# do show spanning-tree summary
Switch is in pvst mode

! Globally change the Spanning Tree mode to RSTP
SW1(config)# spanning-tree mode rapid-pvst

! Verify the change
SW1(config)# do show spanning-tree
VLAN0001
  Spanning tree enabled protocol rstp
...
Note: You must apply this command to every switch in your network topology for Rapid Spanning Tree to function efficiently end-to-end.

Goal: RSTP relies on link types to determine if it can safely transition a port to forwarding. Full-duplex links automatically default to point-to-point, but you can hardcode them.

SW2> enable
SW2# configure terminal
SW2(config)# spanning-tree mode rapid-pvst

! Hardcoding a switch-to-switch link
SW2(config)# interface g0/1
SW2(config-if)# spanning-tree link-type point-to-point
SW2(config-if)# exit

! Verifying link type in the STP output
SW2# show spanning-tree
Interface     Role Sts Cost      Prio.Nbr Type
---------------- --- --- --------- -------- --------------------------------
Gi0/1           Root FWD 4         128.1    P2p
Fa0/3           Altn BLK 19        128.3    P2p

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

Enter global configuration mode on each switch and apply the mode command.

SW1(config)# spanning-tree mode rapid-pvst
SW2(config)# spanning-tree mode rapid-pvst
SW3(config)# spanning-tree mode rapid-pvst

Use the show spanning-tree command. Look at the second line of the output for the word "rstp".

SW1# show spanning-tree
VLAN0001
  Spanning tree enabled protocol rstp

In classic STP, a blocked port shows a role of "Altn" (Alternate) or "Non-Designated" and a status of "BLK" (Blocking). In RSTP, the status changes to the new unified state.

  • The Role will explicitly say Altn (Alternate).
  • The Status (Sts) will still display BLK in the CLI (which represents the Discarding state in RSTP logic). Cisco kept the 'BLK' display so network engineers wouldn't have to learn a completely new CLI syntax, but functionally, it is in the Discarding state.
