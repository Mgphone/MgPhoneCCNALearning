QoS Configuration

Quality of Service (QoS)

Not all network traffic is equal. Learn how to configure the Modular QoS CLI (MQC) to classify traffic (Class Maps), assign priority markings and bandwidth allocations (Policy Maps), and apply these rules to an interface (Service Policies).

🧠 MQC & Packet Tracer Math

The 3 Steps of MQC

  • 1. Class-Map (Identification) Used to identify what traffic we want to affect. Example: match protocol http. By default, class-maps use match-all logic (all statements must match). You can change this to match-any.
  • 2. Policy-Map (Action) Used to define what to do with the matched traffic. We call the class-map and assign an action, such as set ip dscp af31 or priority percent 10.
  • 3. Service-Policy (Application) Ties the policy-map to a specific interface in a specific direction (e.g., service-policy output my_policy). Without this, the QoS rules sit dormant.

PT Quirks: DSCP to Hex Conversions

When inspecting outbound PDUs in Packet Tracer, the 6-bit DSCP value is displayed in Hexadecimal. To verify your markings, you must understand the binary conversion:

  • CS2 → Bin: 010000 → Dec: 16 → Hex: 10
  • AF31 → Bin: 011010 → Dec: 26 → Hex: 1A
  • AF32 → Bin: 011100 → Dec: 28 → Hex: 1C

💻 Device Configuration Lab

Review the commands to construct the Class-Maps, Policy-Map, and Service-Policy.

Goal: Classify HTTPS, HTTP, and ICMP traffic. Create a policy to prioritize and mark them appropriately, then apply it outbound towards R2.

R1> enable
R1# configure terminal

! STEP 1: CLASS MAPS (Identify Traffic)
R1(config)# class-map https_map
R1(config-cmap)# match protocol https
R1(config-cmap)# exit
R1(config)# class-map http_map
R1(config-cmap)# match protocol http
R1(config-cmap)# exit
R1(config)# class-map icmp_map
R1(config-cmap)# match protocol icmp
R1(config-cmap)# exit

! STEP 2: POLICY MAP (Define Actions)
R1(config)# policy-map g0/0/0_out
! Assign actions to the HTTPS class
R1(config-pmap)# class https_map
R1(config-pmap-c)# set ip dscp af31
R1(config-pmap-c)# priority percent 10
R1(config-pmap-c)# exit
! Assign actions to the HTTP class
R1(config-pmap)# class http_map
R1(config-pmap-c)# set ip dscp af32
R1(config-pmap-c)# bandwidth percent 10
R1(config-pmap-c)# exit
! Assign actions to the ICMP class
R1(config-pmap)# class icmp_map
R1(config-pmap-c)# set ip dscp cs2
R1(config-pmap-c)# bandwidth percent 5
R1(config-pmap-c)# exit
R1(config-pmap)# exit

! STEP 3: SERVICE POLICY (Apply to Interface)
R1(config)# interface g0/0/0
R1(config-if)# service-policy output g0/0/0_out

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

We use the match protocol command to leverage NBAR (Network Based Application Recognition) to dynamically identify the traffic types.

R1(config)# class-map https_map
R1(config-cmap)# match protocol https
! (Repeat for http_map and icmp_map)

Within the policy-map, you call the previously created class-maps. The priority command establishes a Strict Priority Queue (usually for Voice, though we used it for HTTPS here for practice). The bandwidth command establishes a minimum bandwidth guarantee during congestion.

R1(config)# policy-map g0/0/0_out
R1(config-pmap)# class https_map
R1(config-pmap-c)# set ip dscp af31
R1(config-pmap-c)# priority percent 10

The rules are dormant until applied using a service-policy.

R1(config)# interface g0/0/0
R1(config-if)# service-policy output g0/0/0_out

Answer: Nothing (by default).

We configured R1 to prioritize and mark the traffic before sending it to R2. However, when R2 receives these packets, it will see the DSCP markings, but unless R2 has its own Policy-Map configured to trust or act upon those markings, R2 will treat them exactly like default, unprioritized traffic. QoS must be configured end-to-end across the network to be fully effective.

📋 Command Reference

🔍
Command Mode Purpose
class-map [name] Global Config Creates a class-map to identify traffic. Defaults to `match-all` logic.
class-map match-any [name] Global Config Creates a class-map where traffic only needs to hit ONE match statement to be classified.
match protocol [protocol] Class-Map Config Uses NBAR to identify specific traffic types (e.g., http, https, icmp).
policy-map [name] Global Config Creates a policy-map to define QoS actions for specified classes.
set ip dscp [value] Policy-Map Class Modifies the IP header's DSCP field to tag the packet with a specific priority (e.g., af31, ef).
priority percent [X] Policy-Map Class Assigns traffic to a Low Latency Queue (LLQ) with a strict bandwidth guarantee.
bandwidth percent [X] Policy-Map Class Assigns traffic to a Class-Based Weighted Fair Queue (CBWFQ) with a minimum guarantee.
service-policy [in/out] [name] Interface Config Attaches the completed policy-map to an interface to activate the QoS rules.
