Configuring STP (PVST+)

Take control of your Layer 2 topology! Leaving Spanning Tree Protocol (STP) to its default settings can result in suboptimal paths. In this lab, we configure PVST+ load balancing, manipulate port costs, and secure edge ports using PortFast and BPDU Guard.

PVST+ & Port Costs

Cisco's default STP mode is PVST+ (Per-VLAN Spanning Tree Plus). It runs a separate spanning-tree instance for each VLAN.

  • Load Balancing By assigning different switches to be the Root Bridge for different VLANs, traffic is distributed across all available uplinks instead of bottlenecking on a single path.
  • Root Port Selection Switches select their Root Port based on the lowest Path Cost to the Root Bridge. If costs tie, it checks the lowest Sender Bridge ID, then Sender Port ID.
  • PortFast & BPDU Guard PortFast skips the Listening/Learning states on access ports, turning them on immediately. BPDU Guard protects these ports by disabling them (err-disabled) if a rogue switch sends a BPDU.

Default STP Port Costs (IEEE 802.1D)

Lower bandwidth = Higher cost. The shortest path wins.

Device Configurations

Goal: Optimize paths by assigning different Root Bridges to different VLANs.

! On Switch 1: Make it the Primary Root for VLAN 1, Secondary for VLAN 2
SW1(config)# spanning-tree vlan 1 root primary
SW1(config)# spanning-tree vlan 2 root secondary

! On Switch 2: Do the exact opposite to balance the traffic load
SW2(config)# spanning-tree vlan 2 root primary
SW2(config)# spanning-tree vlan 1 root secondary

Goal: Force a switch to select a specific Root Port by manually manipulating Path Costs or Port Priorities.

! Change the STP cost of an interface (Higher cost = Less desirable path)
SW4(config)# interface f0/2
SW4(config-if)# spanning-tree vlan 1 cost 100

! Change Port Priority (Used as a tiebreaker if Costs are identical)
! Lower priority number = More preferred. Increments of 16.
SW4(config)# interface f0/1
SW4(config-if)# spanning-tree vlan 1 port-priority 240

Goal: Secure access ports connected to end-users (PCs) to skip STP timers and prevent rogue switches.

SW3(config)# interface f0/3

! Skip Listening/Learning states (Link goes green immediately)
SW3(config-if)# spanning-tree portfast

! Shutdown the port if a BPDU is received (Prevents network loops)
SW3(config-if)# spanning-tree bpduguard enable

! TO RECOVER an err-disabled port after a BPDU Guard violation:
SW3(config-if)# shutdown
SW3(config-if)# no shutdown

Use show spanning-tree. The Root Bridge will explicitly state "This bridge is the root". Look at the port roles to identify the Root Port (RP), Designated Ports (DP), and Alternate/Blocking Ports (BLK).

Use spanning-tree vlan [id] root primary and secondary across your distribution/core switches to ensure different VLANs utilize different physical uplinks.

Go to interface config and change the cost: spanning-tree vlan 1 cost 100. Verify with show spanning-tree vlan 1 to see the port transition from Root to Blocking.

Apply spanning-tree portfast and spanning-tree bpduguard enable to interfaces facing PCs. Test it by connecting a rogue switch to the port and observing the port instantly shut down.
