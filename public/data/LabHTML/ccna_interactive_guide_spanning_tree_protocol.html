Spanning Tree

Analyzing Spanning Tree Protocol (STP)

Master Layer 2 loop prevention. Learn how switches mathematically elect a Root Bridge, calculate the shortest path (Root Ports), and strategically block redundant links (Non-Designated Ports) to create a loop-free topology.

🧠 The STP Election Process

1. Elect the Root Bridge

The switch with the lowest Bridge ID (Priority + MAC Address) wins. Priority defaults to 32768. If priorities tie, the lowest MAC address breaks the tie.

Rule: All ports on a Root Bridge become Designated Ports (Forwarding).

2. Find Root Ports (RP)

Every non-root switch selects ONE port with the lowest Path Cost to the Root Bridge. Common costs: FastEthernet = 19, GigabitEthernet = 4.

Tiebreakers: Lowest Neighbor Bridge ID → Lowest Neighbor Port ID.

3. Determine Designated & Blocking Ports

For every remaining link between switches, only one end can forward traffic (Designated Port). The switch with the lowest Root Path Cost claims the Designated Port. The losing side becomes Non-Designated (Blocking).

Spanning Tree Path Cost to Root

Total cumulative cost for each switch to reach the Root Bridge (SW3).

SW2 prefers traveling through SW4 via Gigabit (Cost 4+4=8) rather than traversing its direct FastEthernet link to SW3 (Cost 19).

Answer: SW3 is the Root Bridge.
By checking the configuration or using show spanning-tree, you discover SW3 has a lower Bridge Priority configured (24576 + SysID 1 = 24577) than the default 32769 used by the other switches.

Because SW3 is the Root Bridge, it does not have a Root Port. The rule states that all active ports on a Root Bridge must be Designated Ports.

  • F0/1: Designated (Forwarding)
  • F0/2: Designated (Forwarding)
  • F0/3: Designated (Forwarding)
  • G0/1: Designated (Forwarding)
  • G0/2: Root Port (Forwarding)
    Logic: Connects directly to SW3 via Gigabit. Path Cost = 4. Lowest cost to root.
  • G0/1: Designated (Forwarding)
    Logic: Connects to SW2. SW4's cost to root is 4. SW2's cost is 8. SW4 has the lower cost, so it claims the Designated Port for this segment.
  • G0/1: Root Port (Forwarding)
    Logic: Even though F0/3 goes directly to the Root Bridge, FastEthernet costs 19. Taking the Gigabit path through SW4 costs 4+4=8. 8 is lower than 19.
  • F0/1 & F0/2: Designated (Forwarding)
    Logic: Connect to SW1. SW2's cost to root is 8. SW1's cost is 19. SW2 wins and claims Designated for these segments.
  • F0/3: Non-Designated / Alternate (Blocking)
    Logic: Connects directly to SW3. Since SW3 is root, SW3 claims the Designated port. SW2 puts this redundant link in Blocking state to stop a loop.
  • F0/4: Root Port (Forwarding)
    Logic: Both F0/3 and F0/4 connect directly to SW3 (Cost 19). Tiebreaker #1: Lowest neighbor bridge ID (Tie, both SW3). Tiebreaker #2: Lowest neighbor port ID. SW3's F0/1 is lower than its F0/2. Since SW1's F0/4 connects to SW3's F0/1, it becomes the Root Port.
  • F0/3: Non-Designated / Alternate (Blocking)
    Logic: Lost the tiebreaker to F0/4. Redundant link to root is blocked.
  • F0/1 & F0/2: Non-Designated / Alternate (Blocking)
    Logic: Connect to SW2. SW1 cost = 19. SW2 cost = 8. SW2 wins Designated role. SW1 must block.

📋 Command Reference

🔍
Command Mode Purpose
show spanning-tree Privileged EXEC Displays STP status, root bridge info, and the role/state/cost of every local interface.
show spanning-tree vlan [id] Privileged EXEC Filters the standard spanning-tree output to only show details for a specific VLAN (PVST+).
show spanning-tree detail Privileged EXEC Provides verbose information, including the total calculated Path Cost to the Root.
show spanning-tree summary Privileged EXEC Shows a high-level count of how many ports are in Blocking, Listening, Learning, and Forwarding states.
