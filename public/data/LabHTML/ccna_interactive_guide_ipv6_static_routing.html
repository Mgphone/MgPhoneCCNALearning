IPv6 Static Routing

IPv6 Static & Floating Routes

Master IPv6 manual routing. You will learn how to rely on SLAAC for host autoconfiguration, configure fully specified static routes using Link-Local Next-Hops, and manipulate Administrative Distance (AD) to create backup floating routes.

🧠 IPv6 Advanced Mechanics

Core Concepts

  • SLAAC & NDP Stateless Address Autoconfiguration (SLAAC) allows a PC to configure its own IPv6 address and gateway without a DHCP server. It relies on the router sending NDP Router Advertisements (RA). The router will only send RAs if ipv6 unicast-routing is enabled.
  • Fully Specified Static Routes If you use a Link-Local address (e.g., FE80::1) as your next-hop, you must also specify the exit interface. Example: ipv6 route 2001:db8::/64 g0/0 fe80::1. This is because link-local addresses are not globally unique.
  • Floating Static Routes By appending a number at the end of a static route command, you change its Administrative Distance (AD). A route with an AD of 5 will remain hidden as a backup until the primary route (default AD 1) fails.

Administrative Distance (AD) Scale

Lower AD values are more trustworthy.

We configure our backup IPv6 static route with an AD of 5. It will float in the background as long as the primary static route (AD 1) is active.

💻 Device Configuration Lab

Review the commands to configure primary static routes and secondary floating routes via link-local addresses.

Goal: Route traffic to R3's LAN. Use the direct link as the primary path, and the link to R2 as the backup floating route.

R1> enable
R1# configure terminal

! Crucial: Enable global IPv6 routing to send NDP RAs for SLAAC
R1(config)# ipv6 unicast-routing

! Primary Route: Pointing directly to R3
! Using a fully specified route with exit interface AND global unicast IP
R1(config)# ipv6 route 2001:db8:0:3::/64 g0/1 2001:db8:0:13::2

! Backup Route: Pointing to R2
! Must specify exit interface because we are using R2's Link-Local address!
! The "5" at the end changes the Administrative Distance to make it a floating route.
R1(config)# ipv6 route 2001:db8:0:3::/64 s0/0/0 fe80::202:16ff:fe17:1645 5

! Verify only the primary route is active
R1(config)# do show ipv6 route
...
S 2001:DB8:0:3::/64 [1/0]
     via 2001:DB8:0:13::2, GigabitEthernet0/1

Goal: Route return traffic to R1's LAN. Use the direct link as primary, and the link to R2 as the backup.

R3> enable
R3# configure terminal

R3(config)# ipv6 unicast-routing

! Primary Route: Pointing directly back to R1
R3(config)# ipv6 route 2001:db8:0:1::/64 g0/1 2001:db8:0:13::1

! Backup Route: Pointing to R2
! Appending "5" makes it a floating static route.
R3(config)# ipv6 route 2001:db8:0:1::/64 s0/0/0 fe80::201:97ff:fe7d:6601 5

Goal: As the transit backup router, R2 needs standard static routes pointing to both R1's LAN and R3's LAN using link-local next-hops.

R2> enable
R2# configure terminal

R2(config)# ipv6 unicast-routing

! Route to R1's LAN
R2(config)# ipv6 route 2001:db8:0:1::/64 s0/0/0 fe80::202:16ff:fe17:1645

! Route to R3's LAN
R2(config)# ipv6 route 2001:db8:0:3::/64 s0/0/1 fe80::203:e4ff:fe4c:8c01

📝 Lab Tasks & Solutions

Check off the steps as you complete them to save your progress.

Command: ipv6 unicast-routing

Answer: Without this command, the routers will not forward IPv6 traffic. Furthermore, this command enables the routers to send out NDP Router Advertisement (RA) messages. The PCs listen for these RAs to learn the network prefix and automatically configure their own IPv6 addresses and default gateways using SLAAC.

If you try to type ipv6 route 2001:db8:0:3::/64 fe80::202:16ff:fe17:1645, Cisco IOS will reject it with an error.

Answer: Link-Local addresses (FE80::) are only unique to the specific physical wire they reside on. The router's routing table cannot resolve which physical port to push the packet out of. You must use a fully specified route (e.g., ... g0/0 fe80::...).

If you run show ipv6 route, you will only see the primary route (AD 1). To verify the floating route was typed correctly, you must inspect the running configuration:

R1# show running-config | include ipv6 route
ipv6 route 2001:DB8:0:3::/64 GigabitEthernet0/1 2001:DB8:0:13::2
ipv6 route 2001:DB8:0:3::/64 Serial0/0/0 FE80::202:16FF:FE17:1645 5

In Packet Tracer, use the "delete" tool to sever the connection between R1 and R3. Then open PC1's command prompt.

C:\> tracert 2001:db8:0:3:201:97ff:fe7d:6601
Tracing route to 2001:db8:0:3:201:97ff:fe7d:6601 over a maximum of 30 hops:
  1    1 ms    1 ms    1 ms  2001:DB8:0:1::1
  2    *       *       *     Request timed out.
  3    2 ms    2 ms    2 ms  2001:db8:0:3:201:97ff:fe7d:6601

Notice the timeout at Hop 2! This is normal. R2 only has link-local addresses on its serial interfaces, which are not routable. It drops the ICMP TTL-exceeded response, but the actual ping packet still successfully reaches the destination (Hop 3).
