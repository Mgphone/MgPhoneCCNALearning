EIGRP Routing

Enhanced Interior Gateway Routing Protocol

EIGRP is an advanced distance-vector routing protocol. In this module, you will master EIGRP terminology (Feasible Distance, Reported Distance), its unique Feasibility Condition for loop prevention, and how to configure Unequal Cost Load Balancing.

🧠 EIGRP Metrics & Concepts

Crucial EIGRP Terminology

  • Feasible Distance (FD) The local router's total metric to reach the destination network.
  • Reported Distance (RD) / Advertised Distance The neighbor router's metric to reach the destination network.
  • Successor The absolute best route to the destination. It is installed in the routing table.
  • Feasible Successor (FS) A valid backup route that meets the Feasibility Condition. It is kept in the topology table for instant failover.

The Feasibility Condition

To become a Feasible Successor (a safe backup route), a route MUST meet this mathematical loop-prevention rule:

RD < Successor's FD

Translation: The neighbor must be strictly closer to the destination than the local router's current best path. If not, it could be a routing loop.

EIGRP Metric Weights (K-Values)

By default, only Bandwidth and Delay are used to calculate the metric.

Metric = Bandwidth (of the slowest link in the path) + Delay (Sum of all link delays).

💻 Device Configuration Lab

Review the commands to configure EIGRP, wildcards, and passive interfaces.

Goal: Configure EIGRP AS 100 using precise wildcard masks, disable auto-summary, and secure the loopback interface.

R1> enable
R1# configure terminal

! Create the loopback interface (Virtual interface)
R1(config)# interface loopback 0
R1(config-if)# ip address 1.1.1.1 255.255.255.255
R1(config-if)# exit

! Initialize EIGRP Process (AS Number MUST match on all routers)
R1(config)# router eigrp 100

! Add specific networks using Wildcard Masks
! (0.0.0.3 represents a /30 subnet)
R1(config-router)# network 10.0.12.0 0.0.0.3
R1(config-router)# network 10.0.13.0 0.0.0.3
R1(config-router)# network 1.1.1.1 0.0.0.0

! Prevent the router from summarizing subnets to their classful boundaries
R1(config-router)# no auto-summary

! Stop sending EIGRP Hello packets out the loopback (Saves CPU)
R1(config-router)# passive-interface loopback 0

Goal: Use a wildcard mask shortcut to instantly enable EIGRP on all active IP addresses on the router.

R4> enable
R4# configure terminal
R4(config)# interface loopback 0
R4(config-if)# ip address 4.4.4.4 255.255.255.255
R4(config-if)# exit

R4(config)# router eigrp 100

! The "Lazy" Network Command
! 255.255.255.255 wildcard means "match any IP address"
R4(config-router)# network 0.0.0.0 255.255.255.255

R4(config-router)# no auto-summary
R4(config-router)# passive-interface loopback 0
R4(config-router)# passive-interface g0/0
Note: While the `0.0.0.0 255.255.255.255` shortcut is fantastic for lab environments to save time, it is not recommended in production because it might accidentally enable EIGRP on guest or internet-facing interfaces.

Goal: Instruct R1 to load-balance traffic to R4 across both the fast and slow paths.

! First, confirm there is a Feasible Successor in the topology table
R1# show ip eigrp topology
...
P 192.168.4.0/24, 1 successors, FD is 28672
        via 10.0.12.2 (28672/28416), GigabitEthernet0/0
        via 10.0.13.3 (30976/28416), FastEthernet0/1

! Enter EIGRP Config
R1# configure terminal
R1(config)# router eigrp 100

! Apply variance multiplier (Successor FD 28672 * 2 = 57344 limit)
! Since 30976 is less than 57344, the backup route is added to the routing table!
R1(config-router)# variance 2

! Verify the routing table now contains TWO routes
R1(config-router)# do show ip route eigrp
D 192.168.4.0/24 [90/28672] via 10.0.12.2, GigabitEthernet0/0
[90/30976] via 10.0.13.3, FastEthernet0/1

A loopback is a stable, virtual interface that never goes down physically. It is often used for router identification.

R1(config)# interface loopback 0
R1(config-if)# ip address 1.1.1.1 255.255.255.255
! Repeat on R2 (2.2.2.2), R3 (3.3.3.3), and R4 (4.4.4.4)

You must use exact wildcard masks to enable EIGRP only on the interfaces connecting to other routers. Disable auto-summary to prevent routing issues with discontiguous networks.

R1(config)# router eigrp 100
R1(config-router)# network 10.0.12.0 0.0.0.3
R1(config-router)# network 10.0.13.0 0.0.0.3
R1(config-router)# network 1.1.1.1 0.0.0.0
R1(config-router)# no auto-summary
R1(config-router)# passive-interface loopback 0

Always make loopback and PC-facing LAN interfaces passive so they don't broadcast EIGRP routing updates to end-users.

! Check Neighbors:
R1# show ip eigrp neighbors

R1# show ip route eigrp
D 192.168.4.0/24 [90/28672] via 10.0.12.2, GigabitEthernet0/0

Notice the route uses the 'D' code for EIGRP (since 'E' is used for EGP).

R1# show ip eigrp topology

R1# configure terminal

R1(config-router)# variance 2

By changing the default variance from 1 to 2, EIGRP takes the best metric (28,672), multiplies it by 2 (57,344), and allows any backup route with a metric lower than 57,344 to be installed into the routing table alongside the primary route.

📋 Command Reference

🔍
Command Mode Purpose
router eigrp [AS_Number] Global Config Enters EIGRP configuration. The AS Number must match across all neighboring routers.
network [ip] [wildcard] Router Config Activates EIGRP on interfaces matching the IP/wildcard.
no auto-summary Router Config Disables classful network summarization. Essential in modern networks.
passive-interface [id] Router Config Stops sending EIGRP Hello packets out the interface, but continues to advertise the network.
variance [multiplier] Router Config Enables Unequal Cost Load Balancing for Feasible Successor routes.
show ip eigrp neighbors Privileged EXEC Verifies that the router has successfully formed adjacencies with neighbors.
show ip eigrp topology Privileged EXEC Displays the EIGRP database, showing all Successors and Feasible Successors.
