IPv4 Addressing

Configuring IPv4 Addresses & Interfaces

This module covers the essential steps for configuring IPv4 addresses on Cisco router interfaces to enable routing between different networks. Master subnet classes, interface statuses, and end-to-end connectivity verification.

Theory & Interface Mechanics

Routers separate broadcast domains and route traffic between different IP networks. For a router to act as a gateway, each active interface must be assigned an IP address belonging to the specific subnet it connects to.

🚦 Interface Status Codes

  • Status (Layer 1 - Physical): Hardware connectivity. If this reads "administratively down," it means the shutdown command is currently active.
  • Protocol (Layer 2 - Data Link): Framing/encapsulation status.
    Note: If Layer 1 is down, Layer 2 will always be down.

💡 Default Behaviors

By default, all interfaces on a Cisco router are shut down. You must explicitly bring them up using the no shutdown command.

Interface descriptions are locally significant only. They do not affect routing, but are critical for real-world administration.

Device Configuration Lab

Review the exact Cisco IOS commands and PC GUI steps required to assign IP addresses across three different subnets.

Goal: Set hostname, apply IPs from 3 different classes to G0/0, G0/1, and G0/2, add descriptions, and enable the interfaces.

Router> enable
Router# configure terminal
Router(config)# hostname R1

! Configure Interface G0/0 (Class A)
R1(config)# interface GigabitEthernet 0/0
R1(config-if)# ip address 15.255.255.254 255.0.0.0
R1(config-if)# description # to switch 1 #
R1(config-if)# no shutdown
R1(config-if)# exit

! Configure Interface G0/1 (Class B)
R1(config)# interface g0/1
R1(config-if)# ip address 182.98.255.254 255.255.0.0
R1(config-if)# description # to switch 2 #
R1(config-if)# no shutdown
R1(config-if)# exit

! Configure Interface G0/2 (Class C)
R1(config)# interface g0/2
R1(config-if)# ip address 201.19.120.254 255.255.255.0
R1(config-if)# description # to switch 3 #
R1(config-if)# no shutdown

! Save Configuration to NVRAM
R1(config-if)# end
R1# copy running-config startup-config

Lab Task Q&A

Below are the expert solutions to the 7 lab tasks from the video. Check off the steps as you complete them to save your progress! Click on a question to expand its answer.

You must enter Global Configuration mode to change the device hostname.

Router> enable
Router# configure terminal
Router(config)# hostname R1

By default, this command is executed in Privileged EXEC mode. If you are in Global Config mode, prepend it with do.

R1(config)# do show ip interface brief

Notice the interfaces are "administratively down" because we haven't configured them yet.

Enter the specific interface configuration mode for each port.

R1(config)# interface g0/0
R1(config-if)# ip address 15.255.255.254 255.0.0.0
R1(config-if)# description # to switch 1 #
R1(config-if)# no shutdown

! ... Repeat for g0/1 (182.98.255.254) and g0/2 (201.19.120.254)

Run the brief summary command again to verify that your IP addresses are applied and the Status/Protocol columns now display "up".

R1# show ip interface brief

Verify the active configuration in RAM, then copy it to the startup configuration in NVRAM.

R1# show running-config
R1# copy running-config startup-config

*Shortcut: write memory or wr.

Done via the Packet Tracer GUI (Config Tab -> FastEthernet0). Packet Tracer will auto-fill the subnet mask based on the first octet.

  • PC1: 15.0.0.1 (Mask: 255.0.0.0)
  • PC2: 182.98.0.1 (Mask: 255.255.0.0)
  • PC3: 201.19.120.1 (Mask: 255.255.255.0)

Open PC1's Desktop tab, open the Command Prompt, and issue the ping commands.

PC1> ping 182.98.0.1
PC1> ping 201.19.120.1

Success indicates that R1 is successfully routing traffic between the three different subnets.
