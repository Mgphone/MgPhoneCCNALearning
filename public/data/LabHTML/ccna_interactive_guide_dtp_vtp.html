🧠 DTP & VTP Fundamentals

Dynamic Trunking Protocol (DTP)

DTP dynamically negotiates whether a link should be an access port or a trunk port. By default, many Cisco switches operate in dynamic auto mode, meaning they passively wait for the other side to initiate trunking.

Security Best Practice: Disable DTP using switchport nonegotiate on static trunks, and hardcode edge ports with switchport mode access to prevent VLAN hopping attacks.

VLAN Trunking Protocol (VTP)

VTP allows a network administrator to manage VLANs on one switch and have those changes automatically propagated to all other switches in the same VTP Domain.

  • Server: The default mode. Can create, modify, and delete VLANs. Propagates changes.
  • Client: Cannot create/modify local VLANs. Syncs its database with the Server's advertisements.
  • Transparent: Can create local VLANs, but does not sync with the Server. It does forward VTP advertisements to other switches.

💻 Device Configuration Lab

Review the specific commands to configure hardcoded trunks and VTP modes.

Goal: Hardcode trunk to SW2, disable DTP, set VTP domain, and create VLANs to propagate.

SW1> enable
SW1# configure terminal

! 1. Hardcode Trunk & Disable DTP Negotiation
SW1(config)# interface g0/1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport nonegotiate
SW1(config-if)# exit

! 2. Configure VTP (Defaults to Server Mode)
SW1(config)# vtp domain CCNA
Changing VTP domain name from null to CCNA

! 3. Create VLANs (Increases Rev Number, syncs to domain)
SW1(config)# vlan 10
SW1(config-vlan)# vlan 20
SW1(config-vlan)# vlan 30
SW1(config-vlan)# exit

! 4. Secure access ports & disable DTP
SW1(config)# interface range f0/1 - 2
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 10

Goal: Hardcode trunks to SW1 & SW3, set VTP Transparent mode, and create isolated VLAN 40.

SW2> enable
SW2# configure terminal

! 1. Hardcode Trunks & Disable DTP Negotiation
SW2(config)# interface range g0/1 - 2
SW2(config-if-range)# switchport mode trunk
SW2(config-if-range)# switchport nonegotiate
SW2(config-if-range)# exit

! 2. Configure VTP Transparent Mode
SW2(config)# vtp mode transparent
Setting device to VTP Transparent mode for VLANS.

! 3. Create Local VLAN (Will NOT sync to domain)
SW2(config)# vlan 40
SW2(config-vlan)# exit

Goal: Hardcode trunk to SW2, set VTP Client mode, and attempt to create a VLAN (which will fail).

SW3> enable
SW3# configure terminal

! 1. Hardcode Trunk & Disable DTP Negotiation
SW3(config)# interface g0/1
SW3(config-if)# switchport mode trunk
SW3(config-if)# switchport nonegotiate
SW3(config-if)# exit

! 2. Configure VTP Client Mode
SW3(config)# vtp mode client
Setting device to VTP Client mode.

! 3. Attempt to create VLAN (Demonstrates Client limitation)
SW3(config)# vlan 50
VTP VLAN configuration not allowed when device is in CLIENT mode.

! 4. Secure access ports & disable DTP
SW3(config)# interface f0/1
SW3(config-if)# switchport mode access
SW3(config-if)# switchport access vlan 10

Administrative Mode: trunk

Answer: Yes. Because SW2 and SW3 had a "null" domain name, they automatically joined the `CCNA` domain upon receiving SW1's advertisement. Since they default to Server mode, they accepted the higher Revision Number and synced VLANs 10, 20, and 30.

SW2(config)# vtp mode transparent
SW2(config)# vlan 40

Answer: No. Transparent switches maintain an isolated VLAN database (Revision Number stays at 0). They do not advertise their local VLANs to the domain. Thus, SW1 and SW3 do not learn about VLAN 40.

SW3(config)# vtp mode client
SW3(config)# vlan 50
VTP VLAN configuration not allowed when device is in CLIENT mode.

Answer: No. VTP Client mode strictly forbids the creation, modification, or deletion of local VLANs. It can only listen and sync to Server advertisements.

SW3(config)# interface f0/1
SW3(config-if)# switchport mode access
SW3(config-if)# do show interface f0/1 switchport

Answer: No. Manually setting switchport mode access automatically disables DTP negotiation on that interface. You will see Negotiation of Trunking: Off in the output.

📋 Command Reference

🔍
Command Mode Purpose
switchport mode trunk Interface Config Hardcodes the interface to operate as an 802.1Q trunk.
switchport nonegotiate Interface Config Stops the switch from sending DTP negotiation frames out the interface.
vtp domain [name] Global Config Sets the VTP domain. Switches with null domains automatically join the first domain they hear.
vtp mode [server|client|transparent] Global Config Changes the VTP operating mode. Default is Server.
vtp password [password] Global Config Secures VTP advertisements. Switches will reject syncs if passwords mismatch.
show vtp status Privileged EXEC Displays VTP version, domain, mode, and crucially, the Configuration Revision number.
show interface [id] switchport Privileged EXEC Reveals Admin/Operational modes and if "Negotiation of Trunking" is On/Off.
