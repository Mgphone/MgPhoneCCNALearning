IPv4 Interfaces & Config

Configuring IPv4 Addresses & Interfaces

This module covers the essential steps for configuring IPv4 addresses on Cisco router interfaces to enable routing between different networks. You will learn how to assign IP addresses based on Class A, B, and C subnets, use descriptions, enable interfaces, and verify connectivity.

🧠 Conceptual Breakdown

Routers separate broadcast domains and route traffic between different IP networks. For a router to function as a gateway, each active interface must be assigned an IP address belonging to the specific subnet it connects to.

Interface Statuses

  • ↳
    Status (Layer 1 - Physical): If this reads "administratively down," it means the shutdown command is currently active.
  • ↳
    Protocol (Layer 2 - Data Link): If Layer 1 is down, Layer 2 will always be down.

Default States & Descriptions

By default, all interfaces on a Cisco router are shut down. You must explicitly bring them up using the no shutdown command.

CCNA Exam Tip: Interface descriptions are locally significant only. They do not affect routing or operational status, but are critical for real-world administration.

💻 Step-by-Step Router (R1) Configuration

Follow these exact CLI steps to apply hostnames, configure interfaces across different subnets, and commit your changes to NVRAM memory.

1. Enter Global Configuration Mode and Set Hostname

Router> enable
Router# configure terminal
Router(config)# hostname R1

2. Configure Interface GigabitEthernet 0/0 (Class A)

R1(config)# interface GigabitEthernet 0/0
R1(config-if)# description # to switch 1 #
R1(config-if)# ip address 15.255.255.254 255.0.0.0
R1(config-if)# no shutdown
R1(config-if)# exit

3. Configure Interface GigabitEthernet 0/1 (Class B)

R1(config)# interface g0/1
R1(config-if)# description # to switch 2 #
R1(config-if)# ip address 182.98.255.254 255.255.0.0
R1(config-if)# no shutdown
R1(config-if)# exit

4. Configure Interface GigabitEthernet 0/2 (Class C)

R1(config)# interface g0/2
R1(config-if)# description # to switch 3 #
R1(config-if)# ip address 201.19.120.254 255.255.255.0
R1(config-if)# no shutdown

5. Save Configuration

R1(config-if)# end
R1# write memory

📋 Command Reference & Verification

Use the search box to quickly filter Cisco IOS commands. Use these commands to configure interfaces and verify the operational status of your network.

🔍
Command Mode Purpose
show ip interface brief Privileged EXEC Displays a summary of all interfaces, their assigned IP addresses, and their current Layer 1 / Layer 2 status.
interface [type] [number] Global Config Enters interface configuration mode for the specified interface (e.g., interface g0/0).
description [text] Interface Config Adds a text description to the interface for documentation purposes.
ip address [ip] [mask] Interface Config Assigns an IPv4 address and subnet mask to the interface.

Verification & Troubleshooting Steps

1. Verify Status

R1# show ip interface brief

Ensure that the Status and Protocol columns both say up for your configured interfaces. If the Status says "administratively down," you forgot to issue the no shutdown command.

2. Verify Config Applied

R1# show running-config

Scroll through the output to verify that your IP addresses, subnet masks, and descriptions are applied correctly under each interface block.

You must enter Global Configuration mode to change the device hostname.

Router> enable
Router# configure terminal
Router(config)# hostname R1

By default, this command is executed in Privileged EXEC mode. If you are in Global Config mode, you must prepend the command with do.

R1# show ip interface brief

You must enter the specific interface configuration mode for each port to apply IPs, descriptions, and bring them online.

GigabitEthernet 0/0 (Class A):

R1(config)# interface g0/0
R1(config-if)# ip address 15.255.255.254 255.0.0.0
R1(config-if)# description # to switch 1 #
R1(config-if)# no shutdown

GigabitEthernet 0/1 (Class B):

R1(config-if)# interface g0/1
R1(config-if)# ip address 182.98.255.254 255.255.0.0
R1(config-if)# description # to switch 2 #
R1(config-if)# no shutdown

GigabitEthernet 0/2 (Class C):

R1(config-if)# interface g0/2
R1(config-if)# ip address 201.19.120.254 255.255.255.0
R1(config-if)# description # to switch 3 #
R1(config-if)# no shutdown

Run the brief summary command again to verify that your IP addresses are applied and the Status/Protocol columns now display "up".

R1# show ip interface brief

Verify the active configuration in RAM, then copy it to the startup configuration in NVRAM.

R1# show running-config
R1# copy running-config startup-config

*(Alternatively, you can use the shortcut command write memory or wr)*

Because PCs do not use the Cisco IOS CLI, this is done via the Packet Tracer GUI.

  1. Click on the PC in the logical workspace.
  2. Go to the Config tab.
  3. On the left panel, click on FastEthernet0 (the Network Interface Card).
  4. Enter the IPv4 address. Packet Tracer will automatically populate the default subnet mask based on the address class.
    • PC1: IP Address 15.0.0.1 (Mask auto-populates to 255.0.0.0)
    • PC2: IP Address 182.98.0.1 (Mask auto-populates to 255.255.0.0)
    • PC3: IP Address 201.19.120.1 (Mask auto-populates to 255.255.255.0)

*Note: You must also configure the Default Gateway (pointing to R1's IP for that subnet) in the Global Settings tab for inter-VLAN routing, though basic pinging on the local subnet works without it.

Open PC1, navigate to the Desktop tab, open the Command Prompt, and issue the ping commands to the IP addresses of the other PCs.

PC1> ping 182.98.0.1
PC1> ping 201.19.120.1

*(Success indicates that R1 is successfully routing traffic between the three different subnets).*
