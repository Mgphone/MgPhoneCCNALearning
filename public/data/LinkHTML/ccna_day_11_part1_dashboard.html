The Routing Process

What exactly does a router do when a frame arrives on its interface?

1

De-encapsulate

The router receives the L2 Frame. It checks the Destination MAC. If it matches the router's interface MAC, it strips off the L2 Header and Trailer.

2

Lookup

The router examines the L3 Packet's Destination IP address. It searches its Routing Table for the best match.

3

Re-encapsulate & Forward

The router packages the L3 Packet into a BRAND NEW L2 Frame (New Source MAC = Router's exit int, New Dest MAC = Next hop). It forwards it out the exit interface.

Rule: IP Addresses stay the same end-to-end.
MAC Addresses change at EVERY router hop.
🗺️

The Routing Table: C & L Routes

A router needs to know where to send packets. The most basic routes are generated automatically when you configure an IP address on an interface and bring it up/up.

C Connected Route

Represents the entire subnet attached to the interface.

C   192.168.1.0/24 is directly connected, GigabitEthernet0/0

L Local Route

Represents the router's exact interface IP address. It always has a /32 mask. (Added in IOS 15).

L   192.168.1.1/32 is directly connected, GigabitEthernet0/0

📏 Longest Prefix Match

The Golden Rule of Routing: If a destination IP matches multiple routes in the routing table, the router will ALWAYS choose the most specific route.

The "most specific" route is the one with the highest subnet mask number (e.g., /24 beats /16).

Scenario: Packet arrives for 10.1.1.50

  • Route 1: 10.0.0.0 /8 (Match! First octet matches)
  • Route 2: 10.1.1.0 /24 (Match! First 3 octets match)

Winner: Route 2. It has a longer prefix (/24 > /8), making it a more precise match.

🪝 Brain Hooks

The Envelope vs The Mail Truck

The Destination IP is the address written on the envelope. It never changes.
The MAC Address is the license plate of the mail truck. It changes every time the envelope is handed off to a new truck (router).

"L" for "Literally Me"

The L (Local) route with a /32 mask means "This is literally my own IP address." If traffic hits this route, it's meant for the router itself (like SSH or Ping).

⌨️ CLI Commands: The Routing Table

Router# show ip route Displays the entire IPv4 routing table. Shows the codes (C, L, O, S) and the routing entries.
Router# show ip route connected Filters the output to display ONLY the directly connected ('C' and 'L') networks.
Router# show ip route 192.168.1.100 Incredibly useful! Forces the router to do a lookup for this specific IP and tells you exactly which route it would use (verifies Longest Prefix Match).

🔧 Troubleshooting

Missing 'C' and 'L' Routes

Issue: You configured an IP address on Gigabit0/0, but it doesn't show up in show ip route.

Fix: Run show ip interface brief. The interface must be physically UP and protocol UP. Did you forget to type no shutdown?

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. True or False: When a router forwards a packet, it rewrites the Source IP address to be its own IP address. ▼
False. IP addresses stay the same end-to-end. The router strips the old L2 frame and creates a new one, meaning only the MAC addresses change.
2. In the routing table, what is the primary difference between a 'C' route and an 'L' route? ▼
'C' (Connected) represents the entire subnet network address. 'L' (Local) is a /32 route representing the exact IP address assigned to the router's interface.
3. What subnet mask is always associated with a Local (L) route? ▼
/32. It is a "Host Route" pointing to one single IP address.
4. A packet arrives for 10.1.1.15. The table has Route A (10.1.1.0 /24) and Route B (10.1.1.0 /26). Which route is chosen and why? ▼
Route B (/26) is chosen. Due to the rule of Longest Prefix Match, the router always selects the most specific route available (the one with the highest slash number).
