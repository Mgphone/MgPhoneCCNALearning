Day 49: Port Security

Layer 2 Endpoint Protection

🛡️

Why Port Security?

By default, switch ports allow any device to plug in and send traffic. Port Security restricts a port's input by limiting and identifying the MAC addresses allowed to use it.

Mitigates MAC Flooding

Attackers use tools (like macof) to generate thousands of fake MAC addresses per second. This overflows the switch's MAC Address Table (CAM Table). When the table is full, the switch acts like a hub and floods all traffic, allowing the attacker to sniff data.

Mitigates Rogue Devices

Prevents users from bringing unmanaged switches from home and plugging them into a wall jack to give their friends access, or plugging in rogue DHCP servers.

The Golden Prerequisite

Port security can ONLY be enabled on a statically configured Access port (or static trunk). It will fail on a port set to `dynamic auto` or `dynamic desirable`.

🧠

MAC Learning Methods

How does the switch know which MAC addresses are "trusted"?

1. Static

You manually type the exact MAC address of the trusted PC into the CLI.

  • Highly secure but terrible for administration.
  • Saved in the running-config.

2. Dynamic (Default)

The switch trusts the first MAC(s) it hears on the wire up to the maximum limit.

  • Easy to configure.
  • Lost on reboot. If the switch reboots, it trusts the first device to speak again.

3. Sticky

Learns dynamically like above, but instantly writes the MAC address into the running-config as if you typed it statically.

  • Best of both worlds.
  • Survives a reboot (if you run `copy run start`).

🚨 Violation Modes

What happens when an unauthorized MAC address tries to send traffic on a secured port? You must memorize this matrix.

Mode Action on Traffic Syslog Message? Violation Counter? Port State
Protect Drops unauthorized No No Stays UP
Restrict Drops unauthorized Yes Increments Stays UP
Shutdown (Default) Drops ALL traffic Yes Increments (to 1) Err-Disabled

* Note: "Shutdown" is the default. It requires manual intervention (shut / no shut) to recover unless `errdisable recovery` is configured globally.

🪝 Brain Hooks

The Bouncer Analogy

Protect: Silent bouncer. Turns away uninvited guests. Doesn't tell the boss.
Restrict: Snitch bouncer. Turns away guests AND radios the boss (Syslog) to report them.
Shutdown: The nuclear bouncer. If an uninvited guest shows up, he locks the front door and shuts down the entire club.

Sticky = Super Glue

Sticky learning "glues" the MAC address directly into your running configuration file.

⌨️ Port Security Configuration

Switch(config-if)# switchport mode access MANDATORY: Disables DTP and forces access mode. Port security will reject the next command if this is missing.
Switch(config-if)# switchport port-security Enables the feature. Applies default settings: Max MACs = 1, Violation = Shutdown, Learning = Dynamic.
Switch(config-if)# switchport port-security maximum 2 Increases the limit to 2 MAC addresses (Useful if a PC is daisy-chained behind an IP Phone).
Switch(config-if)# switchport port-security mac-address sticky Enables Sticky MAC learning.
Switch(config-if)# switchport port-security violation restrict Changes the violation mode from Shutdown to Restrict.

🔧 Verification & Recovery

show port-security interface f0/1

Port Security              : Enabled
Port Status                : Secure-down
Violation Mode             : Shutdown
Maximum MAC Addresses      : 1
Total MAC Addresses        : 1
Security Violation Count   : 1

The master verification command. If 'Port Status' is Secure-down, the port was tripped by a violation.

Recovering from Err-Disable

If a port hits the default Shutdown mode, it goes into `err-disable`. The port LED turns off.

Manual Recovery: Go to the interface, type shutdown, then type no shutdown.

Auto Recovery (Global Config):
errdisable recovery cause psecure-violation
errdisable recovery interval 30 (Auto-recovers after 30 seconds)

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. What is the default violation mode when you enable port security on a Cisco switch? ▼
Shutdown. The port will drop all traffic and enter an err-disabled state upon a violation.
2. Which violation mode drops unauthorized traffic, increments the security violation counter, generates a syslog message, but leaves the port in an UP state? ▼
Restrict. (Remember: Protect mode does all of this EXCEPT it does not generate a syslog message or increment the counter).
3. Which MAC learning method dynamically learns a MAC address from incoming traffic and permanently writes it into the running-config file? ▼
Sticky learning (mac-address sticky).
4. You type `switchport port-security` and the switch replies with "Command rejected". What prerequisite did you likely forget? ▼
You must manually disable DTP by configuring the port statically using switchport mode access (or `mode trunk`). Port security cannot be enabled on a dynamic port.
