Day 58: Wireless Config

WLC Interfaces, WLANs & The AP Join Process

🎛️

WLC Logical Interfaces

A Wireless LAN Controller connects to the network via physical ports, but configuration is done on Logical Interfaces mapped to VLANs.

Management Interface

The primary interface for the WLC. Used by admins to access the GUI (HTTPS) and CLI (SSH). It is also the IP address that APs communicate with to establish CAPWAP tunnels.

Dynamic (VLAN) Interfaces

Created by the admin to handle client traffic. If you have an HR WLAN and a Guest WLAN, you create two Dynamic interfaces, mapping them to VLAN 10 and VLAN 20 respectively.

Virtual Interface

Used for mobility management (roaming) and Web Authentication portals. It is an unroutable, dummy IP address (commonly configured as 1.1.1.1).

AP-Manager Interface

Historically used to terminate CAPWAP tunnels from APs. Note: In modern Cisco WLCs, this function is typically merged directly into the Management Interface.

🤝

The AP Join Process

When a new Lightweight AP is plugged in, how does it find the WLC? It uses a strict order of operations:

  1. Broadcast: The AP sends a local Layer 3 broadcast (255.255.255.255) hoping the WLC is on the same subnet.
  2. DHCP Option 43: When the AP gets its IP address via DHCP, the server provides Option 43, which explicitly lists the WLC's IP address.
  3. DNS Resolution: The AP queries the DNS server for the specific hostname: CISCO-CAPWAP-CONTROLLER.localdomain.

Once the IP is found, the AP sends a CAPWAP Join Request to the WLC (UDP 5246).

⚙️ Creating a WLAN in the GUI

You are expected to know the sequence of creating a new wireless network (WLAN) in the Cisco WLC web interface.

1

General Tab

Define the core identities:

  • Profile Name: Internal admin name.
  • SSID: The name broadcasted to users.
  • WLAN ID: A unique number (e.g., 1-512).
  • Status: Check the box to enable it!
2

Interface Mapping

Still on the General tab (or sometimes Interface tab):

  • Map the WLAN to a Dynamic Interface.
  • This binds the wireless SSID to a specific wired VLAN (e.g., "Guest_SSID" maps to "VLAN20_Interface").
3

Layer 2 Security

Under the Security > Layer 2 tab:

  • Select WPA+WPA2.
  • Choose Authentication Key Management (AKM): PSK (Personal) or 802.1X (Enterprise).
  • Check the AES encryption box.
4

AAA Servers (Enterprise)

If using 802.1X, go to Security > AAA Servers tab:

  • Select your pre-configured RADIUS server from the drop-down list.
  • The WLC will now forward client credentials to ISE/RADIUS for validation.

🔒 L2 vs L3 Security

The WLC Security tab separates policies into Layer 2 and Layer 3.

Layer 2 Security

Handles wireless encryption and access to the radio waves themselves.
(e.g., WPA2/WPA3, PSK, 802.1X, MAC Filtering).

Layer 3 Security

Handles security after the device connects to the radio.
(e.g., Web Authentication captive portals for Guests, IPSec VPN pass-through).

🪝 Brain Hooks

AP Join Checklist (B.D.D)

How does an AP find the controller? Think B.D.D.
1. Broadcast (Local)
2. DHCP Option 43
3. DNS (CISCO-CAPWAP-CONTROLLER)

Virtual Interface = Fake IP

The Virtual Interface (usually 1.1.1.1) isn't a real routing endpoint. It's a "fake" IP used by the WLC to intercept guest traffic and redirect it to a Web Login page.

🔧 Troubleshooting & WLC Verification

AP Won't Join Controller

Check: Is DHCP working? If the AP gets an IP, did the scope include Option 43? If not, verify the DNS server has an A-record for CISCO-CAPWAP-CONTROLLER.

WLAN Not Broadcasting

Check: Go to the WLANs tab. Look at the "Admin Status". Did you forget to check the "Enabled" box on the General tab?

show ap summary

CLI command on the WLC to verify which APs have successfully built CAPWAP tunnels and joined the controller.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Which specific DHCP Option number is configured on a server to hand out the Wireless LAN Controller's IP address to a joining AP? ▼
DHCP Option 43.
2. Which logical WLC interface is used specifically to support mobility management (roaming) and redirect users to Web Authentication portals? ▼
The Virtual Interface (commonly configured as an unroutable IP like 1.1.1.1).
3. In the WLC GUI, under which security tab (Layer 2 or Layer 3) would you configure WPA2 Enterprise (802.1X) authentication? ▼
Layer 2 Security. (Layer 3 handles post-connection policies like captive web portals).
4. To tie a newly created WLAN (SSID) to a specific wired VLAN on the network, what type of logical interface must you map the WLAN to? ▼
A Dynamic Interface. (The admin creates dynamic interfaces and maps them directly to physical VLANs).
