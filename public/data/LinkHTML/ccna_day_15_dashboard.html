What is VLSM?

Variable Length Subnet Masking (VLSM) is the process of "subnetting a subnet." It allows you to use multiple different subnet masks within the same classful network space.

The Problem without VLSM

If you only use one subnet mask (like a /26) for everything, all subnets will have exactly 62 hosts. If you assign a /26 to a point-to-point WAN link (which only needs 2 IPs), you waste 60 IP addresses!

The VLSM Solution

VLSM lets you tailor the subnet size to the exact number of hosts needed. You can carve out a /25 (126 hosts) for a large LAN, and a /30 (2 hosts) for a WAN link, all from the same 192.168.1.0/24 block.

Classless Routing Requirement VLSM only works with Classless routing protocols (OSPF, EIGRP, RIPv2) because they send the subnet mask along with the routing updates. Legacy Classful protocols (RIPv1) do not support VLSM.
📏

The Golden Rule of VLSM

Always allocate the LARGEST subnets first!

If you start by allocating small subnets (like /30s) at the beginning of your address block, you will severely fragment your IP space. You might not have a large enough contiguous block left over to fit a larger subnet (like a /25) later on.

Right Way (Largest -> Smallest)

/25 (128 block) -> /26 (64 block) -> /30 (4 block)

Wrong Way (Smallest -> Largest)

/30 (4 block) -> /26 (64 block) -> ERROR (Fragmented!)

📋 VLSM Cheat Sheet

Memorize these specific prefixes, as they are used constantly in VLSM design scenarios.

  • /25 128 block 126 hosts
  • /26 64 block 62 hosts
  • /27 32 block 30 hosts
  • /28 16 block 14 hosts
  • /29 8 block 6 hosts
  • /30 4 block 2 hosts (P2P WAN)

🛠️ Building a VLSM Plan

Given Network: 192.168.1.0 /24
Needs: HQ (100 hosts), Branch (50 hosts), WAN Link (2 hosts).

Name & Need Mask & Block Network ID Broadcast ID
1. HQ (100) /25 (128 block) 192.168.1.0 192.168.1.127
2. Branch (50) /26 (64 block) 192.168.1.128 192.168.1.191
3. WAN (2) /30 (4 block) 192.168.1.192 192.168.1.195

Notice how we start the next network exactly where the previous broadcast left off (e.g., HQ ends at .127, Branch begins at .128). Nothing overlaps, and space is perfectly optimized.

🪝 Brain Hooks

The Cake Analogy

Think of a /24 as a whole cake. You can slice it in half (two /25s). You can then take the second half and slice it into quarters (two /26s). You are just carving up the same cake into smaller and smaller slices without wasting any.

/30 = WAN

Whenever you see a point-to-point link between two routers, immediately think /30. (It provides exactly 2 usable hosts).

🔧 Troubleshooting VLSM Overlaps

A common exam question presents several IP address/Mask pairs and asks which ones overlap. To solve this:

  1. Calculate the Network ID and Broadcast ID for every single interface/subnet in question.
  2. Look at the ranges (Network ID to Broadcast ID).
  3. If any part of one range falls inside another range, they overlap, and a Cisco router will reject the configuration.

Ex: 10.0.0.0/24 (Range 0-255) overlaps with 10.0.0.128/25 (Range 128-255). You cannot use both on the same router.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. What is the fundamental "Golden Rule" when designing a VLSM addressing scheme? ▼
Always allocate the largest subnets first (the ones requiring the most hosts), and work your way down to the smallest. This prevents address space fragmentation.
2. You need to connect two routers together via a point-to-point WAN link. What is the most efficient subnet mask to use? ▼
A /30 subnet mask. It provides exactly 4 IP addresses (1 Network, 1 Broadcast, and exactly 2 usable hosts for the two routers).
3. What category of routing protocols is required to support VLSM, because they include the subnet mask in their routing updates? ▼
Classless routing protocols (e.g., OSPF, EIGRP, RIPv2). Legacy classful protocols (like RIPv1) do not send subnet mask information and assume default classful boundaries.
4. You need a subnet to support 60 hosts. Which subnet mask provides enough space with the LEAST amount of wasted IP addresses? ▼
A /26 subnet mask. It has a block size of 64, providing exactly 62 usable hosts ($2^6 - 2$). A /27 only provides 30 usable hosts, which is too small.
