Day 48: Security Fundamentals

Core Concepts, Threats, and Defense Mechanisms

📖

Security Vocabulary

Before configuring firewalls or policies, you must master the standard terminology used to describe security postures and events.

Vulnerability

A flaw or weakness in a system's design, implementation, or operation.
Ex: Using an outdated, unpatched version of SSH.

Threat

A potential danger to an asset. Someone or something that might exploit a vulnerability.
Ex: A malicious hacker or a natural disaster.

Exploit

The specific mechanism, script, or tool used by a threat to take advantage of a vulnerability.
Ex: A piece of code that triggers a buffer overflow.

Risk

The likelihood of a threat exploiting a vulnerability and the resulting business impact.
Risk = Threat × Vulnerability × Impact

🛡️

The CIA Triad

The cornerstone model of information security. Every security policy is designed to protect one or more of these three pillars.

Confidentiality

Ensuring data is accessed only by authorized individuals.

Protected by:

  • Encryption (AES, IPsec)
  • Strong Access Controls

Integrity

Ensuring data has not been altered or tampered with in transit.

Protected by:

  • Cryptographic Hashes (SHA)
  • Digital Signatures

Availability

Ensuring systems and data are consistently accessible when needed.

Protected by:

  • Redundancy (FHRP, EtherChannel)
  • DDoS Mitigation

🔑 The AAA Framework

How we securely manage access to networks and devices.

1. Authentication (Who are you?)

Verifying identity using something you know (password), something you have (token), or something you are (biometrics).

2. Authorization (What can you do?)

Determining permissions. Ex: Giving a junior admin Privilege Level 1, but a senior admin Privilege Level 15.

3. Accounting (What did you do?)

Meticulously logging actions, times, and changes for security auditing and forensics. Often centralized via TACACS+ or RADIUS servers.

⚔️ Common Attack Vectors

DoS & DDoS

Denial of Service. Overwhelming a system with illegitimate traffic to exhaust its CPU, RAM, or bandwidth, taking it offline.

Spoofing

Falsifying data to masquerade as a trusted entity. Examples include MAC Spoofing (to bypass Port Security) or IP Spoofing.

Man-in-the-Middle (MitM)

Secretly intercepting and potentially altering communications between two parties. Often achieved via ARP Poisoning on a local LAN.

Brute-Force & Dictionary

Systematic guessing of passwords. Brute-force tries every possible character combination. Dictionary attacks use massive wordlists of common passwords.

🦠 Malware Categories

Type Characteristics
Virus Malicious code attached to a file. Requires human interaction (execution) to spread.
Worm Self-replicating. Actively exploits network vulnerabilities to spread across networks without human intervention.
Trojan Disguised as legitimate, desirable software to trick users into installing it voluntarily.
Ransomware Encrypts user data or system drives, demanding financial payment for the decryption key.

🪝 Brain Hooks

The 90% Rule

Despite advanced firewalls, Social Engineering (phishing) remains the most effective attack vector. Attackers target the human element to voluntarily hand over credentials, bypassing all perimeter security.

Defense in Depth

The strategy of using multiple, overlapping layers of security. If the firewall fails, the IPS catches it. If the IPS fails, the endpoint antivirus stops it.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. What type of malware is self-replicating and can aggressively spread across a network without requiring any user interaction? ▼
A Worm. (Unlike a virus, which requires a user to execute an infected file).
2. Within the AAA framework, which component is primarily responsible for generating audit trails to track what a user did after they logged in? ▼
Accounting.
3. A Distributed Denial of Service (DDoS) attack is primarily designed to compromise which specific pillar of the CIA Triad? ▼
Availability. By exhausting resources, the service becomes unavailable to legitimate users.
4. What is the precise difference between a Vulnerability and an Exploit? ▼
A Vulnerability is the inherent flaw or weakness in the system itself. An Exploit is the specific script, payload, or technique used by an attacker to leverage that flaw.
