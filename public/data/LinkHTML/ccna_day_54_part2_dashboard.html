Containers, NFV, VRFs, and vSwitches

Containers Under the Hood

We know Containers share the Host OS Kernel instead of booting a full Guest OS. But how do they stay secure and isolated from each other while sharing the exact same core OS?

Namespaces (Isolation)

A Linux feature that partitions kernel resources. It ensures that a container only sees its own processes, its own virtual network interfaces, and its own file system. Container A literally cannot "see" Container B.

Cgroups (Control Groups)

Limits resource usage. Cgroups ensure that one container doesn't consume 100% of the host's CPU or RAM, preventing "noisy neighbor" scenarios and stabilizing the host.

The Container Engine

Software like Docker acts as the engine. It handles building the images, deploying them, and configuring the namespaces and cgroups so you don't have to do it manually.

🔀

Virtual Switches (vSwitch)

If you have 20 VMs or 100 Containers running on a single physical server, how do they talk to the network? A server only has one or two physical Network Interface Cards (pNICs)!

The vSwitch Architecture

  • The Hypervisor (or Container Engine) creates a software-based switch inside the server.
  • VMs are given vNICs (virtual NICs) that cable directly into the vSwitch.
  • The vSwitch connects to the physical pNIC (acting as an uplink/trunk) to reach the physical Cisco network.

Docker Network Modes

  • Bridge (Default): Creates an internal private NAT network. Containers can talk to each other, but the outside world must use port forwarding to reach them.
  • Host: Bypasses the vSwitch. The container binds directly to the Host's IP and ports.
  • None: Total isolation. No network access at all.

🖥️ NFV

Network Functions Virtualization is the concept of taking traditional, dedicated hardware appliances (Routers, Firewalls) and turning them into Virtual Machines.

Cisco NFV Examples
  • ASAv: Cisco ASA Firewall as a VM.
  • CSR 1000v / Catalyst 8000v: Full Cisco IOS-XE Routers running as VMs.
  • WLCv: Virtual Wireless LAN Controller.
The Benefit

Instead of shipping physical hardware to a new branch office, you ship a generic x86 server (Cisco ENCS) and simply spin up a Router VM and a Firewall VM remotely!

🔀 VRF (Virtual Routing and Forwarding)

Just as VLANs virtualize a Layer 2 Switch into multiple isolated switches, VRFs virtualize a Layer 3 Router into multiple isolated routing tables.

How it Works

By default, a router has one Global Routing Table. With VRFs, you create multiple tables (e.g., VRF RED, VRF BLUE). Interfaces are assigned to specific VRFs.

Traffic in VRF RED cannot cross over or route into VRF BLUE. They are completely segregated.

The Power of VRFs

Because the routing tables are 100% isolated, you can reuse the exact same IP subnets!

VRF RED can use `10.0.0.0/24` and VRF BLUE can also use `10.0.0.0/24` on the exact same physical router without causing an overlap error.

🪝 Brain Hooks

VLAN vs. VRF

VLAN: Slices a Switch into multiple isolated Layer 2 Broadcast Domains.

VRF: Slices a Router into multiple isolated Layer 3 Routing Tables.

NFV vs SDN

NFV virtualizes the network hardware (turning a physical router into a VM).
SDN virtualizes the network control (separating the control plane from the data plane).

⌨️ VRF Configuration CLI

Router(config)# ip vrf GUEST Creates a new Virtual Routing table named 'GUEST'.
Router(config)# interface g0/1 Router(config-if)# ip vrf forwarding GUEST Assigns the interface to the VRF. WARNING: Doing this instantly deletes any IP address currently on the interface! You must reassign the IP afterward.
Router# show ip route vrf GUEST Views the routing table specifically for the GUEST VRF (ignores the global table).
Router# ping vrf GUEST 10.1.1.5 To ping a device inside a VRF, you must specify the VRF name, otherwise the router checks the global table and fails.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Which two underlying Linux technologies allow Containers to achieve isolation and resource limiting while sharing a single OS kernel? ▼
Namespaces (for process/network isolation) and Cgroups (for CPU/RAM resource limits).
2. Replacing a physical hardware ASA firewall with an ASAv Virtual Machine running on an ESXi host is an example of what concept? ▼
NFV (Network Functions Virtualization).
3. Because VRFs completely isolate routing tables from one another, what specific IP addressing scenario is made possible on a single router? ▼
Overlapping IP Addresses. You can configure the exact same IP subnet (e.g., 192.168.1.0/24) on two different interfaces, as long as they belong to different VRFs.
4. What is the default networking mode for Docker containers, which relies on a virtual switch and NAT to communicate externally? ▼
Bridge mode.
