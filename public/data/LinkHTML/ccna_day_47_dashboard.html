Shaping, Policing, Queuing & MQC

The QoS Toolkit

Once traffic is classified and marked (Day 46), the router must apply policies to it. There are three main tools in the QoS toolbelt.

1. Congestion Management (Queuing)

When an interface is congested (more traffic arriving than can be sent), packets are placed into queues (buffers). The router then decides which queue gets emptied first.

2. Congestion Avoidance

If a queue gets completely full, the router must drop packets (Tail Drop). Congestion Avoidance randomly drops less important packets early to prevent the queue from ever filling up completely.

3. Shaping & Policing

Techniques used to strictly limit the bandwidth rate of certain traffic (e.g., limiting Guest Wi-Fi to 10 Mbps).

✂️

Policing vs. Shaping

You must know the critical difference between these two rate-limiting techniques for the exam.

Policing (The Chopper)

  • Action: Drops (or remarks) traffic that exceeds the configured rate limit.
  • Effect: Causes TCP retransmissions, leading to a jagged, sawtooth-shaped traffic graph.
  • Direction: Can be applied Inbound OR Outbound.
  • Use Case: Dropping excessive traffic arriving from an untrusted source.

Shaping (The Smoother)

  • Action: Buffers (delays) traffic that exceeds the rate limit in a queue, sending it later when the link is free.
  • Effect: Creates a smooth, consistent output graph.
  • Direction: Applied Outbound ONLY (you can't buffer what you haven't received yet).
  • Use Case: Ensure your outbound traffic doesn't overwhelm an ISP's strict policer.

🗂️ Queuing Algorithms

How does a router empty its queues during congestion?

CBWFQ

Class-Based Weighted Fair Queuing. You create different classes (queues) for different traffic and guarantee them a minimum percentage of bandwidth. However, it does not guarantee low delay.

LLQ (Low Latency Queuing)

The absolute industry standard for modern QoS. It is simply CBWFQ with one addition: a Strict Priority Queue (PQ). Traffic in the PQ (Voice) is ALWAYS sent first before any other queue gets a turn.

*To prevent Voice traffic from starving all other data traffic, the Strict Priority Queue has an inherent built-in Policer to cap its maximum rate.

📉 Congestion Avoidance (WRED)

The Problem: Tail Drop

If a queue reaches 100% capacity, every new packet that arrives is dropped. This is called Tail Drop. If 50 TCP sessions all experience Tail Drop simultaneously, they all drastically reduce their window size at the exact same time. This is called TCP Global Synchronization, and it kills network throughput.

The Solution: WRED

Weighted Random Early Detection. Before the queue fills up, WRED starts randomly dropping a few packets. This forces just a few TCP sessions to slow down early, preventing the queue from ever hitting 100% and avoiding Global Synchronization.

"Weighted" means it drops lower priority packets (like Best Effort) before it drops higher priority packets (like AF41).

🪝 Brain Hooks

The Highway Tollbooth

Policing: Speed camera. Go too fast, get a ticket (dropped).
Shaping: Traffic lights on a freeway on-ramp. Delays you in a queue to smooth out traffic flow onto the highway.

LLQ = The Ambulance

The Priority Queue (Voice) is an ambulance. Everyone else pulls over and waits until it passes. But if it goes rogue, the police pull it over (implicit policing) so it doesn't starve the normal traffic.

⌨️ Modular QoS CLI (MQC) Framework

Cisco configures QoS using a very specific 3-step framework known as MQC.

Step 1: Class-Map (The WHO) Router(config)# class-map match-all VOICE_TRAFFIC Router(config-cmap)# match dscp ef Identifies the traffic you want to manipulate. Uses ACLs, DSCP values, or protocols.
Step 2: Policy-Map (The WHAT) Router(config)# policy-map MY_QOS_POLICY Router(config-pmap)# class VOICE_TRAFFIC Router(config-pmap-c)# priority percent 10 Calls the class-map and applies an action (e.g., shaping, policing, or placing it in the Strict Priority queue via the `priority` keyword).
Step 3: Service-Policy (The WHERE) Router(config)# interface g0/1 Router(config-if)# service-policy output MY_QOS_POLICY Applies the completed policy to an interface in a specific direction (input/output).

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. What is the fundamental difference in how Traffic Shaping and Traffic Policing handle excess packets? ▼
Traffic Shaping buffers (delays) excess traffic to smooth the rate. Traffic Policing immediately drops (or remarks) excess traffic.
2. Which queuing algorithm is the standard for Voice traffic because it provides a Strict Priority Queue? ▼
LLQ (Low Latency Queuing).
3. What congestion avoidance mechanism prevents TCP Global Synchronization by randomly dropping packets before the queue fills up? ▼
WRED (Weighted Random Early Detection).
4. What are the three configuration steps in the Cisco MQC (Modular QoS CLI) framework? ▼
1. Class-map (match the traffic), 2. Policy-map (define the action), 3. Service-policy (apply to interface).
