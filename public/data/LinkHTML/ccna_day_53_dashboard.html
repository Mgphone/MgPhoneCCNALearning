Day 53: WAN Architectures

MPLS, Metro Ethernet, VPNs & SD-WAN

🌍

The WAN Evolution

A Wide Area Network (WAN) connects LANs separated by large geographic distances. You lease this connection from a Service Provider (SP).

Legacy: Leased Lines (T1/T3)

Dedicated, point-to-point copper circuits. Highly secure, guaranteed bandwidth, but incredibly expensive and slow (a T1 is only 1.544 Mbps). Required special serial interfaces.

Modern: Metro Ethernet

The SP provides a standard Ethernet handoff (RJ45 or Fiber SFP) to the customer. It's cheap, fast (1 Gbps - 100 Gbps), and easily integrates with the customer's existing LAN equipment.

MPLS (Multiprotocol Label Switching)

A highly scalable routing technique used internally by SPs to direct traffic quickly using short path labels rather than complex IP routing table lookups.

🏷️

MPLS Architecture & Roles

MPLS creates private, secure virtual networks for customers over a shared Service Provider infrastructure (Layer 3 VPNs). You must know the router roles.

CE (Customer Edge)

The router sitting at the customer's site. It connects to the SP.

  • Runs standard IP routing (OSPF, EIGRP).
  • Unaware of MPLS. It does not run MPLS or see labels.
CRITICAL

PE (Provider Edge)

The SP's router that connects to the CE.

  • The workhorse of MPLS.
  • Takes incoming IP packets and adds (pushes) an MPLS label.
  • Removes (pops) labels before sending back to the CE.

P (Provider Core)

The routers buried deep inside the SP's network.

  • They do NOT look at destination IP addresses.
  • They only look at the MPLS label, swap it for a new one, and forward it at hardware speeds.

🔒 Internet-Based VPNs

MPLS is expensive. The Internet is cheap. To use the public internet as a WAN, you must create a Virtual Private Network (VPN) to secure the data.

IPsec Site-to-Site VPN

Connects two fixed locations (e.g., HQ to Branch). Encrypts data to provide Confidentiality, Integrity, and Authentication over the public web.

DMVPN (Dynamic Multipoint VPN)

Cisco solution. Allows branch routers to dynamically build direct VPN tunnels to each other on-demand (Spoke-to-Spoke), reducing traffic load on the HQ Hub router.

PPPoE (PPP over Ethernet)

Used on DSL/Fiber internet links. Allows the ISP to enforce Authentication (using PAP/CHAP usernames and passwords) over a standard Ethernet connection.

☁️

Cisco SD-WAN Architecture

Software-Defined WAN decouples the control plane from the data plane. Routers no longer make their own routing decisions; they take orders from a centralized controller.

vManage (Management Plane)

The centralized GUI dashboard. This is where network admins log in to design policies, deploy configurations, and monitor the entire network.

vSmart (Control Plane)

The "brains" of the operation. Calculates routing topologies and distributes routing information and data-plane policies to the edge routers (using OMP protocol).

vBond (Orchestration Plane)

The gatekeeper. Authenticates new edge routers attempting to join the network and tells them exactly how to find the vManage and vSmart controllers. (Acts like a STUN server for NAT traversal).

vEdge / cEdge (Data Plane)

The physical (or virtual) routers sitting at the remote branches. They actually forward the user traffic based on the rules handed down by vSmart.

🪢 Underlay vs. Overlay

The Underlay

The physical network infrastructure that provides basic IP connectivity. (e.g., The physical cables, the raw ISP Internet connection, or the raw MPLS circuit).

The Overlay

The logical, virtual network built on top of the Underlay using tunnels. (e.g., The IPsec VPN tunnels, or the SD-WAN fabric). The overlay hides the complexity of the underlay.

🪝 Brain Hooks & Analogies

MPLS Labels = ZIP Codes

When mailing a package, postal sorting facilities (P Routers) don't read the exact street address (IP Address). They just look at the ZIP Code (MPLS Label) to quickly toss it into the correct bin for the next city.

SD-WAN Naming Cheat Sheet

Manage = Management (GUI)
Smart = Control Plane (The BrainS)
Bond = Orchestration (Brings them together)

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. In an MPLS network, which specific router is responsible for taking incoming customer IP packets and adding (pushing) the initial MPLS label onto them? ▼
The PE (Provider Edge) router. The Customer Edge (CE) router is completely unaware of MPLS.
2. In the Cisco SD-WAN architecture, which component represents the Management Plane, providing a centralized GUI for admins to configure the network? ▼
vManage. (vSmart handles control, vBond handles orchestration).
3. What Cisco technology is used to allow branch routers to dynamically build temporary IPsec VPN tunnels directly to each other, rather than routing all traffic through the headquarters? ▼
DMVPN (Dynamic Multipoint VPN). It uses a hub-and-spoke setup to facilitate dynamic spoke-to-spoke tunnels.
4. When discussing SD-WAN or VPNs, what term describes the logical, virtual network (tunnels) built on top of the physical ISP connection? ▼
The Overlay network. (The physical ISP connection is the Underlay).
