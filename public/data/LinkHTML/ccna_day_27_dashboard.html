Day 27: OSPF (Pt 2)

Network Types, DR/BDR & States

🖧

OSPF Network Types

OSPF behaves differently depending on the type of physical link it's running over.

1. Broadcast

Default for Ethernet links (LANs). Assumes multiple routers could be connected to the same switch.

  • Multicasts Hello packets to 224.0.0.5.
  • Elects a DR and BDR.
  • Timers: Hello 10s, Dead 40s.

2. Point-to-Point (P2P)

Default for Serial links. Only two routers can possibly exist on this link.

  • Multicasts Hello packets to 224.0.0.5.
  • NO DR/BDR Election! (Not needed, only 2 routers).
  • Timers: Hello 10s, Dead 40s.
Manual Override If you have a direct Ethernet cable between two routers, OSPF still thinks it's a "Broadcast" network and wastes time electing a DR. You can manually configure ip ospf network point-to-point on the interface to speed up adjacency!
👑

The DR & BDR Election

On Broadcast networks, if 10 routers exchange full LSDBs with each other, it creates chaos ($n(n-1)/2$ adjacencies). OSPF elects a Designated Router (DR) and a Backup (BDR) to manage this.

How They Communicate

Regular routers (DROther) form a "FULL" adjacency ONLY with the DR and BDR. They stay in a "2-WAY" state with each other.

  • DROthers send updates to the DR/BDR using multicast 224.0.0.6.
  • The DR forwards that update to everyone else using multicast 224.0.0.5.

The Election Rules

Elected per-segment (per-subnet). Highest wins!

  1. Highest OSPF Interface Priority: Default is 1. (0 means "Never become DR/BDR").
  2. Highest Router ID (RID): Used as a tie-breaker.
  3. No Preemption: If a new router boots up with a better priority, it does NOT steal the DR role. It must wait until the current DR dies.

🔄 The 7 OSPF Neighbor States

When two OSPF routers connect, they progress through a strict series of states before they can route traffic.

1. Down

No OSPF hellos have been received from the neighbor. (Initial state).

2. Init

Router received a Hello packet, but its own Router ID is NOT listed in the packet yet. (1-way communication).

3. 2-Way

Bi-directional communication established! DR/BDR election happens here. DROthers stay in this state with each other.

4. ExStart

Routers negotiate who will be the "Master" and "Slave" for the upcoming database exchange (Highest RID is Master).

5. Exchange

Routers trade Database Description (DBD) packets—a summary "menu" of their LSDB.

6. Loading

Routers request specific full details for missing routes using Link-State Requests (LSR) and receive Link-State Updates (LSU).

7. FULL

Adjacency is complete! Both routers have identical, synchronized LSDBs. Normal routing occurs.

🤫 Passive Interfaces

If you use the network command for a subnet that connects to PCs, OSPF will continuously multicast Hello packets out of that switch port to the PCs.

The Security Risk

PCs don't speak OSPF. A hacker could plug in a rogue router, form an adjacency, and inject fake routes to intercept your traffic!

The Solution

Configure the interface as passive. The router will still advertise the subnet into OSPF, but it will stop sending Hellos out of that interface, preventing any adjacencies from forming there.

🪝 Brain Hooks

DR = The Class Representative

Instead of 30 students all shouting their updates to each other simultaneously (broadcast storm), they all whisper to the Class Rep (DR) on 224.0.0.6. The Rep then stands up and announces the update to the whole class on 224.0.0.5.

"Highest Wins" Election

Unlike Spanning Tree where the lowest MAC wins, in OSPF the Biggest, Baddest Router wins the DR election (Highest Priority, then Highest RID).

⌨️ Advanced OSPF CLI

Router(config-if)# ip ospf priority 200 Changes interface priority (0-255) to manipulate the DR election. A priority of 0 ensures it is NEVER elected.
Router(config-router)# passive-interface g0/1 Stops OSPF Hellos from leaving this specific interface.
Router(config-router)# passive-interface default Best Practice: Makes ALL interfaces passive by default. You then use `no passive-interface [int]` only on links connected to other routers.
Router# clear ip ospf process Forces OSPF to restart, breaking all adjacencies and forcing a new DR/BDR election. Required if you change the Router ID.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. True or False: A DR/BDR election happens on every single OSPF link, including serial Point-to-Point links. ▼
False. A DR/BDR election ONLY happens on Broadcast networks (like Ethernet). Point-to-Point networks skip this step because there are only ever two routers.
2. Which multicast IP address do normal OSPF routers (DROthers) use to send their updates directly to the DR and BDR? ▼
224.0.0.6. (The DR then multicasts the update to the rest of the network using 224.0.0.5).
3. In a Broadcast network, two DROther routers will stop the adjacency process and remain permanently in which OSPF state with each other? ▼
2-Way. They only form a FULL adjacency with the DR and the BDR.
4. Router A is the DR. Router B boots up with a much higher OSPF priority (255). Why doesn't Router B immediately become the DR? ▼
Because OSPF DR elections are non-preemptive. Once a DR is elected, it remains the DR until it fails or the OSPF process is manually reset, ensuring network stability.
