Classic STP States & Timers (802.1D)

Classic Spanning Tree is slow. It takes up to 50 seconds for a port to transition from Blocking to Forwarding if a link fails.

The Timers

Hello Timer (2s): How often BPDUs are sent.
Forward Delay (15s): Time spent in Listening, and again in Learning state (30s total).
Max Age (20s): How long a switch holds onto BPDU info before discarding it if it stops hearing from the Root.

The Port States

  • 1. Blocking: Receives BPDUs only. Drops data.
  • 2. Listening (15s): Sends/receives BPDUs to elect Root. Drops data. No MAC learning.
  • 3. Learning (15s): Sends/receives BPDUs. Begins populating MAC address table. Drops data.
  • 4. Forwarding: Normal operation. Forwards data.
⚡

RSTP (802.1w)

Rapid Spanning Tree Protocol drastically improves convergence times (almost immediate) by actively negotiating states rather than waiting on passive timers.

Simplified Port States

RSTP combines the old Disabled, Blocking, and Listening states into a single state:

  • Discarding
  • Learning
  • Forwarding

BPDU Handling

In classic STP, only the Root Bridge generates BPDUs. In RSTP, all switches generate BPDUs every 2 seconds as a keepalive mechanism.

🗂️ RSTP Port Roles

RSTP keeps Root Port and Designated Port, but expands on the "Blocking" role by giving it two specific backup roles.

Root Port (RP) & Designated Port (DP) Function exactly the same as in Classic 802.1D.
Alternate Port (ALT) A backup for the Root Port. If the RP fails, the Alternate port immediately transitions to forwarding. Discards data normally.
Backup Port (BKUP) A backup for a Designated Port. Only occurs if a switch connects to a hub (shared collision domain) with two of its own ports. Very rare today.

🛡️ Edge Ports (PortFast & BPDU Guard)

When you plug a PC into a switch, you don't want it to wait 30 seconds (Listening -> Learning) before it can request a DHCP address. It will time out!

PortFast (Edge Port)

Bypasses the Listening and Learning states. The port transitions immediately from Blocking to Forwarding.

WARNING:

Only configure PortFast on ports connected to END DEVICES (PCs, Servers). If you enable it on a switch-to-switch link, you risk causing a temporary broadcast storm!

BPDU Guard

The safety net for PortFast. If an end-user sneakily plugs a rogue switch into a wall jack (a PortFast port), that rogue switch might send a BPDU.

If BPDU Guard is enabled, the port will detect the BPDU and immediately shut down (err-disable) the port to protect the network from loops.

🪝 Brain Hooks

RSTP Roles: Alt vs Backup

Alternate = Alternative path to the Root.
Backup = Bad design (Hubs).

The Guard Dog

BPDU Guard is the guard dog. PortFast leaves the door wide open for fast entry. The Guard Dog sits at the door; if it sees another switch (a BPDU) trying to sneak in, it slams the door shut (err-disable).

⌨️ RSTP & PortFast Configuration

Switch(config)# spanning-tree mode rapid-pvst Changes the STP mode from classic PVST+ to Rapid PVST+ (RSTP). Essential command for modern networks!
Switch(config-if)# spanning-tree portfast Enables PortFast on a specific interface. (Bypasses Listening/Learning).
Switch(config-if)# spanning-tree bpduguard enable Enables BPDU Guard on a specific interface.
Switch(config)# spanning-tree portfast default Global command: Automatically enables PortFast on ALL interfaces configured as Access ports.
Switch(config)# spanning-tree portfast bpduguard default Global command: Automatically enables BPDU Guard on ALL PortFast-enabled interfaces.

🔧 Troubleshooting

PC takes 30s to get IP

Symptom: When a user plugs in a PC, the port light stays amber for 30 seconds before turning green.

Fix: You forgot to enable spanning-tree portfast on the interface. It is grinding through the classic Listening and Learning states.

Port in Err-Disable

Reason: If show interfaces status shows err-disabled, and it's a user port, they likely plugged in an unmanaged switch. BPDU guard tripped. Remove the rogue switch, then shut / no shut the port.
