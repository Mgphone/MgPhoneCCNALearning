Day 59: Network Automation

SDN, APIs, and Configuration Management

🏗️

The Scaling Problem

Historically, network engineers configured routers and switches "box-by-box" using SSH and the CLI. As networks grew to thousands of devices, this approach failed.

Configuration Drift

When humans manually type commands into 500 switches, typos happen. Over time, no two switches have the exact same configuration, making troubleshooting a nightmare.

Slow Provisioning

Deploying a new enterprise-wide QoS policy or updating an SNMP string takes weeks of manual labor and maintenance windows.

The Solution: SDN

Software-Defined Networking (SDN) centralizes management, moving away from distributed, manual configurations to programmable, automated controller-based networks.

🧠

Control Plane vs. Data Plane

To understand SDN, you must understand how a router's brain is separated from its brawn.

Control Plane (The Brain)

The logic that determines how traffic should be routed.

  • Runs routing protocols (OSPF, EIGRP, BGP).
  • Builds the Routing Table and ARP Table.
  • Historically processed by the CPU on every single router.
  • In SDN: This plane is extracted from the router and moved to a centralized Controller.

Data Plane (The Brawn)

The physical act of moving packets from one port to another.

  • Forwarding IP packets and switching Ethernet frames.
  • Applying Access Control Lists (ACLs) and QoS.
  • Processed by specialized hardware (ASICs) for ultimate speed.
  • In SDN: The router keeps this plane but takes its forwarding orders from the Controller.

🔌 APIs in SDN

Application Programming Interfaces (APIs) allow software to talk to other software. In SDN, the centralized Controller sits in the middle and talks in two directions.

Northbound APIs

Communication flowing "UP" from the SDN Controller to Management Applications or Python Scripts.

Allows you to write code that extracts network data or pushes intents to the controller. Typically uses REST APIs (HTTP/HTTPS).

Southbound APIs

Communication flowing "DOWN" from the SDN Controller to the actual physical network devices (switches/routers).

Used to program the forwarding tables. Common protocols: NETCONF, RESTCONF, OpenFlow, CAPWAP.

📜 Data Serialization Formats

When APIs exchange data, they need a structured format that both humans and machines can read. CCNA tests three primary formats.

JSON

JavaScript Object Notation. Extremely popular for REST APIs.

{
  "interface": "Gigabit0/1",
  "status": "up",
  "vlans": [10, 20]
}

XML

eXtensible Markup Language. Older, used heavily by NETCONF.

<interface>
  <name>Gigabit0/1</name>
  <status>up</status>
</interface>

YAML

YAML Ain't Markup Language. Highly human-readable. Uses strict indentation.

interface: Gigabit0/1
status: up
vlans:
  - 10
  - 20

🛠️ Configuration Management Tools

These tools automate the deployment of configurations across hundreds of devices simultaneously to enforce consistency.

Feature Ansible Puppet Chef
Architecture Agentless (Uses SSH) Agent-based Agent-based
Mechanism Push Pull Pull
Language Python Ruby Ruby

* Note: Ansible is heavily favored in modern network automation because it does not require installing third-party agent software on Cisco routers.

🪝 Brain Hooks

"A" for Ansible

Ansible is Agentless.
(Puppet and Chef require you to install a software "Agent" on the target router).

North vs South

Think of a map. The Controller is in the middle.
North = Up to the Cloud/Apps/Scripts.
South = Down to the dirty hardware (Switches).

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. In a Software-Defined Network (SDN) architecture, which "plane" is extracted from the physical routers and centralized into a software controller? ▼
The Control Plane. (The routers keep the Data Plane to forward traffic, but receive instructions from the controller).
2. If you write a Python script that uses HTTP REST calls to query the SDN Controller for network health data, are you using a Northbound or Southbound API? ▼
A Northbound API. (Southbound APIs communicate down to the physical hardware).
3. Which of the three major configuration management tools (Ansible, Puppet, Chef) is "Agentless" and relies on standard SSH to push configurations? ▼
Ansible.
4. Which data serialization format omits brackets and quotation marks, relying instead on strict indentation and dashes to define its structure? ▼
YAML (YAML Ain't Markup Language).
