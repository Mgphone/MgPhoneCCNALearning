Day 37: NTP

Network Time Protocol & Clock Synchronization

⏱️

Why Network Time?

Network Time Protocol (NTP) ensures all devices on a network share the exact same time. It operates over UDP Port 123.

1. Syslog Correlation

If a network outage occurs, you must piece together logs from multiple routers and switches. If their clocks are off by even 30 seconds, figuring out exactly what broke first becomes impossible.

2. Cryptography & Certificates

Digital certificates (like those used for HTTPS, VPNs, and SSH) have strict valid dates. If a router thinks it's 1993, it will reject modern certificates as "not yet valid" and secure connections will fail.

3. Two Clocks

Cisco devices have two clocks: the Hardware Clock (Calendar/battery-backed) and the Software Clock (runs only while powered on). NTP updates the Software Clock by default.

📶

Stratum Levels

NTP uses a hierarchical system called "Stratum" to measure the distance (in hops) from the highly accurate reference clock.

Stratum 0 Highly precise reference clocks (Atomic clocks, GPS). Cannot be connected to directly over a network.
Stratum 1 Primary time servers. Devices physically attached to a Stratum 0 clock.
Stratum 2 Secondary servers. Devices that sync their time over the network from a Stratum 1 server.

Stratum 16 = Unsynchronized

The maximum valid stratum is 15. If a device shows a stratum level of 16, it means the clock is unsynchronized and unreliable. It will not be trusted by other NTP clients.

🏛️ NTP Architecture Roles

Devices can participate in NTP in different ways depending on their configuration.

  • Client Mode

    Polls an NTP server for time. It updates its own clock but does not provide time to other devices.

  • Server Mode

    Provides time to clients. It can also act as a client itself by syncing to a higher-stratum server (e.g., getting time from the internet, then serving it to the local LAN).

  • Symmetric Active (Peer)

    Two routers are configured to peer with each other. If one loses its connection to the internet time server, it can temporarily sync its clock from its peer.

🔐 NTP Authentication

Attackers can spoof NTP packets, sending the network back in time to invalidate certificates or bypass time-based Access Control Lists. NTP Authentication prevents this.

How it Works

Instead of blindly trusting any NTP packet, the router uses an MD5 hash and a pre-shared key. The client hashes the packet using the key, and the server verifies it.

Note: The timestamp itself isn't encrypted, but the packet is authenticated so the receiver knows the source is legitimate.

Auth Configuration

ntp authenticate ntp authentication-key 1 md5 CISCO123 ntp trusted-key 1 ntp server 10.1.1.1 key 1

🪝 Brain Hooks

"Sixteen is Sunk"

When looking at Stratum levels, remember 16 means the ship is sunk. The clock is unsynchronized and cannot be trusted.

UDP 123

NTP uses UDP Port 123. Just remember the clock counts up: 1, 2, 3!

⌨️ Configuration & Verification CLI

Router(config)# ntp server 8.8.8.8 Configures the router as an NTP client to sync its clock to the specified IP address.
Router(config)# ntp master 5 Forces the router to act as an authoritative NTP server for the network, claiming to be Stratum 5. (Useful if the router loses internet connectivity but still needs to provide a unified time to the internal LAN).
Router# show ntp status Crucial! The first line will say Clock is synchronized or Clock is unsynchronized. Also displays the local Stratum and reference IP.
Router# show ntp associations Displays a table of all configured NTP servers/peers. Shows their Stratum, Delay, Offset (how far off your clock is), and Jitter. An asterisk (*) indicates the server you are currently synced to.

🔧 Troubleshooting

"Clock is unsynchronized"

Action 1: Be patient! NTP is a slow protocol. It can take 5-10 minutes for a device to fully synchronize after a reboot.

Action 2: Check ACLs and firewalls. Are you blocking UDP Port 123 outbound?

Stratum 16 in Associations

Issue: You run `show ntp associations` and the server you pointed to shows a stratum of 16.

Fix: The upstream server itself has lost its time source and is currently dead/unreliable. Point to a different server.

4. If you enter the global configuration command `ntp master 4`, what specific role does the router assume? ▼
It makes the router act as an authoritative NTP Server at Stratum level 4, allowing it to provide time to downstream clients even if it loses its own internet connection.
