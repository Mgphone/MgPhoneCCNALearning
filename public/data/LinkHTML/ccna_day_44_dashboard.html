Day 44: NAT (Pt 1)

Network Address Translation & Static NAT

🎭

Why NAT?

We ran out of IPv4 addresses years ago. To fix this, RFC 1918 defined Private IP Addresses (10.x, 172.16.x, 192.168.x). These addresses are free to use inside your LAN, but they are completely unroutable on the public internet.

The Translation Process

When a PC with a Private IP tries to reach Google, the edge router (NAT router) intercepts the packet. It strips the Private Source IP, replaces it with a Public Source IP, and sends it to the internet.

The Translation Table

The router remembers this swap in its NAT Table. When Google replies, the router intercepts the packet again, translates the Destination IP back to the Private IP, and sends it to the PC.

Security Byproduct NAT inherently hides your internal network topology from the outside world. An attacker on the internet only sees your one public IP, not the 500 private PCs behind it.
📖

The 4 Cisco NAT Terms

You absolutely must memorize these terms for the CCNA. Cisco's terminology is very specific.

IL

1. Inside Local (IL)

The Private IP address assigned to a host on the inside network.

Ex: 192.168.1.50
IG

2. Inside Global (IG)

The Public IP address that represents your inside host to the outside world.

Ex: 203.0.113.5
OG

3. Outside Global (OG)

The actual Public IP address of the remote host (e.g., Google's web server) on the internet.

Ex: 8.8.8.8
OL

4. Outside Local (OL)

How the remote host appears to the inside network. Usually, it is identical to the Outside Global address.

Ex: 8.8.8.8

🔒 Static NAT

Static NAT creates a permanent, 1-to-1 mapping between a private IP (Inside Local) and a public IP (Inside Global).

  • The Use Case: Servers

    Dynamic NAT is fine for outbound browsing. But if you are hosting a Web Server or Email Server locally, people on the internet need a predictable public IP to initiate inbound connections to it.

  • Permanent Table Entry

    Because it's a permanent mapping, the entry never times out of the NAT table. It wastes one public IP exclusively for one internal device.

🪝 Brain Hooks

Translating the Terms

Inside = "My stuff" (My PC, My Router).
Outside = "Their stuff" (Google, Internet).

Local = "Private IP" (Viewed from the inside).
Global = "Public IP" (Viewed from the outside).

The PBX Analogy

An Inside Local IP is like a corporate extension (x405). An Inside Global IP is the company's direct 1-800 phone number that outsiders actually dial.

⌨️ Static NAT Configuration CLI

Step 1: Create the Translation Mapping Router(config)# ip nat inside source static 192.168.1.100 203.0.113.10 Translates the Inside Local IP (192.168.1.100) to the Inside Global IP (203.0.113.10).
Step 2: Assign Interfaces (Crucial!) Router(config)# interface gigabitethernet0/0 Router(config-if)# ip nat inside Identifies the port facing your private LAN. Router(config)# interface gigabitethernet0/1 Router(config-if)# ip nat outside Identifies the port facing the Internet/WAN.
Verification Router# show ip nat translations Displays the active NAT table. For static NAT, this mapping will be visible immediately and permanently.

🔧 Troubleshooting

Table is Empty / Routing Fails

Symptom: You created the `ip nat inside source static` command, but traffic from the server drops at the router.

Fix: The #1 mistake is forgetting to apply ip nat inside and ip nat outside on the respective physical interfaces! Without both, the router will not trigger the translation process.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Static NAT relies on what kind of mapping ratio between Private and Public IPs? ▼
It creates a permanent 1-to-1 mapping (One private IP corresponds exactly to one public IP).
2. In Cisco terminology, which term describes the Public IP address assigned to a device sitting on your internal network? ▼
Inside Global. ("Inside" means it's your device; "Global" means it's the public IP viewed by the internet).
3. Which CLI command is used to view the table mapping Private IPs to Public IPs? ▼
show ip nat translations
4. What is the primary real-world use case for configuring Static NAT instead of Dynamic NAT? ▼
Providing internet users consistent inbound access to a local server (such as a Web Server or Email server).
