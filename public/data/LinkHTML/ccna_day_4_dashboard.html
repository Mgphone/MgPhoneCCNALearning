Cisco IOS Modes

The CLI is hierarchical. You must enter higher-privileged modes to make changes.

1. User EXEC Mode Router>

Default mode. View-only (limited `show` commands, `ping`). Cannot make any changes.

2. Privileged EXEC Mode Router#

"Enable" mode. Full access to all `show` commands. Save/erase configurations.

3. Global Configuration Mode Router(config)#

Make device-wide changes (hostnames, banners, passwords).

4. Sub-Configuration Mode Router(config-if)#

Configure specific parts of the device (interfaces, line vty, routing protocols).

🔌

Access Methods

  • Console Port: Used for initial configuration. Requires physical access and a console cable (rollover cable). This is known as Out-of-Band management.
  • SSH (Secure Shell): Secure, encrypted remote access over the network. This is In-Band management. The modern standard.
  • Telnet: Older remote access method over the network. Sends data in clear text (unencrypted). Never use in production.
💾

Memory Types

  • RAM: Volatile. Stores the running-config. Changes take effect immediately but are lost on reboot.
  • NVRAM: Non-volatile. Stores the startup-config. Loaded into RAM when the device boots.
  • Flash: Stores the Cisco IOS software image (the OS itself). Non-volatile.
  • ROM: Read-Only. Stores the bootstrap program (loads the IOS on startup).

🧠 Shortcuts & Brain Hooks

? (Context-Sensitive Help)
Type `?` at any time to see available commands. `cl?` shows all commands starting with 'cl'. `clock ?` shows what comes next.
Tab (Auto-Complete)
Type partial command and press Tab. E.g., `con[Tab]` completes to `configure`. If it doesn't complete, it's ambiguous.
Cursor Navigation
Ctrl + A: Move to start of line.
Ctrl + E: Move to end of line.
The "Panic Button"
Ctrl + Shift + 6: Aborts annoying DNS lookups, pings, or traceroutes. (Highly tested concept!)

⌨️ Core CLI Navigation Commands

Router> enable Moves from User EXEC to Privileged EXEC mode. (Short: `en`)
Router# disable Moves from Privileged EXEC down to User EXEC mode.
Router# configure terminal Enters Global Configuration mode. (Short: `conf t`)
Router(config)# exit Moves down one level (e.g., config to Priv EXEC).
Router(config-if)# end Drops you all the way back to Privileged EXEC mode. (Shortcut: `Ctrl+Z`)
Router# copy running-config startup-config Saves RAM config to NVRAM so it survives a reboot. (Old way: `write memory` or `wr`)

🔧 Troubleshooting the CLI

Translating "xxxxx"... domain server

Issue: You typed a typo (like `enabel`). The router thinks it's a hostname and tries to resolve it via DNS, freezing your console.

Fix: Press Ctrl + Shift + 6 to cancel it. To prevent it permanently, use no ip domain-lookup in global config.

% Incomplete command.

Issue: You pressed enter before giving all required parameters.

Fix: Press Up arrow to retrieve the command, press Space, and type ? to see what is missing.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Which mode are you in, and what does the prompt look like, immediately after typing 'enable'? ▼
You enter Privileged EXEC mode. The prompt changes to end with a hash/pound sign: Router#
2. You just configured a hostname. If you reboot the router now, the configuration will be lost. What command prevents this? ▼
copy running-config startup-config (This saves the volatile RAM config to the non-volatile NVRAM).
3. You are taking a brand new Cisco router out of the box. Which access method MUST you use to configure it for the first time? ▼
The Console port. You must use a console cable because the router has no IP addresses assigned yet to support SSH or Telnet (Out-of-band management).
4. You mistakenly typed a bad command and the CLI is stuck on "Translating... domain server". What shortcut kills this process? ▼
Ctrl + Shift + 6. This is the Cisco IOS abort sequence.
