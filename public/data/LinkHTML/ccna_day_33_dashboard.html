Day 33: IPv6 (Pt 3)

Static Routing, Multicast & Transition Mechanisms

🛣️

IPv6 Static Routing

The logic is identical to IPv4, but the syntax uses `ipv6 route` instead of `ip route`. You still have three ways to define the forwarding path.

1. Directly Attached (Exit Interface)

Points to the local exit interface. Best used only on Point-to-Point links.

ipv6 route 2001:db8:acad:2::/64 serial0/0/0

2. Next-Hop IP

Points to the neighbor's Global Unicast Address. Requires a recursive lookup.

ipv6 route 2001:db8:acad:2::/64 2001:db8:acad:1::2

3. Fully Specified (The Link-Local Rule)

If you use a Link-Local Address (LLA) as the next-hop, you MUST provide the exit interface as well, because LLAs are not globally unique and the router won't know which port to send it out of!

ipv6 route 2001:db8:acad:2::/64 g0/0 fe80::2
🌐

IPv6 Default Route

In IPv4, the default route is `0.0.0.0 0.0.0.0`. In IPv6, it is vastly simplified.

::/0

  • Matches ALL IPv6 addresses.
  • Operates exactly like the Gateway of Last Resort.
  • Used when a destination is not explicitly found in the routing table.
ipv6 route ::/0 g0/0

🎯 Solicited-Node Multicast

Since IPv6 has no broadcasts, how does NDP (Neighbor Discovery Protocol) ask for a MAC address without interrupting every PC on the network?

It uses a Solicited-Node Multicast address. Every time an IPv6 interface is assigned a Unicast address, it automatically joins a specific multicast group.

ff02::1:ffxx:xxxx

The address always starts with ff02::1:ff and appends the last 24 bits of the device's Unicast IPv6 address.

Result: An NDP NS message is only processed by the exact device it's intended for, saving massive CPU cycles across the network.

🌉 IPv4 to IPv6 Transition Mechanisms

We cannot turn off IPv4 overnight. The internet is in a decades-long transition phase. Cisco outlines three primary methods for networks to handle both protocols simultaneously.

1. Dual Stack

The device runs both IPv4 and IPv6 protocol stacks simultaneously.

  • Has both an IPv4 and IPv6 address.
  • Can talk natively to IPv4 hosts and IPv6 hosts.
  • Preferred migration method.

2. Tunneling

Used to send IPv6 traffic over an IPv4-only network.

  • Encapsulates the IPv6 packet inside an IPv4 packet.
  • Requires a tunnel interface at both ends to pack and unpack the data.

3. Translation (NAT64)

Allows an IPv6-only device to communicate with an IPv4-only device.

  • A router sitting in the middle literally translates the IPv6 header into an IPv4 header (and vice versa).
  • Similar to how NAT overload works today.

🪝 Brain Hooks

Dual Stack = Bilingual

A Dual Stack router is a person who speaks both English and Spanish perfectly. If spoken to in English (IPv4), they reply in English. If Spanish (IPv6), they reply in Spanish.

::/0 The Black Hole

Visually, ::/0 looks like a gaping hole. It's the gateway of last resort, where all unknown packets are tossed.

⌨️ IPv6 Routing CLI

Router(config)# ipv6 unicast-routing MANDATORY: Enables IPv6 routing globally. Without this, none of your `ipv6 route` commands will actually forward traffic.
Router# show ipv6 route Displays the IPv6 routing table. Look for 'S' for Static, 'C' for Connected, and 'L' for Local (/128 routes to the router's own IP).
Router# show ipv6 route static Filters the routing table to show only manually configured static routes.
Router# ping ipv6 2001:db8:acad:1::10 Tests Layer 3 connectivity specifically over the IPv6 stack.

🔧 Troubleshooting

Static Route Fails to Install

Symptom: You type ipv6 route ... fe80::1 but it doesn't show in the routing table.

Fix: You used a Link-Local Address (fe80) as the next-hop but didn't specify the exit interface. Delete the route and re-enter it as a Fully Specified route (e.g., ipv6 route ... g0/0 fe80::1).
