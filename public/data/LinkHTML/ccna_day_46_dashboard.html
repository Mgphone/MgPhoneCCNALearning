Day 46: QoS (Pt 1)

Quality of Service & Voice VLANs

🚦

Why do we need QoS?

When network bandwidth is abundant, everything works fine. But when a link gets congested, packets line up in a queue. Without QoS, it's First-In, First-Out (FIFO). A critical voice packet might get stuck behind a massive file download.

Data Traffic (TCP)

Forgiving. If a packet drops, TCP retransmits it. It doesn't matter if an email takes 1 second or 3 seconds to arrive.
Requirements: Low loss, delay is acceptable.

Voice Traffic (UDP)

Unforgiving. If a voice packet is dropped, it's gone. If it's delayed, the conversation overlaps and echoes. It MUST be placed at the very front of the line.
Requirements: Strict limits on delay, jitter, and loss.

Video Traffic (UDP)

Similar to voice but requires significantly more bandwidth. Bursty traffic.
Requirements: High bandwidth, strict limits on delay and loss.

🎙️

Voice Traffic Strict Constraints

You must memorize these three specific metrics for the CCNA. Voice is highly sensitive to network congestion.

1. One-Way Delay (Latency)

≤ 150 ms

The time it takes a packet to travel from speaker to listener. Above 150ms, people start talking over each other.

2. Jitter

≤ 30 ms

The variation in delay. If packet 1 takes 50ms, and packet 2 takes 120ms, the jitter is 70ms. Causes choppy, robotic audio.

3. Packet Loss

≤ 1%

Since Voice uses UDP (RTP), lost packets are not retransmitted. High loss results in missing syllables or dropped calls.

☎️ Voice VLAN Mechanics

To save on cabling, PCs are usually plugged into the back of the IP Phone, and the IP Phone plugs into the wall jack (Switch).

  • The "Mini-Trunk"

    Even though the switchport is configured as an access port, it acts like a miniature trunk, carrying two VLANs.

  • Untagged Data

    The PC sends raw Ethernet frames. The phone acts as a hub for these, passing them straight to the switch. The switch places this untagged traffic into the configured Access (Data) VLAN.

  • Tagged Voice

    The switch uses CDP (Cisco Discovery Protocol) to tell the phone which VLAN ID to use for voice. The phone then applies an 802.1Q tag to its own voice packets.

🏷️ QoS Markings (Classification)

To prioritize traffic, we must first "mark" or "color" the packets so routers and switches know what they are dealing with.

Layer 2: Class of Service (CoS)

Only exists inside the 802.1Q VLAN Tag. If a frame has no VLAN tag, it has no CoS! It uses 3 bits, giving 8 values (0-7).

  • CoS 0: Best Effort (Data)
  • CoS 5: Voice Payload
  • CoS 6/7: Network Control (STP, Routing)

Layer 3: DSCP

Differentiated Services Code Point. Exists inside the IPv4/IPv6 Header. It travels end-to-end across routers. Uses 6 bits, giving 64 values.

  • DF: Default (Best Effort)
  • AF: Assured Forwarding (Video/Important Data)
  • EF (46): Expedited Forwarding (Voice)

🪝 Brain Hooks

The Theme Park Analogy

Classification/Marking: Giving a VIP guest a fast-pass wristband at the front gate.
Queuing: When the roller coaster gets busy, the employee sees the wristband and puts them in the shorter, faster line.

Trust Boundaries

Never trust a PC's markings. Users can easily download software to tag their BitTorrent traffic as "Voice" to get priority. We establish a Trust Boundary at the IP Phone or the Switch Port.

⌨️ Voice VLAN Configuration CLI

Step 1: Standard Access Port Setup Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10 Assigns the untagged PC data traffic to VLAN 10.
Step 2: Voice VLAN Assignment Switch(config-if)# switchport voice vlan 20 Uses CDP to instruct the attached IP phone to tag its voice packets with VLAN 20 and CoS 5.
Step 3: Establish the Trust Boundary (Optional but recommended) Switch(config-if)# mls qos trust device cisco-phone Tells the switch port to trust the QoS markings (CoS 5) coming from the Cisco phone, but to overwrite/un-trust any markings coming from the PC attached behind the phone.
Verification Switch# show interfaces f0/1 switchport Look for "Access Mode VLAN: 10" and "Voice VLAN: 20".

🔧 Troubleshooting

Phone has no dial tone / no IP

Action 1: Is CDP disabled? (`no cdp run`). The switch relies on CDP to negotiate the Voice VLAN ID with the phone.

Action 2: Is there a DHCP server configured specifically for the Voice VLAN subnet?

Portfast Warning

Always configure spanning-tree portfast on voice VLAN ports so the phone boots up immediately without waiting 30 seconds for STP timers.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. For acceptable voice quality over an IP network, one-way delay (latency) should not exceed what specific millisecond value? ▼
150 ms (milliseconds).
2. Which protocol does a Cisco switch use to dynamically communicate the Voice VLAN ID to a connected Cisco IP Phone? ▼
CDP (Cisco Discovery Protocol).
3. In the Layer 3 IPv4 header, what specific DSCP value is universally recommended for marking Voice payload traffic? ▼
EF (Expedited Forwarding), which equates to a decimal value of 46.
4. Why is it impossible for a standard untagged data frame from a PC to contain a Layer 2 CoS (Class of Service) marking? ▼
Because the CoS field (3 bits) only exists inside the 802.1Q VLAN tag. Untagged frames do not have an 802.1Q header, so they cannot carry Layer 2 QoS markings.
