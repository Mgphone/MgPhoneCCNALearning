Day 54: Virtualization & Cloud

Hypervisors, Containers, and Cloud Models

💻

The Shift to Virtualization

Historically, a company bought one physical server for one specific application (e.g., an Email server). This was incredibly inefficient; most servers sat at 10% CPU utilization while consuming maximum power and space.

Traditional Architecture

Hardware -> Single Operating System -> Single Application.
Result: High cost, massive physical footprint, resource waste.

Server Virtualization

Hardware -> Hypervisor -> Multiple Virtual Machines (VMs).
Result: One physical server acts as 20 virtual servers. CPU, RAM, and Storage are dynamically pooled and shared, driving utilization to 80%+.

The Magic of Abstraction Virtualization abstracts (separates) the software from the underlying hardware. You can seamlessly move a running VM from a Dell server to an HP server with zero downtime.
🧠

Hypervisors (VMM)

The Virtual Machine Monitor (Hypervisor) is the software that creates and manages VMs. You must know the difference between the two types.

Type 1: Bare Metal

The hypervisor is installed directly onto the physical hardware. There is no traditional "Host OS" in the way.

  • Maximum performance, efficiency, and security.
  • Used exclusively in Enterprise Data Centers.
  • Examples: VMware ESXi, Microsoft Hyper-V, KVM.

Type 2: Hosted

The hypervisor is installed as an application on top of an existing Host OS (like Windows 11 or macOS).

  • Slower performance (the Host OS steals resources).
  • Used for testing, labs, and end-user desktops.
  • Examples: VMware Workstation, Oracle VirtualBox.

📦 VMs vs. Containers

Containers are the modern evolution of virtualization, heavily tested on the new CCNA.

Virtual Machines (VMs)

Every VM requires its own complete Guest Operating System (e.g., a full 20GB Windows Server install). Booting up takes minutes. Highly isolated, but resource-heavy.

Containers (e.g., Docker)

Containers share the Host OS Kernel. They only package the app and its required bins/libs (often just megabytes in size). They boot in milliseconds. You can run 100 containers on the same hardware that could only support 5 VMs.

☁️ Cloud Service Models

Cloud computing shifts Capital Expenditure (buying hardware) to Operational Expenditure (renting resources). You must know who manages what.

IaaS

Infrastructure as a Service

Provider gives you raw virtual hardware (CPU/RAM). You manage the OS, Apps, and Data.

Ex: AWS EC2, Azure VMs

PaaS

Platform as a Service

Provider manages hardware AND the OS framework. You just upload your App and Data.

Ex: Heroku, AWS Beanstalk

SaaS

Software as a Service

Provider manages everything. You just log in and consume the software.

Ex: Office 365, Salesforce

🏗️ Cloud Deployment Models

Public Cloud

Infrastructure is owned by a third-party provider (AWS, Google, Microsoft) and accessed over the internet. Resources are shared among millions of customers (Multi-tenant).

Private Cloud

Infrastructure is operated exclusively for a single organization (Single-tenant). Often hosted on-premises, providing maximum security and control.

Hybrid Cloud

A combination of Public and Private clouds bound together by proprietary technology or VPNs. (e.g., Keeping sensitive database on-prem Private, but hosting the web front-end on AWS Public).

Community Cloud

Infrastructure shared by several organizations with shared concerns (e.g., specific compliance, security, or jurisdiction constraints like a group of local hospitals).

🪝 Brain Hooks

VMs vs. Containers

VMs are like buying separate Houses. Each has its own plumbing, foundation, and roof (Guest OS).

Containers are like renting Apartments. They all share the same building foundation and plumbing (Shared Kernel), but have separate locked doors.

Type 1 vs Type 2

Type 1 (Bare Metal) = "Number 1" choice for enterprise performance.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. VMware ESXi is installed directly onto a physical server's hardware without an underlying Windows or Linux host OS. What type of hypervisor is this? ▼
A Type 1 (Bare Metal) Hypervisor.
2. What is the primary architectural difference that makes a Container (like Docker) much more lightweight and faster to boot than a traditional Virtual Machine (VM)? ▼
A Container shares the underlying Host OS Kernel and only packs the application bins/libraries. A VM requires a heavy, complete Guest Operating System to function.
3. If you rent a blank virtual server from AWS where you must manually install Linux, configure the firewall, and install your own application, which Cloud Service Model are you using? ▼
Infrastructure as a Service (IaaS). You are renting the raw infrastructure, but the OS and application layer management remains your responsibility.
4. A company keeps its highly confidential customer database on its own physical servers in its basement, but hosts its customer-facing website on Microsoft Azure. What deployment model is this? ▼
A Hybrid Cloud. It securely connects a Private Cloud (on-premise database) with a Public Cloud (Azure web servers).
