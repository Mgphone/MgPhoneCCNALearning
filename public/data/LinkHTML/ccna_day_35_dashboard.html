Day 35: Extended ACLs

Precision Traffic Filtering & Named ACLs

🔍

Extended Capabilities

While Standard ACLs only filter based on the Source IP, Extended ACLs act as a true stateless firewall, inspecting deep into Layer 3 and Layer 4 headers.

Filtering Parameters

  • Protocol: IP, TCP, UDP, ICMP, OSPF.
  • Source: IP Address & Wildcard Mask.
  • Destination: IP Address & Wildcard Mask.
  • Port (L4): Specific source or destination port (e.g., eq 80, eq 443).

Numbered Ranges

Cisco routers identify legacy numbered Extended ACLs using specific ID ranges:

  • 100 - 199
  • 2000 - 2699 (Expanded range)
📍

Placement Rule

Extended ACLs are surgically precise, which changes where we should place them compared to Standard ACLs.

Place Extended ACLs as close to the
Source as possible.

Why? Because an Extended ACL knows exactly where the packet is going and what port it's trying to hit, we can drop unwanted traffic immediately. Why let a packet travel across 5 routers and waste WAN bandwidth just to drop it at the destination?

🧩 Syntax Anatomy

Extended ACL statements must follow a strict, logical order: Action, Protocol, Source, Destination, Port.

access-list 100 permit tcp 192.168.1.0 0.0.0.255 host 10.0.0.5 eq 80
Action: permit or deny
Protocol: ip, tcp, udp, icmp
Source: IP & Mask (or host/any)
Dest: IP & Mask (or host/any)

L4 Operators (Used with TCP/UDP)

  • eq = Equal to
  • gt = Greater than
  • lt = Less than
  • neq = Not equal to
  • range = Port range

📛 Named ACLs

Numbers are hard to remember. Modern Cisco best practice uses Named ACLs.

Instead of a single line, it drops you into a sub-configuration mode. You can easily insert or delete individual lines using sequence numbers!

Router(config)# ip access-list extended BLOCK_WEB Router(config-ext-nacl)# deny tcp 10.1.1.0 0.0.0.255 any eq 80 Router(config-ext-nacl)# permit ip any any

🪝 Brain Hooks

"Permit IP ANY ANY"

In a Standard ACL, you write permit any to bypass the implicit deny. In an Extended ACL, you MUST specify the protocol. permit ip any any means "Permit the IP protocol, from any source, to any destination."

Source / Source / Dest

Standard ACLs check the Source, and go near the Destination.
Extended ACLs check everything, so put them near the Source to save bandwidth.

⌨️ Extended ACL CLI

Router(config)# access-list 101 deny icmp host 10.1.1.5 any echo Blocks PC 10.1.1.5 from pinging ANY destination. (Note: 'echo' is the specific ICMP type for a ping request).
Router(config)# ip access-list extended SERVER_DMZ Router(config-ext-nacl)# 5 deny tcp any host 192.168.50.5 eq 22 Creates a named ACL and uses a manual sequence number (5) to inject a rule denying SSH (port 22) to a specific server.
Router(config)# interface g0/1 Router(config-if)# ip access-group SERVER_DMZ in CRITICAL: Applies the named ACL to the interface in the inbound direction.
Router# show access-lists Displays the ACLs, their sequence numbers, and match counters.

🔧 Troubleshooting

"I blocked port 80, but I still can't browse the web!"

Symptom: You added an `eq 80` rule but DNS resolution is failing.

Fix: Remember the Implicit Deny! If you only permit TCP port 80, you are blocking UDP port 53 (DNS). A PC cannot load a webpage if it can't resolve the URL first. Ensure DNS is permitted.

The Sequence Issue

Fix: If you use numbered ACLs (e.g., 100) and type a new line, it goes to the bottom. If a `permit ip any any` is already above it, your new line will never be hit. Use Named ACLs to insert sequence numbers directly (e.g., `15 deny ...`).

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. According to Cisco best practices, where should an Extended ACL be placed in the network topology? ▼
As close to the Source as possible. (This drops unwanted traffic immediately, saving bandwidth across the rest of the network).
2. You write a specific 'deny' statement to block a PC from reaching a Web Server. What statement MUST you put at the bottom of the Extended ACL to allow all other traffic to pass? ▼
permit ip any any. (Unlike a Standard ACL which just needs `permit any`, an Extended ACL requires a protocol specification).
3. You see the command `access-list 101 permit tcp any any eq 443`. Just by looking at the number "101", is this a Standard or Extended ACL? ▼
It is an Extended ACL. (Standard ACLs use 1-99, while Extended use 100-199).
4. If you want to use an Extended ACL to block a host from "pinging" a server, which protocol keyword must you specify in the command? ▼
You must specify ICMP. (e.g., deny icmp host 10.1.1.5 host 192.168.2.2 echo). Ping does not use TCP or UDP.
