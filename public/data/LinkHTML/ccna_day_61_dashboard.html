What is REST?

An API (Application Programming Interface) allows two applications to talk to each other. REST (Representational State Transfer) is a specific architectural style for building network-based APIs using standard HTTP.

1. Client-Server Architecture

The Client (e.g., your Python script) sends a request. The Server (e.g., Cisco DNA Center) listens, processes the request, and sends a response. They evolve independently.

2. Statelessness (CRITICAL)

The server does NOT remember past requests. Every single request from the client must contain all the information (including authentication tokens) needed to process it. No "sessions" are stored on the server.

3. Cacheable

Responses must explicitly state whether the client is allowed to cache the data to improve performance and reduce server load.

HTTP Methods (CRUD Mapping)

REST relies on standard HTTP methods (verbs) to perform operations on resources. These directly map to traditional database CRUD operations (Create, Read, Update, Delete).

UPDATE

🧩 Anatomy of an API Request (URI)

A Uniform Resource Identifier (URI) tells the server exactly what you want.

https://api.cisco.com/v1/interfaces?status=up
Protocol / Scheme HTTP or HTTPS. Defines how the data is transferred and if it is encrypted.
Host / Domain The IP address or DNS name of the specific server hosting the API.
Resource Path The specific object you want to interact with (e.g., interfaces, devices, users). Often includes versioning (v1).
Query Parameters Starts with a ?. Used to filter, sort, or paginate data (e.g., only show interfaces that are UP). Additional params use &.

🚦 HTTP Status Codes

The server always returns a 3-digit status code.

2xx (Success)

200 OK: Request successful (Standard GET).
201 Created: Resource created (Standard POST).

4xx (Client Error)

400 Bad Req: Malformed syntax.
401 Unauthorized: Bad/missing token.
403 Forbidden: Token good, but lack permissions.
404 Not Found: Resource doesn't exist.

5xx (Server Error)

500 Internal Server Error: The API crashed.
503 Service Unavail: Server overloaded/down.

🔐 API Authentication

Because REST is stateless, every request must prove who you are.

Basic Auth

Sends the username and password encoded in Base64 in the HTTP Header. Highly insecure unless used strictly over HTTPS.

API Keys

A long string generated by the server given to the client. The client attaches this string to the URI or Header of every request.

Bearer Tokens

Client logs in once and receives a temporary Token (like a wristband). This token is placed in the Authorization: Bearer header. (Standard for OAuth/JWT).

🪝 Brain Hooks

Status Code Cheat Sheet

2xx = "Here you go."
3xx = "Go away, it's over there."
4xx = "You messed up."
5xx = "I messed up."

Stateless = Goldfish Memory

A REST API has the memory of a goldfish. You cannot say "Delete the router I just asked you about." You must specify exactly which router you want to delete in every single request.

🛠️ Interacting with REST APIs

Postman (GUI)

A popular graphical desktop application used to test API endpoints, format JSON payloads, and manage authorization tokens easily without writing code.

curl (CLI)

A command-line tool available on Linux/Mac to send HTTP requests.

curl -X GET "https://api.cisco.com/v1/devices" -H "Authorization: Bearer xyz123"

Python 'requests'

The standard Python library for calling APIs in automation scripts.

import requests
url = "https://api.example.com/data"
res = requests.get(url)

# Parse the JSON response
data = res.json()

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Which HTTP method is specifically mapped to the "Create" operation in the CRUD lifecycle? ▼
POST. You POST data to the server to create a new resource.
2. If you send an API request but forget to include your API Key in the header, what specific HTTP status code will the server likely return? ▼
401 Unauthorized. (A 403 Forbidden means you gave a valid key, but don't have permission to do what you asked).
3. Which fundamental constraint of REST architecture dictates that the server retains absolutely no memory of previous client requests? ▼
Statelessness. Every request must be independent and fully self-contained.
4. In the URI `https://api.cisco.com/v1/devices?type=router`, what is the exact term for the section after the question mark (`?type=router`)? ▼
A Query Parameter (or Query String). It is used to filter, sort, or modify the data returned by the API resource.
