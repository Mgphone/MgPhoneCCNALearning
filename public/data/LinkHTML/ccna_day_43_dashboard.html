Day 43: FTP & TFTP

File Transfer Protocols & Device Management

📂

File Transfer Basics

As a network admin, you frequently need to move files to and from Cisco routers and switches. These files are typically configuration backups or new IOS firmware images.

Configuration Backups

If a router dies, you don't want to retype 500 lines of CLI commands. You use FTP or TFTP to copy the running-config to an external server for safe keeping.

IOS Image Upgrades

To upgrade a switch's operating system, you download the .bin file from Cisco, host it on a local TFTP/FTP server, and tell the switch to pull the file into its Flash memory.

The Golden Command

In Cisco IOS, moving files is always done with the copy [source] [destination] command.

🤝

FTP (File Transfer Protocol)

FTP is robust, reliable, and uses authentication. It is the heavier of the two protocols.

The 2 Ports of FTP

FTP is unique because it uses TWO separate TCP ports simultaneously:

  • TCP Port 21 (Control): Used to establish the session, authenticate the user, and send commands (like `ls` or `get`).
  • TCP Port 20 (Data): The actual file payload is transferred over this port.

Key Characteristics

  • Reliable: Uses TCP. Dropped packets are retransmitted automatically.
  • Authentication: Requires a Username and Password (though it is sent in clear text).
  • Navigation: You can list, create, and delete directories on the remote server.

🚀 TFTP

Trivial File Transfer Protocol. It is stripped down, bare-bones, and designed to be as simple as possible.

UDP Port 69

TFTP uses UDP, meaning it is connectionless and unreliable. It relies on its own internal application-layer acknowledgments rather than TCP overhead.

No Security or Auth

TFTP has ZERO security. It does not ask for a username or password. If you can reach the server's IP, you can download or upload files.

No Directory Navigation

You cannot list the contents of a folder (`ls` or `dir`). You must know the exact name of the file you want to download beforehand.

⚖️ The Showdown

Feature FTP TFTP
Transport Protocol TCP (Reliable) UDP (Unreliable)
Port Number(s) 20 (Data), 21 (Control) 69
Authentication Yes (User/Pass) No (None)
Directory Nav Yes No
Primary Cisco Use Large file transfers, remote WANs LAN backups, device bootstrapping

🪝 Brain Hooks

"Trivial = UDP 69"

The 'T' in TFTP stands for Trivial. It doesn't care about security, it doesn't care about directory trees, it just wants to send data. Because it's trivial, it uses UDP (no handshakes).

FTP Port Mnemonic

FTP uses two ports: 20 and 21.
21 establishes the rules (Control), and then 20 does the heavy lifting (Data).

⌨️ IOS File Transfer CLI

The Golden Format: Router# copy [source] [destination]
Backing up config to a TFTP Server: Router# copy running-config tftp: The router will then prompt you for the server's IP address and the desired filename.
Restoring a config from a TFTP Server: Router# copy tftp: startup-config Downloads a file and places it directly into NVRAM for the next reboot.
FTP Authentication Config (Required before copy): Router(config)# ip ftp username admin Router(config)# ip ftp password cisco123 Router# copy flash:c2960.bin ftp: You must configure the FTP credentials in global config mode before the router can access the FTP server.

🔧 Troubleshooting

"Error opening tftp..."

Symptom: You execute the copy command, it stalls for 30 seconds, then fails.

Fix 1: Routing issue. Can the router ping the TFTP server IP?

Fix 2: ACL issue. Is a firewall blocking UDP Port 69?

Fix 3: Application issue. Is the TFTP server software actually running on the target PC?
