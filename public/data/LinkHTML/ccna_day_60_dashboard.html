Day 60: JSON, XML & YAML

Data Serialization Formats for Network Automation

📦

Data Serialization

When a Python script on your laptop needs to talk to a Cisco router, they can't just send raw memory objects to each other. They need a universal, standardized text format.

Serialization (Encoding)

Taking a complex data structure (like a Python dictionary containing interface IPs) and translating it into a standard text string (JSON, XML, YAML) to send over the network.

Deserialization (Decoding/Parsing)

Taking that received text string and converting it back into a native object that the local programming language or router OS can natively understand and manipulate.

The Basic Data Types
  • String: Text (e.g., "GigabitEthernet1")
  • Integer: Whole number (e.g., 1500)
  • Boolean: True or False
  • List/Array: An ordered collection of items
  • Dictionary/Object: A collection of Key:Value pairs
{ }

JSON (JavaScript Object Notation)

The absolute most popular format for modern web applications and REST APIs (like RESTCONF).

  • Syntax Rules: Relies heavily on brackets.
  • Objects (Dictionaries) are wrapped in curly braces { }.
  • Arrays (Lists) are wrapped in square brackets [ ].
  • Keys and string values MUST be in "double quotes".
  • Pairs are separated by a colon : and elements by a comma ,.
{
  "interface": {
    "name": "GigabitEthernet1",
    "enabled": true,
    "vlans": [10, 20, 30]
  }
}

<> XML

eXtensible Markup Language. Much older, highly verbose, but very strictly structured. The primary format used by NETCONF.

  • Syntax Rules: Everything is wrapped in tags, similar to HTML.
  • Every opening tag <tag> must have a matching closing tag </tag>.
  • Data resides between the tags.
  • Highly hierarchical (tree structure), but difficult for humans to read quickly due to visual clutter.
<interface>
  <name>GigabitEthernet1</name>
  <enabled>true</enabled>
  <vlans>
    <vlan>10</vlan>
    <vlan>20</vlan>
  </vlans>
</interface>

- YAML

YAML Ain't Markup Language. Designed specifically to be as human-readable as possible. The primary format used for Ansible Playbooks.

  • Syntax Rules: Drops brackets and tags entirely. Relies strictly on Indentation (spaces, never tabs).
  • Key-value pairs use a colon followed by a space key: value.
  • Lists (Arrays) are denoted by a dash and a space - item at the same indentation level.
  • Quotes around strings are usually optional unless they contain special characters.
--- # Optional start of document
interface:
  name: GigabitEthernet1
  enabled: true
  vlans:
    - 10
    - 20
    - 30

🪝 Brain Hooks

Visualizing the Formats

XML: Looks like HTML. <tags> everywhere.
JSON: Looks like code. {"Brackets": "Quotes"} everywhere.
YAML: Looks like a minimalist shopping list. Just words and - dashes.

Protocol Pairings

RESTCONF uses JSON (or XML).
NETCONF uses strictly XML.
Ansible uses YAML for playbooks.

🔎 Exam Strategy: Spot the Error

You will be shown snippets of code and asked to identify why they are invalid. Look for these common traps:

JSON Error: Missing Quotes `{ interface: "Gigabit1" }` INVALID. Keys MUST have double quotes. `{"interface": "Gigabit1"}` is valid.
JSON Error: Trailing Comma `{"ip": "1.1.1.1",}` INVALID. You cannot have a comma after the very last item in a JSON object or array.
XML Error: Mismatched Tags `<name>Gigabit1</Name>` INVALID. XML is case-sensitive. The closing tag must match exactly.
YAML Error: Bad Indentation If a child element is at the exact same indentation level as its parent, the YAML parser will throw a fatal syntax error. Spaces matter!

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Which data serialization format relies heavily on strict indentation and dashes (for lists), completely omitting curly braces or tags to maximize human readability? ▼
YAML (YAML Ain't Markup Language).
2. If you are interacting with a Cisco router using the NETCONF protocol, what is the primary data format you must use to encode your configuration payloads? ▼
XML (eXtensible Markup Language).
3. In JSON, what specific character(s) MUST wrap every single "Key" name in a Key-Value pair? ▼
Double quotes (" "). E.g., "hostname": "R1". (Single quotes are technically invalid in strict JSON).
4. When reading a JSON payload, what data type is indicated when values are enclosed in square brackets `[ ]` ? ▼
An Array (or List). It represents an ordered collection of values.
