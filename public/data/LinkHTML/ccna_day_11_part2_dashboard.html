Static Route Anatomy

A static route is manually configured by the administrator. It overrides dynamic routing protocols.

Router(config)# ip route [dest-network] [mask] [next-hop | exit-int]
  • Destination Network: The remote subnet you want to reach.
  • Subnet Mask: The mask of the destination network.
  • Forwarding IP/Int: How to get there (either the neighbor's IP or your own local exit interface).
🔀

Next-Hop vs Exit Int

You have two choices for instructing the router where to forward the packet.

Next-Hop IP Address

"Send to my neighbor's IP." Requires a Recursive Lookup (router checks the table once for the destination, then a second time to find out how to reach the next-hop IP). Best practice for Ethernet links.

Exit Interface

"Send directly out my local port (e.g., G0/0)." No recursive lookup needed. Best practice for point-to-point (serial) links. Using this on Ethernet can cause proxy ARP issues.

🌐

The Default Route

The "Gateway of Last Resort." Used when a packet's destination IP doesn't match any specific route in the routing table.

ip route 0.0.0.0 0.0.0.0 203.0.113.1

0.0.0.0 /0 means "Match any IP address".

Because of the Longest Prefix Match rule (/0 is the shortest possible prefix), this route is only used if absolutely no other route matches. Essential for routing traffic to the Internet!

Symbol in Routing Table: S*

🏆 Administrative Distance

AD is the "trustworthiness" of a route source. If a router learns about the exact same network from multiple sources, it chooses the one with the Lowest AD.

Route Source Default AD
Connected Interface0
Static Route1
eBGP20

OSPF110
🛟

Floating Static Routes

A floating static route is a backup route. It is configured with an Administrative Distance higher than the primary route.

How it Works

Because its AD is artificially raised, it "floats" invisibly in the configuration file, staying OUT of the active routing table.

If the primary route (e.g., an OSPF route with AD 110) goes down, the router looks for a backup. The floating static route (e.g., AD 115) immediately drops into the routing table and takes over.

Configuration Syntax

Simply add the desired AD value to the very end of your normal static route command.

ip route 10.1.1.0 255.255.255.0 192.168.2.2 115

*(In this example, 115 is the manually configured AD).*

⌨️ Static Route Config & Verification

Router(config)# ip route 172.16.1.0 255.255.255.0 10.0.0.2 Creates a standard static route pointing to the next-hop IP of 10.0.0.2.
Router(config)# ip route 0.0.0.0 0.0.0.0 g0/1 Creates a default route pointing out the local interface GigabitEthernet0/1.
Router# show ip route static Filters the routing table to show ONLY routes manually configured (Static routes).
Router# show ip route 172.16.1.5 Checks which route the router will use for this specific destination IP (including AD and metrics).

🔧 Troubleshooting

The Disappearing Route

Issue: You configured a static route using a next-hop IP, but it doesn't show up in show ip route.

Reason: A router will NOT put a static route into the routing table if the next-hop IP is completely unreachable (or if the exit interface is down). Verify layer 1/2 connectivity to the next-hop router!

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. If a static route uses a Next-Hop IP instead of an Exit Interface, what specific action must the router perform to find out how to forward the packet? ▼
A Recursive Lookup. The router must check the routing table a second time to determine which exit interface is used to reach the next-hop IP address.
2. A router learns about 10.1.1.0/24 from both OSPF (AD 110) and a Static Route (AD 1). What is Administrative Distance, and which route wins? ▼
Administrative Distance (AD) measures the trustworthiness of the routing protocol. The lowest AD wins, so the Static Route (AD 1) is installed in the routing table.
3. What is the purpose of a "Floating Static Route"? ▼
It serves as a backup route. By artificially configuring it with a higher AD than the primary routing protocol, it remains inactive until the primary path fails.
4. What is the exact syntax (IP and Mask) used to configure a Default Route? ▼
ip route 0.0.0.0 0.0.0.0 [next-hop IP or exit-interface]. The 0.0.0.0 /0 mask matches all destination IPs not explicitly found elsewhere in the table.
