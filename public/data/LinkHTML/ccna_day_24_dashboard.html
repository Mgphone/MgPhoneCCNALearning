Static vs. Dynamic

Why move away from static routes? In large networks, maintaining static routes becomes impossible. Dynamic routing protocols automate path discovery and recovery.

Static Routing

  • High administrative overhead.
  • Does not react to topology changes (if a link fails, the route stays until manually removed or the interface goes down).
  • Low CPU/RAM usage.
  • Highly secure (no routing updates to spoof).

Dynamic Routing

  • Low administrative overhead (scales easily).
  • Automatically adapts to topology changes (reroutes traffic if a link fails).
  • Higher CPU, RAM, and bandwidth usage (sends periodic or event-driven updates).
  • Requires authentication to prevent rogue routers from injecting false routes.
🌍

IGP vs EGP & The Autonomous System

An Autonomous System (AS) is a collection of networks under a single administrative domain (e.g., an ISP, a large enterprise). The Internet is essentially a web of connected Autonomous Systems.

IGP (Interior Gateway Protocol)

Used to route traffic within a single Autonomous System.

  • Optimized for speed and fast convergence.
  • Finds the absolute fastest path.
  • Examples: OSPF, EIGRP, IS-IS, RIP.

EGP (Exterior Gateway Protocol)

Used to route traffic between different Autonomous Systems.

  • Optimized for scale and administrative policy.
  • Path selection is based on rules, not necessarily speed.
  • Example: BGP (Border Gateway Protocol) - the protocol of the Internet.

🧩 Types of IGPs: Distance Vector vs Link State

1. Distance Vector

"Routing by Rumor." A router only knows what its direct neighbors tell it. It has no map of the full network.

  • Sends its entire routing table to neighbors.
  • Updates are usually sent periodically (e.g., every 30s).
  • Slow to converge; prone to routing loops.
  • Uses Bellman-Ford algorithm.

Protocols: RIPv1, RIPv2

2. Advanced Distance Vector

A Cisco-proprietary hybrid. Acts like a Distance Vector protocol but utilizes features of Link State protocols.

  • Sends partial, triggered updates instead of periodic full-table updates.
  • Maintains a topology table of neighbor routes.
  • Extremely fast convergence (DUAL algorithm).

Protocols: EIGRP

3. Link State

"The Map Maker." Every router floods information about its own links to every other router in the area.

  • Every router builds an identical, complete topology map of the network.
  • Each router independently calculates the best path to every destination using Dijkstra's Shortest Path First (SPF) algorithm.
  • Updates are sent only when a change occurs.

Protocols: OSPF, IS-IS

📏

Routing Metrics

A metric is the "cost" of a path. If a routing protocol learns two different ways to reach 10.1.1.0/24, it compares their metrics. The lowest metric wins.

  • RIP (Hop Count)

    Counts the number of routers crossed. A 10-Gig link and a 10 Mbps link are treated exactly the same. (Max hops = 15).

  • OSPF (Cost)

    Based inversely on bandwidth. Faster links have lower costs. 10-Gig is preferred over a 10 Mbps link.

  • EIGRP (Composite)

    A complex formula that defaults to using the lowest Bandwidth along the path plus the cumulative Delay of all links.

🏆 AD vs Metric

Metric

Used to decide between multiple paths learned by the SAME routing protocol. (e.g., OSPF Path A vs OSPF Path B).

Administrative Distance (AD)

Used to decide between paths learned by DIFFERENT routing protocols. Represents the "trustworthiness" of the source.

Core AD Values to Memorize (Lowest Wins):

Connected: 0
Static: 1
eBGP: 20
EIGRP: 90
OSPF: 110
IS-IS: 115
RIP: 120
Unknown: 255

⌨️ Dynamic Routing Verification

Router# show ip protocols Shows which routing protocols are currently active on the router, their AD, timers, and which networks they are advertising.
Router# show ip route Displays the routing table. Look at the code on the far left: O = OSPF, D = EIGRP, R = RIP, B = BGP.
Router# clear ip route * Forces the router to clear its routing table and rebuild it from scratch.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Which protocol category is used to route traffic between different Autonomous Systems across the Internet? ▼
EGP (Exterior Gateway Protocol). The only EGP in use today is BGP.
2. Which type of IGP builds a complete, identical topology map of the network on every router? ▼
Link State protocols (like OSPF and IS-IS).
3. A router learns about 10.1.1.0/24 from both OSPF and EIGRP. Which protocol's route will be injected into the routing table? ▼
EIGRP. Because the protocols are different, the router uses Administrative Distance. EIGRP (AD 90) is lower and more trusted than OSPF (AD 110).
4. What is the fundamental difference in how a router uses a Metric versus Administrative Distance? ▼
Metric is used to choose the best path when a single protocol finds multiple ways to reach a destination. Administrative Distance is used to choose which protocol to trust when multiple protocols find the same destination.
