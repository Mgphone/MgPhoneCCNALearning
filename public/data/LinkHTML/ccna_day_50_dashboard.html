Day 50: DHCP Snooping

Layer 2 Defense: Rogue Servers & Starvation

🕵️‍♂️

The DHCP Threats

By default, a switch blindly forwards all DHCP broadcasts. This exposes the network to two critical Layer 2 attacks.

1. Rogue DHCP Server

An attacker plugs in a device acting as a DHCP server. When a PC broadcasts a DHCP Discover, the attacker replies with a DHCP Offer faster than the real server. They assign the PC an IP and set their own IP as the Default Gateway, initiating a massive Man-in-the-Middle (MitM) attack.

2. DHCP Starvation Attack

An attacker uses a tool (like Yersinia) to generate thousands of DHCP Discover messages per second, each with a spoofed Source MAC address. The real DHCP server hands out every IP in its pool until it is exhausted, denying service to legitimate users.

The Solution: DHCP Snooping

A Cisco IOS feature that acts like a firewall between untrusted hosts and DHCP servers, validating DHCP messages and dropping malicious ones.

🧱

Trusted vs. Untrusted Ports

To stop rogue servers, DHCP Snooping categorizes every port on the switch into one of two states.

Untrusted Ports (Default)

Rule: Allowed to send DHCP Discover/Request messages. FORBIDDEN from sending DHCP Offer/Ack messages.

  • All ports are Untrusted by default when the feature is enabled globally.
  • Used for ports connecting to end-users (PCs, printers).
  • If an untrusted port sends a DHCP Offer, the switch drops it immediately!

Trusted Ports

Rule: Allowed to send ANY DHCP messages, including Offers and Acks.

  • Must be explicitly configured by the admin.
  • Used for the port connecting directly to the legitimate DHCP server.
  • Used for uplink (trunk) ports connecting to other switches leading to the DHCP server.

🗄️ The Binding Database

As DHCP Snooping watches legitimate DHCP transactions occur on untrusted ports, it records the details in a table called the DHCP Snooping Binding Database.

For every PC that successfully leases an IP, the switch logs:

  • MAC Address
  • IP Address Assigned
  • Lease Time
  • VLAN Number
  • Switch Interface (e.g., F0/1)

Why is this important?

This table becomes the absolute source of truth for the network. Advanced security features like Dynamic ARP Inspection (DAI) and IP Source Guard (IPSG) rely completely on this database to function!

🪝 Brain Hooks

Trust the Bartender

Think of a bar. Patrons (PCs) are allowed to ask for drinks (DHCP Discover). Only the Bartender (Trusted Port) is allowed to hand out drinks (DHCP Offer). If a patron tries to hand out drinks, they get thrown out (Packet Dropped).

Starvation & Speed Limits

To stop Starvation attacks, we apply a rate limit to untrusted ports. If a PC asks for 100 IPs in one second, the port shuts down.

⌨️ Configuration CLI

Switch(config)# ip dhcp snooping Step 1: Turns the feature on globally for the switch. (But does nothing yet!).
Switch(config)# ip dhcp snooping vlan 10,20 Step 2: Activates snooping on specific VLANs. Now all ports are untrusted by default!
Switch(config)# interface g0/1 Switch(config-if)# ip dhcp snooping trust Step 3: Configures the uplink/server port as Trusted. (CRITICAL: If you forget this, NO ONE gets an IP).
Switch(config)# interface range f0/1 - 24 Switch(config-if-range)# ip dhcp snooping limit rate 10 Step 4: Protects against Starvation attacks by limiting untrusted ports to 10 DHCP packets per second.
Switch(config)# no ip dhcp snooping information option Pro-Tip: Cisco switches insert Option 82 into DHCP packets by default. Some DHCP servers reject this. This command disables it to ensure smooth IP leases.

🔧 Verification

show ip dhcp snooping Shows global status, which VLANs are active, and lists all explicitly Trusted ports. Use this to verify you didn't forget Step 3!
show ip dhcp snooping binding Displays the critical database table showing which MACs on which ports were assigned which IP addresses.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. When you globally enable DHCP Snooping on a switch, what trust state are all interfaces placed into by default? ▼
They default to Untrusted. You must manually configure specific ports (like uplinks) to be trusted.
2. What action does the switch take if it receives a DHCP Offer message arriving on an Untrusted port? ▼
The switch drops the packet immediately, preventing the rogue server from assigning an IP.
3. How does DHCP Snooping prevent a DHCP Starvation attack where a PC spams thousands of Discover messages? ▼
By applying a rate limit (ip dhcp snooping limit rate [number]) on untrusted ports. If the port exceeds the limit of packets per second, it is put into the err-disable state.
4. What specific table is built by observing legitimate DHCP transactions, which is then heavily relied upon by Dynamic ARP Inspection (DAI)? ▼
The DHCP Snooping Binding Database. It maps the MAC address, assigned IP, VLAN, and physical interface.
