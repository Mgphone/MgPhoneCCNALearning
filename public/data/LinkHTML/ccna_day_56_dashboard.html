Day 56: Wireless Architectures

APs, WLCs, Split-MAC & CAPWAP

🎛️

AP Architecture Types

Managing a wireless network with 5 Access Points is easy. Managing one with 500 is impossible without changing the architecture.

Autonomous APs ("Fat APs")

Standalone devices. Every single AP must be individually configured with SSIDs, security policies, and RF channels. It performs all management and real-time functions itself. Does not scale well.

Lightweight APs (LAPs)

"Thin APs". They have zero configuration on them. They boot up, find a Wireless LAN Controller (WLC), and download their configuration. The WLC acts as the central brain for hundreds of LAPs.

The Paradigm Shift With LAPs, you no longer configure APs; you configure the WLC, and the WLC pushes the policy to the entire network instantly.
🧠

Split-MAC Architecture

In a Lightweight deployment, the duties of the MAC layer are split between the AP and the WLC.

LAP (Real-Time Functions)

The physical AP handles anything that requires immediate, millisecond response times over the airwaves.

  • Transmitting Beacons and Probes.
  • CSMA/CA and MAC acknowledgments.
  • Packet encryption/decryption (so plain text goes to WLC).

WLC (Management Functions)

The Controller handles the heavy lifting, security, and global coordination.

  • Client Authentication (e.g., 802.1X/RADIUS).
  • Roaming coordination between APs.
  • Radio Frequency Management (RRM) - Auto-tuning power and channels.

🚇 CAPWAP Protocol

Control and Provisioning of Wireless Access Points. It is the protocol that bridges the LAP and the WLC. It builds two separate UDP tunnels.

CAPWAP Control (UDP 5246)

Used to manage the AP. Always encrypted via DTLS (Datagram Transport Layer Security). Protects configuration and passwords.

CAPWAP Data (UDP 5247)

Used to tunnel the actual user traffic (like web browsing) from the AP to the WLC. Unencrypted by default (can be enabled if hardware supports it).

🏢 WLC Deployment Models

Where should the WLC sit physically? Cisco outlines four primary deployment models.

1. Centralized (Unified)

A hardware appliance sitting in the core data center. Supports thousands of APs. All client traffic from every AP is tunneled all the way back to this WLC before hitting the LAN.

2. FlexConnect (Branch)

Used for remote branches connected over slow WANs. APs get config from a central WLC, but can drop client traffic locally onto the branch switch without tunneling it to HQ. Has survivability if WAN drops.

3. Embedded

The WLC software is built directly into an access-layer switch (e.g., Catalyst 9300). Used for very small campuses (~200 APs max) to save money on buying a dedicated appliance.

4. Cloud (Meraki)

WLC is hosted as a service in the public cloud. APs connect over the internet to get their config. Extremely simple GUI, scales infinitely.

🪝 Brain Hooks

CAPWAP Ports Mnemonic

5246 = Control (C is a round letter, like 6)
5247 = Data (D is an angular letter, like 7)

FlexConnect Analogy

Think of FlexConnect like a remote retail store. The store manager (WLC) is at HQ. The cashier (AP) usually asks the manager what to do. If the phone lines (WAN) go down, the cashier has permission to keep checking out local customers (Local Switching) until the manager returns.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. A Lightweight Access Point (LAP) cannot function on its own. What central device must it discover and tunnel traffic to? ▼
A Wireless LAN Controller (WLC).
2. CAPWAP establishes two UDP tunnels. What are the port numbers, and which one is encrypted by default? ▼
Control traffic uses UDP 5246 and is always encrypted via DTLS. Data traffic uses UDP 5247 and is unencrypted by default.
3. In the Split-MAC architecture, which device is responsible for authenticating clients via 802.1X and managing roaming? ▼
The Wireless LAN Controller (WLC). The AP only handles real-time airwave tasks.
4. Which WLC deployment model allows branch office APs to drop user traffic directly onto the local switch, maintaining functionality even if the WAN link to HQ goes down? ▼
FlexConnect.
