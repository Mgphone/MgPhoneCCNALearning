IPv4 Header Structure (20 Bytes Minimum)

Every IP packet has this header. It is exactly 32 bits wide (4 Bytes) per row. The standard header is 5 rows deep, totaling 20 Bytes.

Version (4)
IHL (4)
DSCP (6)
ECN(2)
Total Length (16)
Identification (16)
Flags(3)
Fragment Offset (13)
TTL (8)
Protocol (8)
Header Checksum (16)
Source IP Address (32)
Destination IP Address (32)

*(Hover over fields to see their function. Options/Padding row is omitted as it's rarely used in modern IPv4).*

🔥

Highly Tested Fields

TTL (Time to Live)

Starts at a specific number (like 64, 128, or 255) depending on the OS. Every time a router forwards the packet, it subtracts 1. If TTL hits 0, the router drops the packet and sends an ICMP "Time Exceeded" message back. This prevents packets from looping forever if there is a routing error.

Protocol

Tells the receiving device what is inside the payload so it can pass it to the correct Layer 4 process. Memorize these numbers:

1 = ICMP
6 = TCP
17 = UDP
89 = OSPF
DSCP (Differentiated Services Code Point)

Formerly called the ToS (Type of Service) byte. Used for QoS (Quality of Service). Routers look at this to prioritize traffic (e.g., Voice over IP gets higher priority than web browsing).

🧩

Fragmentation

If a packet is larger than an interface's MTU (Maximum Transmission Unit, typically 1500 bytes), a router must slice it into smaller pieces.

  • Identification: All fragments of a single original packet get the same ID number, so the destination knows they belong together.
  • DF Flag (Don't Fragment): If set to 1, the router is forbidden from fragmenting. If the packet is too big, it drops it and sends an ICMP error.
  • MF Flag (More Fragments): Set to 1 on all fragments except the very last one, which is set to 0.
  • Fragment Offset: Tells the destination exactly what order to reassemble the pieces in.

🪝 Brain Hooks

Protocol Number Trick

TCP has 3 letters. 3 x 2 = 6.
UDP has 3 letters... well, just memorize 17 (it's prime and fast, like UDP).
ICMP is number 1 (Ping is the #1 tool we use).

TTL = The Expiration Date

Think of a packet like a gallon of milk passed down an assembly line. Every person (router) subtracts a day. If it hits 0, it's thrown in the trash (dropped).

⌨️ How CLI Tools Exploit the Header

You don't configure the header directly in the CLI, but your standard troubleshooting tools manipulate it!

traceroute (or Windows 'tracert')

Traceroute ingeniously hacks the TTL field.
1. It sends a packet with a TTL of 1. The 1st router drops it and replies. Traceroute records Router 1's IP.
2. It sends a packet with a TTL of 2. The 2nd router drops it. Traceroute records Router 2's IP.
3. It repeats this until the packet reaches the destination, effectively mapping every hop on the path.

ping

When you ping, your PC crafts an IPv4 packet, sets the Destination IP, and sets the Protocol field to 1 to indicate an ICMP Echo Request payload is inside.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. What is the minimum length of an IPv4 header? ▼
20 Bytes. (Without any optional padding).
2. Which field specifically prevents a packet from endlessly circulating in a routing loop? ▼
TTL (Time to Live). Every router decrements it by 1, dropping the packet if it reaches 0.
3. You capture a packet in Wireshark and see the Protocol field is set to '6'. What payload is this packet carrying? ▼
TCP. (1=ICMP, 6=TCP, 17=UDP, 89=OSPF).
4. Which field is primarily used by routers to implement Quality of Service (QoS) and prioritize voice traffic? ▼
DSCP (Differentiated Services Code Point), formerly known as the ToS (Type of Service) byte.
