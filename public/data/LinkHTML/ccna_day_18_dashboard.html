Day 18: VLANs (Pt 3)

Inter-VLAN Routing: ROAS & SVIs

🚧

The VLAN Boundary

VLANs completely isolate Layer 2 broadcast domains. If PC1 in VLAN 10 wants to talk to PC2 in VLAN 20, a standard Layer 2 switch cannot forward that traffic.

The Layer 3 Requirement

To move traffic between different subnets/VLANs, you absolutely must have a Layer 3 routing process. We call this Inter-VLAN Routing.

Method 1: Legacy (Router per VLAN)

Historically, you'd run a physical cable from the switch to the router for every single VLAN. Highly inefficient, wastes router ports, and doesn't scale.

Modern Methods

We solve this today using two primary designs: Router-on-a-Stick (ROAS) or Multilayer (L3) Switching.

🍭

Router-on-a-Stick (ROAS)

ROAS uses a single physical link between the Switch and the Router, configured as an 802.1Q Trunk.

Subinterfaces Instead of assigning an IP to the physical router interface (like G0/0), we divide it into logical subinterfaces (like G0/0.10 and G0/0.20). Each subinterface acts as the default gateway for one specific VLAN.
The Switch Side The switch port connected to the router MUST be configured as a Trunk port, allowing all necessary VLANs to pass up to the router.
🎛️

Multilayer Switching & SVIs

A Layer 3 Switch (Multilayer Switch) can perform hardware-based routing internally, making it much faster than ROAS. We don't need a separate physical router.

  • Switch Virtual Interface (SVI): A logical, virtual interface built inside the switch (e.g., interface vlan 10). It acts as the Layer 3 default gateway for devices in that VLAN.
  • IP Routing: By default, Cisco switches only perform Layer 2 switching. You must explicitly turn on the routing engine using the global command ip routing.
  • Routed Ports: You can also turn a physical switch port into a router port (removing its Layer 2 switching ability) by using the no switchport command.

🪝 Brain Hooks

The Office Building

ROAS: Every floor (VLAN) must go all the way down to the lobby receptionist (Router) to send a message to another floor. Bottleneck!

L3 Switch: Every single floor has its own local receptionist (SVI). Much faster internal communication!

⌨️ ROAS Config (Router)

Router(config)# interface g0/0 Router(config-if)# no shutdown Power on the main physical interface (NO IP address goes here).
Router(config)# interface g0/0.10 Create subinterface. (Matching the VLAN ID is best practice).
Router(config-subif)# encapsulation dot1q 10 CRITICAL: You MUST specify the VLAN tagging (dot1q) before it will accept an IP address!
Router(config-subif)# ip address 192.168.10.1 255.255.255.0 Assign the default gateway IP.

⌨️ L3 Switch Config (SVIs)

Switch(config)# ip routing CRITICAL: Turns on the routing engine. If you forget this, the SVIs will function locally but won't route traffic between them!
Switch(config)# vlan 20 Create the L2 VLAN first.
Switch(config)# interface vlan 20 Create the L3 SVI.
Switch(config-if)# ip address 192.168.20.1 255.255.255.0 Switch(config-if)# no shutdown Assign gateway IP and enable.

🔧 Troubleshooting

"IP routing not enabled"

Symptom: SVIs are 'up/up', PCs can ping their gateway, but they cannot ping a PC in another VLAN.

Fix: You forgot to type ip routing in global config mode on the L3 switch.

ROAS Ping Fails

Fix: Check the switch port connected to the router. It must be explicitly configured as switchport mode trunk.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. When configuring a Router-on-a-Stick subinterface, which command must be entered before the router will accept an `ip address` command? ▼
encapsulation dot1q [vlan-id]. The router must know which 802.1Q VLAN tag this subinterface is responsible for before it will assign an IP to it.
2. In a ROAS topology, what operational mode MUST the switch port connecting to the router be in? ▼
A Trunk port (switchport mode trunk). It must carry tagged frames for multiple VLANs up to the router's subinterfaces.
3. You have configured SVIs for VLAN 10 and VLAN 20 on a Multilayer Switch. What global command is required for the switch to actually route packets between them? ▼
ip routing
4. What does the acronym SVI stand for, and what is its primary purpose in inter-VLAN routing? ▼
Switch Virtual Interface. It is a logical Layer 3 interface inside a switch that acts as the Default Gateway for all hosts in that specific VLAN.
