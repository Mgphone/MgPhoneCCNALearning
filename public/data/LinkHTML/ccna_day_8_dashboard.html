The 2 Reserved Addresses

Every single network subnet has TWO addresses that cannot be assigned to a PC or Router.

1. Network Address

Identifies the subnet itself. All host bits are set to binary 0.

Ex: 192.168.1.0/24

2. Broadcast Address

Used to send data to ALL hosts in the subnet. All host bits are set to binary 1.

Ex: 192.168.1.255/24

Usable Hosts Formula: 2H - 2

(H = number of Host bits. We subtract 2 for the Network and Broadcast addresses!)

🔪

Slash Notation (CIDR)

Classless Inter-Domain Routing (CIDR) notation is a shorthand way to write the subnet mask. The number after the slash tells you exactly how many network bits (1s) there are.

  • /8 255.0.0.0
  • /16 255.255.0.0
  • /24 255.255.255.0
  • /32 255.255.255.255 (Host Route)
🌍

Private IP Ranges

Defined in RFC 1918, these IPs are NOT routable on the public internet. They are used inside LANs. NAT (Network Address Translation) is required to reach the web.

Class A Range: 10.0.0.0 — 10.255.255.255 (/8)
Class B Range: 172.16.0.0 — 172.31.255.255 (/12)
Class C Range: 192.168.0.0 — 192.168.255.255 (/16)
👽

Special IP Addresses

Loopback Address 127.0.0.0/8

Usually 127.0.0.1. Pinging this tests if your computer's own TCP/IP stack is functioning correctly.

APIPA / Link-Local 169.254.0.0/16

Automatic Private IP Addressing. If a PC tries to get an IP from DHCP and fails, Windows automatically assigns an IP in this range.

🪝 Brain Hooks & Analogies

The Office Extension Analogy

Public IP: The company's main phone number. Anyone in the world can dial it.
Private IP: Your desk extension (x402). Only people inside the building can call you directly using it.

The "Minus 2" Rule

Imagine a street (Subnet).
The street sign at the corner is the Network Address. You can't live in the street sign.
The loud PA speaker hitting all houses is the Broadcast Address. You can't live in the speaker.
Subtract those 2, and you have your usable houses (Hosts).

🔧 Troubleshooting

Bad Mask Config

Error: Bad mask /x for address x.x.x.x

Fix: You tried to assign a Network or Broadcast address to an interface, or your subnet mask overlaps with the host portion. Check your math.

"I can't reach the Internet!"

Check: Run ipconfig on Windows. If the IP starts with 169.254.x.x, the PC cannot reach the DHCP server. Check the cable, switch port, and DHCP scope.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Can you assign the IP address 192.168.1.255 /24 to a PC? ▼
No. Because the subnet mask is /24, the host portion is the last octet. An octet value of 255 means all host bits are '1', which makes this the Broadcast Address.
2. If a network has 8 host bits (a /24 mask), how many USABLE host addresses are there? ▼
254. The formula is $2^8 - 2$. (256 - 2 = 254).
3. You run ipconfig on a user's PC and see the IP address is 169.254.10.5. What is the immediate problem? ▼
The PC has generated an APIPA (Link-Local) address. This indicates it failed to contact a DHCP server to obtain a valid IP address.
4. Is the IP address 172.20.5.5 Public or Private? ▼
Private. It falls within the RFC 1918 Class B range of 172.16.0.0 through 172.31.255.255.
