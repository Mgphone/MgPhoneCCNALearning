Day 30: TCP & UDP

Layer 4 (Transport) Protocols & Ports

🔀

Transport Layer (L4)

Layer 3 (IP) gets the packet to the correct computer. Layer 4 gets the data to the correct Application on that computer.

Multiplexing via Ports

Your PC has one IP address, but you can have Chrome, Spotify, and Discord running simultaneously. L4 uses Port Numbers (16-bit) to multiplex these connections.

Sockets

A socket is a combination of an IP Address + Port Number.
Ex: 192.168.1.5:443

Port Ranges

  • Well-Known (0-1023): Server applications (HTTP, SSH).
  • Registered (1024-49151): Vendor specific apps.
  • Ephemeral (49152-65535): Temporary, random source ports picked by clients when initiating a connection.
🤝

TCP

Transmission Control Protocol. IP Protocol Number: 6.

  • Connection-Oriented Must establish a session before sending data (3-way handshake).
  • Reliable & Ordered Uses sequence and acknowledgment numbers. Lost segments are retransmitted. Segments arrive in order.
  • Flow Control Uses "Windowing" to adjust transmission speed dynamically based on the receiver's buffer capacity.
🚀

UDP

User Datagram Protocol. IP Protocol Number: 17.

  • Connectionless Just sends data. No handshakes, no setup. "Fire and forget."
  • Unreliable (Best Effort) No sequence numbers, no acknowledgments. If data is dropped, it's gone.
  • Fast & Lightweight Header is only 8 bytes (compared to TCP's 20 bytes). Perfect for real-time traffic (Voice, Video, gaming).

🔄 TCP Handshakes & Teardown

TCP uses control flags (bits) in its header to manage sessions.

The 3-Way Handshake (Setup)

  • 1. Client -> Server: SYN
  • 2. Server -> Client: SYN, ACK
  • 3. Client -> Server: ACK

Once step 3 completes, the connection is ESTABLISHED and data transfer begins.

The 4-Way Teardown (Close)

  • 1. Client -> Server: FIN
  • 2. Server -> Client: ACK
  • 3. Server -> Client: FIN
  • 4. Client -> Server: ACK

Both sides must gracefully close their half of the connection.

🪝 Brain Hooks

"UDP = U Don't Promise"

UDP doesn't promise delivery, ordering, or reliability. It just dumps the packets on the wire and hopes for the best.

The Registered Mail vs Postcard

TCP: Registered mail. You sign for it when it arrives (ACK). If it doesn't arrive, the sender mails it again (Retransmission).
UDP: A postcard. You drop it in the mailbox. If it gets lost, nobody knows.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Does HTTP/HTTPS use TCP or UDP at the transport layer, and why? ▼
TCP. Web browsing requires reliable, ordered delivery. If a packet carrying part of an image or text is dropped, it must be retransmitted, otherwise the webpage would be corrupted.
2. What are the three specific flags exchanged during the TCP 3-Way Handshake? ▼
SYN (Synchronize), SYN/ACK (Synchronize-Acknowledge), and ACK (Acknowledge).
3. You need to backup a Cisco router configuration to a TFTP server. What protocol and port number must be open on the firewall? ▼
UDP Port 69. (TFTP uses UDP because it implements its own lightweight reliability mechanism at the application layer, avoiding TCP overhead).
4. When your PC connects to a web server on Port 443, it randomly generates its own source port number (e.g., 51004). What is this type of temporary port called? ▼
An Ephemeral Port. It allows the PC to track the return traffic for that specific browser tab/session.
