Day 41: SYSLOG

System Logging Protocol & Network Monitoring

📜

What is Syslog?

Syslog is a standard protocol used to send system log or event messages to a specific server. It is critical for network monitoring, auditing, and troubleshooting.

Protocol & Port

Syslog operates over UDP Port 514. Because it uses UDP, it is a "fire-and-forget" protocol. The router doesn't care if the server actually receives the log.

The Anatomy of a Log

A typical Cisco Syslog message looks like this:

*Mar 1 00:15:24.015: %LINK-3-UPDOWN: Interface GigabitEthernet0/0, changed state to up
  • Timestamp: When it happened (requires NTP!).
  • Facility: What generated the log (e.g., LINK).
  • Severity: How bad it is (e.g., 3).
  • Mnemonic: Short description (e.g., UPDOWN).
  • Message: Human-readable details.

📥 Logging Destinations

Where do the logs go? A Cisco router can send syslogs to four primary locations.

1. Console (Default)

Logs are printed directly to your screen when physically connected via a console cable.

2. VTY Lines (SSH/Telnet)

Disabled by default! If you SSH into a router, you won't see logs popup automatically unless you run terminal monitor.

3. Buffer (RAM)

Saves logs internally to the device's RAM. You can view them later using show logging. If the router reboots, these are wiped.

4. External Syslog Server

Sends logs over the network to a dedicated NMS (like Splunk or SolarWinds). Best practice for centralized logging and historical retention.

🎛️ Filtering & Timestamps

The "Trap" Rule

If you configure the router to send logs to a server using logging trap 4 (Warning), the router sends Level 4 AND EVERYTHING LOWER (0, 1, 2, 3, and 4).

You cannot single out just one level. It always sends the specified level and all levels more critical than it.

The NTP Dependency

Syslog without NTP is useless. If a router's clock is wrong, its logs will have the wrong timestamp.

To force the router to put a human-readable date/time on every log (instead of uptime), you must use:
service timestamps log datetime msec

⌨️ Configuration & Verification CLI

Router(config)# logging host 10.1.1.50 Tells the router to send syslog messages to the external server at 10.1.1.50. (Sometimes just `logging 10.1.1.50`).
Router(config)# logging trap 4 Restricts the messages sent to the syslog server to level 4 (Warning) and below (0-4). Helps prevent overwhelming the server with level 7 debugs!
Router(config)# logging buffered 16000 Enables logging to the internal RAM buffer and sets the size to 16,000 bytes.
Router# terminal monitor Privileged EXEC Command! If you are SSH'd into a router, you MUST type this to see syslog messages pop up live on your screen.
Router# show logging Displays the contents of the internal buffer and shows the configuration of your logging destinations.

🔧 Troubleshooting

"I'm SSH'd in but can't see logs!"

Issue: You shut an interface, but no log message appeared on your screen.

Fix: By default, logs only go to the console cable. Type terminal monitor to mirror console logs to your VTY (SSH/Telnet) session.

1. Which Syslog severity level corresponds to highly detailed "Debugging" output that can severely spike a router's CPU? ▼
Level 7 (Debugging).
2. Which Transport Layer protocol and port number does Syslog use to send logs to an external server? ▼
UDP Port 514. (It is unacknowledged/unreliable).
3. If you configure a Cisco router with the command `logging trap 3`, which specific severity levels will be forwarded to the syslog server? ▼
Levels 0, 1, 2, and 3. (Emergency, Alert, Critical, and Error). The command sets the threshold, allowing that level and all levels more severe (lower numbers).
