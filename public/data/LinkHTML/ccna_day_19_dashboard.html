DTP (Dynamic Trunking)

Dynamic Trunking Protocol is a Cisco-proprietary protocol that automatically negotiates whether a link should be an access port or a trunk port.

DTP Modes

  • Dynamic Desirable: Actively tries to form a trunk. Sends DTP frames.
  • Dynamic Auto: Passively waits. Will form a trunk only if the other side asks. (Default on modern switches).

Negotiation Matrix

Auto + Auto =Access
Desirable + Auto =Trunk
Desirable + Desirable =Trunk
On + Auto/Desirable =Trunk
Security Best Practice 🚨 Never leave DTP running in a production network. Attackers can spoof DTP frames to negotiate a trunk and perform VLAN Hopping. Hardcode ports as `access` or `trunk` and use `switchport nonegotiate`.
📡

VTP (VLAN Trunking Protocol)

VTP is a Cisco-proprietary protocol that synchronizes the VLAN database (creation, modification, and deletion of VLANs 1-1005) across all switches in the same VTP domain. It sends updates over Trunk links.

👑 Server (Default)

  • Can create, modify, delete VLANs.
  • Sends VTP updates.
  • Syncs its database with other updates.
  • Saves VLANs to NVRAM (`vlan.dat`).

🎧 Client

  • Cannot create, modify, delete VLANs.
  • Forwards VTP updates.
  • Syncs its database with the Server.
  • Does NOT save VLANs to NVRAM (in older versions).

👻 Transparent

  • Can create LOCAL VLANs only.
  • Forwards updates (acts like a bridge).
  • Does NOT sync its own database.
  • Saves local VLANs to running-config.

💣 The Configuration Revision #

The Configuration Revision (CR) number is the most important concept in VTP. Every time you add, delete, or modify a VLAN on a VTP Server, the CR number increases by 1.

The Rule of the Realm:

Switches will ALWAYS overwrite their own VLAN database if they receive a VTP update with a HIGHER revision number than their own, even if they are a VTP Server!

The "VTP Bomb"

If you take an old lab switch that has a CR of 500 (with all its VLANs deleted) and plug it into a production network (CR of 40), the entire production network will sync to CR 500 and delete all their VLANs!

🪝 Brain Hooks

The Office Gossip (VTP)

Server: Creates the rumors and listens to newer ones.
Client: Just listens and passes them on. Can't start a rumor.
Transparent: Doesn't care about your drama (won't sync), but will politely pass the note to the next person.

Defusing the Bomb

To reset a VTP Revision Number to 0 before plugging a switch in: Change the VTP Domain name to a fake name, then change it back. Or, change it to Transparent mode, then back to Server/Client.

⌨️ VTP & DTP Configuration

Switch(config)# vtp domain CISCO Sets the VTP domain. (Switches with no domain will automatically adopt the domain name of the first VTP packet they receive!)
Switch(config)# vtp mode {server | client | transparent} Changes the VTP operating mode. Default is server.
Switch(config)# vtp password CCNA Sets a password. If set, it MUST match on all switches for updates to sync.
Switch# show vtp status Critical verification command. Shows VTP Version, Domain Name, Mode, and the Configuration Revision number.
Switch# show dtp interface g0/1 Shows detailed DTP information for a port, including if it is actively negotiating.

🔧 Troubleshooting

VLANs aren't syncing

Check 1: Are the links between switches actively forming a Trunk? (VTP only travels over trunks).

Check 2: Does the VTP Domain name match exactly (case-sensitive)?

Check 3: Is a password configured on one switch but not the other?

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Switch A's interface is set to `dynamic desirable`. Switch B's interface is set to `dynamic auto`. Will a trunk successfully form between them? ▼
Yes, a trunk will form. Switch A actively initiates the negotiation (desirable), and Switch B passively accepts it (auto).
2. Which VTP mode allows you to create local VLANs on the switch, but ignores VTP updates from the network while still forwarding them to downstream switches? ▼
VTP Transparent mode.
3. You plug a lab switch (VTP Server) into your production network (VTP Server). Suddenly, all production VLANs disappear. Why did the production switch overwrite its own database? ▼
Because the lab switch had a higher Configuration Revision number. VTP unconditionally trusts the update with the highest revision number.
4. What is the standard administrative method to safely reset a switch's VTP Revision Number back to 0 before connecting it to a network? ▼
Temporarily change the VTP Domain name to a bogus name and change it back, OR change the VTP mode to Transparent and then back to Server/Client.
