IPv6 Anatomy

IPv4 gave us 4.3 billion addresses (which we exhausted). IPv6 gives us 340 undecillion addresses. To accommodate this massive size, the format is entirely different.

128 Bits Total 8 Hextets (Quartets) 16 Bits per Hextet Hexadecimal Format

2001:0db8:0000:0000:0000:ff00:0042:8329

  • Written in Hexadecimal (0-9 and a-f). Case-insensitive.
  • Separated by colons (:), not periods.
  • Each block between colons is called a Hextet (or Quartet), representing 16 bits (4 hex digits).
✂️

Shortening Rules

Writing 32 hex characters is exhausting. You must master the two rules used to compress IPv6 addresses.

Rule 1: Omit Leading Zeros

Within any hextet, you can remove zeros that appear at the beginning. You cannot remove trailing zeros!

0db8 : 0042 : 0000

 db8 :   42 :    0

Rule 2: Zero Compression (::)

A single continuous string of all-zero hextets can be replaced with a double colon (::). You can ONLY do this ONCE per address.

2001:db8:0:0:0:ff00:42:8329

2001:db8::ff00:42:8329

🗂️ IPv6 Address Types

You can easily identify what an IPv6 address is used for simply by looking at its first few characters.

Global Unicast (GUA) 2000::/3

The IPv6 equivalent of a Public IPv4 address. Routable across the entire internet. Globally unique.

Typically starts with 2 or 3.

Unique Local (ULA) fc00::/7

The IPv6 equivalent of Private IPv4 addresses (RFC 1918). Routable only within your organization. Not allowed on the internet.

Always starts with fc or fd.

Link-Local (LLA) fe80::/10

Routable ONLY on the local physical link. Routers will never forward these packets. Mandatory for IPv6 to function.

Always starts with fe80.

Multicast ff00::/8

Sends a single packet to multiple specific destinations simultaneously. Replaces broadcasts entirely.

Always starts with ff.

🚫

No Broadcasts!

In IPv4, Broadcasts (sending to everyone) cause immense CPU overhead on local networks. IPv6 completely eliminates broadcasts.

What replaces them?

IPv6 relies heavily on Multicast. Instead of shouting to the whole network, IPv6 hosts join specific multicast groups (like a mailing list). If a protocol needs to find something, it multicasts only to devices that care.

Anycast

A new type in IPv6. Multiple routers share the exact same IP address. A packet sent to this address goes only to the closest router. (One-to-closest).

🪝 Brain Hooks

Link-Local is Mandatory

In IPv4, 169.254.x.x (APIPA) means your DHCP is broken. In IPv6, a Link-Local (fe80::) address is required on every interface to handle routing protocol chatter (like OSPFv3) and neighbor discovery.

The Double Colon Danger

If you use :: twice, the router won't know how many zeros belong to the first one vs the second one. That's why it's illegal.

⌨️ Basic IPv6 Configuration

Router(config)# ipv6 unicast-routing CRITICAL: IPv6 routing is disabled by default on Cisco routers! You MUST run this global command, or the router will not forward any IPv6 packets.
Router(config-if)# ipv6 address 2001:db8:1::1/64 Assigns a Global Unicast address to the interface. Note that the prefix length (/64) is attached directly to the address without a space.
Router(config-if)# ipv6 enable Automatically generates a Link-Local (fe80::) address for the interface without assigning a Global Unicast address.
Router# show ipv6 interface brief The IPv6 equivalent of `show ip int br`. You will notice every active interface has at least two IPs: a Global Unicast AND a Link-Local address.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Which of the following is the IPv6 equivalent of an IPv4 Private address: 2000::/3, fd00::/8, or ff00::/8? ▼
fd00::/8 (and fc00::/7). These are Unique Local Addresses (ULA) and are not routable on the public internet.
2. True or False: The address 2001::db8::1 is a validly formatted IPv6 address. ▼
False. You can only use the double colon (::) once per address. Otherwise, the router cannot determine how many blocks of zeros belong to each section.
3. You configure IPv6 addresses on two router interfaces, but the router refuses to forward packets between them. What global command did you forget? ▼
ipv6 unicast-routing. By default, Cisco routers only act as IPv6 hosts until this command enables routing functionality.
4. Every active IPv6 interface must automatically generate an address used solely for communication on its own physical segment. What prefix do these addresses start with? ▼
fe80::/10. These are Link-Local Addresses (LLA).
