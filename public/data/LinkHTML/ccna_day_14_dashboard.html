Extracting the Subnet

On the exam, you won't just be asked to build subnets from scratch. You will be given a random host IP and asked: "What network does this IP belong to?"

The 4-Step Extraction Method:

  1. Identify the Interesting Octet: Look at the CIDR mask. Which octet is not 255 or 0?
  2. Find the Magic Number: 256 - (Interesting Octet Value) = Block Size.
  3. List the Multiples: Start at 0 and count up by your block size (0, 32, 64...).
  4. The "Price is Right" Rule: Find the multiple that is closest to the IP address's value without going over. That is your Network ID!
🧪

Example: 192.168.1.140 /26

Goal: Find the Network ID, Broadcast ID, and valid host range.

Step 1 & 2: Magic Number

Mask /26 = 255.255.255.192

Interesting octet is the 4th.

256 - 192 = 64 (Block Size)

Step 3 & 4: "Price is Right"

Multiples of 64: 0, 64, 128, 192

Our target IP value is 140.

Closest multiple without going over is 128.

Network ID: 192.168.1.128

First Host: 192.168.1.129

Last Host: 192.168.1.190

Broadcast ID: 192.168.1.191 (1 less than next network 192)

🤯 Class B & A Subnetting (2nd/3rd Octets)

The exact same rules apply, but you must remember to fill out the remaining octets with 0s (for Network) or 255s (for Broadcast).

Example: 172.16.100.5 /19

  1. /19 is 255.255.224.0. The 3rd octet is interesting.
  2. Magic Number: 256 - 224 = 32.
  3. Multiples: 0, 32, 64, 96, 128.
  4. Target is 100. The multiple without going over is 96.
Network ID 172.16.96.0

4th octet is all host bits (zeros).

Broadcast ID 172.16.127.255

Next net is 128.0. One less is 127.255.

🪝 Brain Hooks

"The Price is Right" Rule

When looking at your multiples (0, 16, 32, 48...), you want to bid as close to the target IP's octet value as possible without going over. That winning bid is your Network ID!

The .255 Rollover

If the next network is 10.1.64.0, what is 1 minus 0? You can't do it! You borrow from the 64 (making it 63) and the 0 rolls backward to 255. Broadcast = 10.1.63.255.

🔧 Troubleshooting

The "Overlaps" Error

% 192.168.1.64 overlaps with GigabitEthernet0/0

Issue: A router requires every active interface to be on a completely different subnet. If you try to assign an IP that falls into the same subnet block as another interface, IOS blocks it.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Find the Network ID for the IP address 192.168.1.75 /28. ▼
192.168.1.64. A /28 is 255.255.255.240. Magic number is 16 (256 - 240). Multiples: 0, 16, 32, 48, 64, 80. The closest to 75 without going over is 64.
2. In the IP address 10.50.100.5 /11, which octet is the "interesting" octet? ▼
The 2nd octet. A /11 mask is 255.224.0.0. The 2nd octet (224) is the one where the subnet boundary falls.
3. If your Network ID is 172.16.32.0 and the next subnet block starts at 172.16.64.0, what is the broadcast address for your subnet? ▼
172.16.63.255. It is exactly one address prior to the start of the next subnet.
4. Host A is 10.1.1.15 /27. Host B is 10.1.1.40 /27. Can they ping each other directly if connected to the same Layer 2 switch? ▼
No. A /27 has a block size of 32. Host A is in the 10.1.1.0 subnet. Host B is in the 10.1.1.32 subnet. Because they are on different subnets, they require a Router (Default Gateway) to communicate.
