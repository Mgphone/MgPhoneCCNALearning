The Problem: L2 Loops

Redundancy is good, but connecting switches in a physical loop without protection causes catastrophic network failure. Why? Layer 2 frames have no TTL (Time To Live) field!

1. Broadcast Storms

If a PC sends a broadcast (like ARP), SW1 floods it to SW2, SW2 floods to SW3, SW3 floods back to SW1. It loops infinitely, consuming all bandwidth and CPU.

2. MAC Table Instability

Because the same frame arrives on different ports from the loop, the switch constantly updates the Source MAC address to different ports (MAC Flapping).

3. Multiple Frame Transmission

The destination PC receives the exact same unicast frame multiple times, confusing applications and breaking connections.

🛡️

The Solution: STP (802.1D)

Spanning Tree Protocol (IEEE 802.1D) intelligently blocks specific ports to create a loop-free logical topology, while keeping the physical links ready as backups.

Bridge Protocol Data Units (BPDUs)

Switches communicate STP information by sending BPDUs out all ports every 2 seconds.

  • Contain the sender's Bridge ID.
  • Contain the Cost to reach the Root.
  • Sent to a multicast MAC address (01:80:C2:00:00:00).

👑 The Root Bridge Election

STP elects a single "King" of the network called the Root Bridge. All other switches map their shortest paths to it. The switch with the LOWEST Bridge ID (BID) wins.

Anatomy of the Bridge ID (8 Bytes)

  1. Bridge Priority (2 Bytes): Default is 32768. Must be modified in increments of 4096.
  2. Extended System ID: Added to the priority. It is simply the VLAN ID (e.g., Priority 32768 + VLAN 10 = 32778).
  3. MAC Address (6 Bytes): The tie-breaker! If priorities are equal, the switch with the lowest (oldest) MAC address wins.

Rule: LOWER IS ALWAYS BETTER IN STP!

🚦

Port Roles & Path Costs

After the Root Bridge is elected, switches assign roles to their ports based on the shortest path (lowest cost) to the Root.

1. Root Port (RP) The single port on a non-root switch with the lowest cost path to the Root Bridge. Forwards traffic.
2. Designated Port (DP) The port on a segment/link that has the lowest cost to the Root. Forwards traffic. (All ports on the Root Bridge are DPs).
3. Non-Designated / Blocking Any port that is not an RP or DP is put into a Blocking state to break the loop. Drops data traffic.

*Note: Path cost is calculated based on INCOMING ports towards the Root.

🪝 Brain Hooks

The Royal Family

Root Bridge: The King. Every port leaving his castle is a Designated Port (DP). He has no Root Ports because he IS the Root.
Root Port (RP): The fastest road back to the King's castle.

"Lower is Better"

In the world of Spanning Tree, think like golf. Lowest Priority wins. Lowest MAC wins. Lowest Path Cost wins.

⌨️ STP Core Commands

Switch# show spanning-tree The master command. Shows the Root Bridge ID, the local switch's Bridge ID, and the role/status/cost of every active port.
Switch(config)# spanning-tree vlan 10 root primary A macro command. The switch checks the current root's priority and automatically sets its own priority significantly lower (usually 24576) to guarantee it becomes the Root for VLAN 10.
Switch(config)# spanning-tree vlan 10 priority 4096 Manually hardcodes the bridge priority. MUST be a multiple of 4096.
Switch(config-if)# spanning-tree cost 15 Manually changes the STP cost of an interface to manipulate which path is chosen as the Root Port.

🔧 Troubleshooting

"Who is the Root?"

Action: Run show spanning-tree. Look at the top block ("Root ID"). If it says "This bridge is the root", you've found it! If not, it lists the MAC of the actual root.

Wrong Switch is Root

Issue: An old, slow switch has the oldest MAC address and won the election by default because all priorities were 32768.

Fix: Lower the priority on your core switch using the root primary command.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Why does a broadcast frame loop infinitely in a redundant Layer 2 network without STP, whereas a Layer 3 packet would eventually be dropped? ▼
Because Layer 2 Ethernet frames do not have a TTL (Time To Live) field. Unlike IP packets which are dropped when TTL hits 0, an L2 frame will cycle endlessly.
2. SW1 and SW2 both have the default Bridge Priority of 32768. SW1's MAC is 00:00:11:11:11:11. SW2's MAC is 00:00:00:AA:AA:AA. Which becomes the Root Bridge? ▼
SW2 wins the election. Because the priorities tie, STP uses the MAC address as a tie-breaker. SW2 has the lowest MAC address.
3. According to the IEEE standard, what is the default STP cost of a 100 Mbps (FastEthernet) link? ▼
The cost is 19. (Gigabit is 4, 10-Gig is 2).
4. How many Root Ports (RP) will you find on the Root Bridge? ▼
Zero. A Root Port points *towards* the Root. Because the Root Bridge is already the Root, all of its active ports become Designated Ports (DPs).
