Day 34: Standard ACLs

Access Control Lists & Traffic Filtering

📋

Standard ACL Basics

An Access Control List (ACL) is a sequential list of permit or deny statements that apply to IP addresses. It acts as a stateless firewall inside the router.

The Golden Limitation

Standard ACLs can filter traffic based on Source IP Address ONLY. They cannot check destination IPs, and they cannot check port numbers (TCP/UDP).

Numbered Ranges

When creating a Standard ACL, you must use a specific ID number so the router knows its type:

  • 1 - 99
  • 1300 - 1999 (Expanded range)

Inbound vs Outbound

Once created, an ACL must be applied to an interface in a specific direction.

  • IN: Traffic is filtered before the router makes a routing decision. Saves CPU if traffic is denied.
  • OUT: Traffic is filtered after the routing decision, right before it leaves the port.
📍

The Rule of Placement

Because Standard ACLs only filter based on the Source IP, where you apply them on the network is absolutely critical.

Place Standard ACLs as close to the
Destination as possible.

Why? Imagine you want to block PC1 from reaching Server A. If you place a Standard ACL on the router interface closest to PC1 (the source), it will block PC1's traffic. But since the ACL only checks the Source IP, it blocks PC1's traffic to everywhere, including the internet and other servers!

By placing it close to Server A (the destination), PC1 is only blocked when it tries to enter the specific network where Server A resides. It can still reach everywhere else.

⏳ Processing Rules

Top-Down Processing

The router reads ACL statements from top to bottom. First match wins. Once a packet matches a statement, the router executes the action (permit/deny) and stops reading the rest of the list.

Rule: Always put your most specific statements at the top, and general statements at the bottom.

The Implicit Deny

Every single ACL has an invisible, un-deletable statement at the very bottom: deny any.

If a packet makes it through your entire list and hasn't matched anything, it is dropped. Therefore, every ACL must have at least one `permit` statement, or it will block 100% of traffic.

🃏 Wildcards & Keywords

ACLs use Wildcard Masks to define how much of an IP address must match. (0 = Must Match, 255 = Don't Care).

The 'host' Keyword

Used to match one exact, specific IP address. Replaces the wildcard mask `0.0.0.0`.

access-list 10 permit 192.168.1.50 0.0.0.0

access-list 10 permit host 192.168.1.50

The 'any' Keyword

Used to match absolutely anything. Replaces the IP/Mask combo `0.0.0.0 255.255.255.255`.

access-list 10 permit 0.0.0.0 255.255.255.255

access-list 10 permit any

🪝 Brain Hooks

The VIP Bouncer (Top-Down)

An ACL is a bouncer reading a VIP list from top to bottom. If your name is at the top as "Deny", you are thrown out immediately. He doesn't read the rest of the list. If you aren't on the list at all, the club is private (Implicit Deny), and you are thrown out.

Standard = Destination

Standard ACLs go near the Destination.
(Extended ACLs go near the Source).

⌨️ Configuration & Application CLI

Step 1: Create the List (Global Config) Router(config)# access-list 10 deny host 192.168.1.50 Router(config)# access-list 10 permit 192.168.1.0 0.0.0.255 Denies one specific host, but permits the rest of the /24 subnet. (Everything else is implicitly denied).
Step 2: Apply to an Interface (Sub-Config) Router(config)# interface g0/1 Router(config-if)# ip access-group 10 out CRITICAL: Creating an ACL does nothing until it is applied to an interface using `ip access-group` and a direction (in/out).
Router# show access-lists Displays all configured ACLs, their statements, and the number of times a packet has "matched" each statement.

🔧 Troubleshooting

"I permitted one host, but now nobody else can connect!"

Fix: You forgot the Implicit Deny. If you create an ACL with just `permit host 10.1.1.5`, every other IP hits the invisible `deny any` at the bottom. Add access-list X permit any to the bottom of your list.

Wrong Order

Symptom: You wrote `permit any` first, and then `deny host 10.1.1.5` second. 10.1.1.5 can still get through.

Fix: Top-down processing! 10.1.1.5 matched `permit any` and the router stopped reading. Specifics must go at the top.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. According to Cisco best practices, where should a Standard ACL be placed in the network topology? ▼
As close to the Destination as possible. (Because it filters based on source IP, putting it near the source would block that host from reaching any other part of the network).
2. What is the ONLY parameter a Standard ACL can use to filter traffic? ▼
The Source IP address.
3. You create an ACL with a single statement: `access-list 10 permit 192.168.1.0 0.0.0.255`. What happens to a packet from 10.0.0.5? ▼
It is dropped. It does not match the permit statement, so it falls through to the invisible Implicit Deny statement at the end of the list.
4. Write the interface configuration command to apply Standard ACL number 20 to filter incoming traffic. ▼
ip access-group 20 in
