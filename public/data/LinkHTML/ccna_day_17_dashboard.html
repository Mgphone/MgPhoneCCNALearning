Day 17: VLANs (Pt 2)

Trunking, 802.1Q & DTP Mastery

🛣️

The Trunk Port

While Access ports carry traffic for only one VLAN, a Trunk port carries traffic for multiple VLANs simultaneously over a single physical link.

The Use Case

Used for links between Switches, or between a Switch and a Router (Router-on-a-Stick). Without trunks, you would need a separate physical cable for every single VLAN between two switches!

How does the receiving switch know?

If a trunk carries frames from VLAN 10 and VLAN 20, they get mixed together on the wire. The sending switch must "tag" the frame so the receiving switch knows which VLAN it belongs to.

🏷️

802.1Q Frame Tagging

IEEE 802.1Q (often called "dot1q") is the industry standard for VLAN tagging. It inserts a 4-Byte (32-bit) tag into the original Ethernet frame.

802.1Q TAG
TYPE
DATA (IPv4)
FCS
Insertion Point The tag is inserted after the Source MAC and before the Type/Length field. Because the frame was modified, a NEW FCS must be calculated.
The VLAN ID (VID) Inside the 4-byte tag is a 12-bit field for the VLAN ID. $2^{12} = 4096$. Therefore, you can have VLANs numbered from 1 to 4094.
👻

The Native VLAN

By default, 802.1Q tags every frame... except for one specific VLAN.

  • Definition: The Native VLAN is the VLAN on a trunk port that carries untagged traffic.
  • Default: VLAN 1 is the default Native VLAN on all Cisco switches.
  • The Mismatch Rule: The Native VLAN MUST MATCH on both ends of the trunk. If SW1 says Native=1 and SW2 says Native=99, Spanning Tree loops can form, and CDP will flood the console with angry warning messages.
  • Security: Best practice is to change the Native VLAN to an unused, "dead" VLAN (like 999) to prevent VLAN hopping attacks.

🤝 DTP (Dynamic Trunking Protocol)

DTP is a Cisco-proprietary protocol that actively negotiates whether a link should become an access port or a trunk port.

Mode Auto Desirable Trunk (On)
Auto (Passive) ACCESS TRUNK TRUNK
Desirable (Active) TRUNK TRUNK TRUNK

* Modern Cisco switches usually default to "Dynamic Auto". Two Autos will NEVER form a trunk!

🪝 Brain Hooks

The DTP Dating Scene

Desirable = Actively asking for a date. Will date anyone who says yes (Auto) or asks them back (Desirable).
Auto = Too shy to ask. Will say yes if asked, but if two Autos sit together, nobody speaks (No Trunk).

The Native VLAN = Naked

Think of Native VLAN frames as naked—they have no 802.1Q tag clothing them as they travel across the trunk link.

⌨️ Trunk Configuration CLI

Switch(config-if)# switchport mode trunk Hardcodes the interface to be a trunk. (Disables DTP negotiation).
Switch(config-if)# switchport trunk allowed vlan add 10 Adds VLAN 10 to the permitted list. CRITICAL: If you forget the word 'add', you will OVERWRITE the entire list and kill all other VLAN traffic on the trunk!
Switch(config-if)# switchport trunk native vlan 99 Changes the Native VLAN from 1 to 99. (Must be done on both ends).
Switch(config-if)# switchport nonegotiate Turns off DTP frames completely. Port must be statically set to access or trunk.
Switch# show interfaces trunk The most important Trunk command. Shows all active trunk ports, their Native VLAN, and which VLANs are allowed/forwarding across them.

🔧 Troubleshooting

"Native VLAN Mismatch" CDP Error

Symptom: Console gets flooded with %CDP-4-NATIVE_VLAN_MISMATCH.

Fix: Run show interfaces trunk on both switches. Ensure switchport trunk native vlan X matches on both sides of the link.

VLANs not crossing the trunk

Fix: Check the "Vlans allowed on trunk" section in show interfaces trunk. Did someone overwrite the list by forgetting the add keyword?

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. By definition, what type of traffic does the Native VLAN carry over an 802.1Q trunk link? ▼
It carries untagged traffic. (Frames belonging to the native VLAN do not get a 4-byte 802.1Q tag).
2. Switch A is set to DTP "dynamic auto" and Switch B is set to DTP "dynamic auto". Will a trunk form between them? ▼
No. Because both are passively waiting for the other to initiate the trunk, they will fall back and form an Access link instead.
3. You have an active trunk passing VLANs 10, 20, and 30. You want to permit VLAN 40. What critical keyword MUST you include in the `switchport trunk allowed vlan` command? ▼
The `add` keyword. (switchport trunk allowed vlan add 40). If you omit it, VLANs 10, 20, and 30 will be deleted from the list!
4. Exactly how many bytes does the 802.1Q tag add to an Ethernet frame? ▼
4 Bytes (32 bits). Because this changes the frame length and contents, the switch must recalculate the FCS at the end of the frame.
