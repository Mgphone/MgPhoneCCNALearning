Day 29: FHRP

First Hop Redundancy Protocols (HSRP, VRRP, GLBP)

⚠️

The Core Problem

PCs are typically configured with a single Default Gateway IP address. If that specific router goes offline, the PC loses all access to external networks, even if another physical router exists on the same LAN.

The FHRP Solution

FHRPs group multiple physical routers together into a single Virtual Router. They share a Virtual IP (VIP) and a Virtual MAC address.

How it Works

  • You configure the PCs to point to the Virtual IP (VIP).
  • One physical router acts as the "Active" forwarder.
  • If the Active router dies, the "Standby" router instantly takes over the VIP and Virtual MAC. The PCs never know a failure occurred!
🔱

The Big Three Protocols

1. HSRP

Hot Standby Router Protocol

  • Cisco Proprietary. (Most heavily tested on CCNA).
  • Active / Standby roles.
  • Uses Multicast: 224.0.0.2 (v1) or 224.0.0.102 (v2).

2. VRRP

Virtual Router Redundancy Protocol

  • IEEE Standard (Open).
  • Master / Backup roles.
  • Can use the physical IP of the Master as the Virtual IP.
  • Uses Multicast 224.0.0.18.

3. GLBP

Gateway Load Balancing Protocol

  • Cisco Proprietary.
  • Active / Active! (True load balancing).
  • Uses one Virtual IP, but hands out different Virtual MACs to different PCs.

🔥 HSRP Mechanics

How does HSRP choose which router becomes the Active forwarder?

1. Priority (Highest Wins)

Default priority is 100. The router with the highest priority becomes Active. If there is a tie, the router with the highest physical IP address wins.

2. Preemption (Critical!)

By default, HSRP preemption is Disabled. If Router A (Priority 150) dies, Router B (Priority 100) takes over. When Router A comes back online, it will NOT take its job back! You must manually configure standby preempt to allow it to reclaim the Active role.

🪪 Virtual MAC Formats

When a PC ARP requests the Default Gateway's Virtual IP, the Active router replies with a specific Virtual MAC address. You must be able to identify these on sight!

HSRP Version 1

0000.0c07.acXX

Where XX is the HSRP Group Number in Hexadecimal.

(e.g., Group 10 = 0000.0c07.ac0A)

HSRP Version 2

0000.0c9f.fXXX

Where XXX is the HSRP Group Number in Hexadecimal.

(e.g., Group 10 = 0000.0c9f.f00a)

🪝 Brain Hooks

"AC" = "ACtive"

To memorize the HSRPv1 MAC address format, remember that it ends in acXX. "AC" stands for Active!

Preemption = Stealing the Crown

Without Preemption, the king is dead, long live the new king. The old king can never return to the throne. With Preemption, the old king boots the backup off the throne the moment he returns.

⌨️ HSRP Configuration CLI

Router(config-if)# standby version 2 Changes from the default HSRP v1 to v2. Must be done before applying other standby commands.
Router(config-if)# standby 10 ip 192.168.1.254 Enables HSRP Group 10 and sets the Virtual IP Address that the PCs will point to.
Router(config-if)# standby 10 priority 150 Increases the priority (Default is 100) to force this router to become the Active forwarder.
Router(config-if)# standby 10 preempt Allows this router to forcefully take back the Active role if it reboots and comes back online.
Router# show standby brief Master verification command. Shows Group, Priority, State (Active/Standby), Active Router IP, Standby IP, and Virtual IP.

🔧 Troubleshooting HSRP

Both Routers think they are "Active"

Symptom: show standby brief shows 'State: Active' on BOTH routers simultaneously (a Split-Brain scenario).

Fix: They cannot hear each other's Multicast Hello packets. Check Layer 1/2 connectivity between them, or ensure no ACLs are blocking UDP port 1985 / Multicast IP 224.0.0.2.

Traffic flow didn't revert after recovery

Symptom: R1 (Priority 150) failed, R2 (Priority 100) took over. R1 is fixed and back online, but R2 is still forwarding all the traffic.

Fix: You forgot to type standby 10 preempt on R1. By default, HSRP does not preempt.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. When setting up HSRP, what specific IP address must you configure all the end-user PCs to use as their Default Gateway? ▼
The Virtual IP (VIP) address. Do NOT point them to the physical interface IPs of the routers, or redundancy will fail.
2. What is the default priority for HSRP, and does the highest or lowest priority win the Active role? ▼
The default priority is 100. The highest priority wins. (In case of a tie, the highest physical IP address wins).
3. You capture an ARP reply containing the MAC address `0000.0c07.ac05`. What HSRP Version is this, and what is the Group Number? ▼
It is HSRP Version 1 (indicated by `acXX`), and the Group Number is 5 (05 in hex = 5 in decimal).
4. Which First Hop Redundancy Protocol is Cisco-proprietary and uniquely allows for true Active/Active load balancing out of the box? ▼
GLBP (Gateway Load Balancing Protocol).
