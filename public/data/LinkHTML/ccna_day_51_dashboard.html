Day 51: Dynamic ARP Inspection

Layer 2 Defense: ARP Poisoning & MitM

🦹‍♂️

The Threat: ARP Poisoning

ARP is a fundamentally trusting protocol. It has no built-in authentication. By default, devices believe any ARP reply they receive, even if they didn't ask for it!

How the Attack Works

An attacker sends a "Gratuitous ARP" broadcast to the entire subnet saying: "I am the Default Gateway (192.168.1.1), and my MAC address is [Attacker's MAC]!"

The Result: Man-in-the-Middle

Every PC on the subnet overwrites its ARP cache. When a PC tries to go to the internet, it sends the traffic to the attacker's MAC address. The attacker intercepts/reads the traffic, then quietly forwards it to the real router so the victim never notices.

The Solution: DAI

Dynamic ARP Inspection (DAI) intercepts all ARP packets on a switch port and validates them before forwarding them.

🔗

The Ultimate Dependency

How does the switch know if an ARP packet is lying about its IP-to-MAC mapping?

The Source of Truth

DAI does not have its own database! It relies entirely on the DHCP Snooping Binding Database created by the switch.

  • If DHCP Snooping isn't running, DAI drops almost all ARP traffic on untrusted ports!
  • DAI intercepts the ARP packet, compares the Sender MAC & IP to the DHCP Snooping database.
  • If they match: Forwarded.
  • If they mismatch: Dropped and Logged.

The Pipeline

1. DHCP Snooping

↓ creates ↓

2. Binding Database

↓ empowers ↓

3. DAI / IPSG

🧱 DAI Trust Boundaries

Just like DHCP Snooping, DAI uses the concept of Trusted and Untrusted ports—but the configuration commands are completely separate!

Untrusted Ports (Default)

All ports are untrusted when DAI is enabled. ARP packets arriving here are intercepted and rigidly checked against the DHCP Snooping Database.

Trusted Ports

ARP packets arriving here bypass all DAI checks and are forwarded normally. You MUST configure this on uplinks between switches, otherwise legitimate cross-switch ARP traffic gets dropped!

📌 The Static IP Dilemma

The Problem

If a printer has a Static IP, it never asks DHCP for an address. Therefore, it is NOT in the DHCP Snooping database. DAI will blindly drop all ARP packets from the printer!

The Solution: ARP ACLs

You must manually create an ARP Access-List to map the static IP to the MAC address, and tell DAI to check this ACL in addition to the DHCP snooping database.

🪝 Brain Hooks

The Nightclub Bouncer

DHCP Snooping is the person handing out wristbands at the door and keeping a guest list.
DAI is the bouncer inside checking IDs. He can't do his job unless he has a copy of the guest list from DHCP Snooping!

⌨️ Configuration CLI

Switch(config)# ip arp inspection vlan 10 Step 1: Enables DAI globally for a specific VLAN. (All ports in VLAN 10 immediately become Untrusted for ARP).
Switch(config)# interface g0/1 Switch(config-if)# ip arp inspection trust Step 2: Configures uplink/trunk ports as Trusted. (This is a separate command from `ip dhcp snooping trust`!).
Handling Static IPs (ARP ACL) Switch(config)# arp access-list PRINTERS Switch(config-arp-nacl)# permit ip host 10.1.1.5 mac host 0011.2233.4455 Switch(config)# ip arp inspection filter PRINTERS vlan 10 Creates an ARP ACL for static devices and applies it to the DAI process for VLAN 10.

🔧 Verification

show ip arp inspection Shows global DAI configuration, which VLANs are active, and packet drop statistics.
show ip arp inspection interfaces Quickly verifies which interfaces are configured as Trusted versus Untrusted.
Symptom: Valid IPs getting blocked Verify DHCP Snooping is running and the show ip dhcp snooping binding database is populated!

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. By sending a gratuitous ARP packet claiming to be the Default Gateway, an attacker is attempting to execute what type of attack? ▼
A Man-in-the-Middle (MitM) attack. They are redirecting all outbound traffic through their own device.
2. Dynamic ARP Inspection absolutely relies on the existence of which specific database table to function? ▼
The DHCP Snooping Binding Database. It uses this table to verify that an incoming ARP packet contains legitimate IP-to-MAC mapping.
3. Because devices with statically assigned IP addresses do not appear in the DHCP Snooping database, what must an admin configure to allow their ARP packets through DAI? ▼
An ARP Access-List (ARP ACL). This manually defines the valid IP-to-MAC mapping and is filtered by DAI.
4. True or False: If you configure a port with `ip dhcp snooping trust`, that port automatically becomes trusted for Dynamic ARP Inspection as well. ▼
False. They are distinct features. You must explicitly configure ip arp inspection trust on the interface.
