Day 36: CDP & LLDP

Layer 2 Discovery Protocols

📡

Layer 2 Discovery

Discovery protocols allow network devices to advertise their identity and capabilities to their directly connected neighbors. They are invaluable tools for mapping out undocumented network topologies.

Layer 2 Operation

These protocols operate strictly at Layer 2 (Data Link). This means they do NOT require IP addresses to function! Even if two routers have completely mismatched IP subnets, they can still discover each other.

Information Shared

  • Hostname: The name of the neighbor device.
  • Local Interface: The port on your device receiving the data.
  • Port ID: The port on the neighbor's device sending the data.
  • Device Capability: Is it a Router, Switch, IP Phone, or AP?
  • Platform: The specific hardware model and OS version.
  • IP Addresses: The management IPs of the neighbor (even though the protocol runs at L2).
⚠️

The Security Risk

Discovery protocols are a double-edged sword. While they help admins map the network, they hand attackers a silver platter of reconnaissance data.

Best Practice

Never run CDP or LLDP on edge ports (ports connected to PCs, servers, or untrusted devices). An attacker can run Wireshark, listen to the multicast CDP frames, and immediately know your switch's management IP, OS version, and exact port number.

Only enable discovery protocols on switch-to-switch or switch-to-router links.

🥊 CDP vs. LLDP

CDP (Cisco Discovery Protocol)

  • Ownership: Cisco Proprietary.
  • Status: Enabled globally by default on Cisco gear.
  • Hello Timer: 60 seconds.
  • Hold Timer: 180 seconds.
  • Use Case: Communicating with Cisco IP Phones (Voice VLAN assignment, PoE power negotiation).

LLDP (Link Layer Discovery Protocol)

  • Ownership: Open Standard (IEEE 802.1AB).
  • Status: Usually disabled globally by default on Cisco gear.
  • Hello Timer: 30 seconds.
  • Hold Timer: 120 seconds.
  • Use Case: Multi-vendor environments. Discovering non-Cisco servers, VMs, or switches.

🪝 Brain Hooks

"Enable" vs "Transmit/Receive"

When configuring an interface:
CDP uses one command: cdp enable.
LLDP uses two: lldp transmit & lldp receive. (You can set it to listen only!).

The Timers Rule

Hold timers are generally 3x to 4x the Hello timer. CDP sends every minute, holds for 3. LLDP sends every 30s, holds for 120s (4x).

⌨️ Discovery Configuration CLI

Global Config (Turn on/off entire switch) Switch(config)# no cdp run Disables CDP globally.
Global Config Switch(config)# lldp run Enables LLDP globally (disabled by default).
Interface Config (CDP) Switch(config-if)# no cdp enable Disables CDP on a specific port (Security best practice).
Interface Config (LLDP) Switch(config-if)# lldp transmit Switch(config-if)# lldp receive LLDP direction must be enabled separately.

🔎 Verification CLI

show cdp neighbors Quick summary of attached devices, their capability (R, S, I), and the local/remote ports.
show cdp neighbors detail Crucial! Shows the IP Address and specific iOS version of the neighboring device.
show lldp neighbors The LLDP equivalent of the summary command.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. You have a Cisco switch connected to a Juniper router. Which discovery protocol must you use to map this connection? ▼
LLDP (Link Layer Discovery Protocol), because it is an open IEEE 802.1AB standard. CDP is Cisco-proprietary.
2. At which OSI layer do CDP and LLDP operate? ▼
Layer 2 (Data Link Layer). They use multicast MAC addresses and do not require IP connectivity to function.
3. You run `show cdp neighbors` but it only lists hostnames and interfaces. Which command will reveal the actual IP Address of the neighbor? ▼
show cdp neighbors detail (or show cdp entry *).
4. What is the difference between the commands `no cdp run` and `no cdp enable`? ▼
no cdp run is a Global Configuration command that turns CDP off for the entire switch. no cdp enable is an Interface Configuration command that disables CDP on a single, specific port.
