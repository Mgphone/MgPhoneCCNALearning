Step 1: The Host Decision

Before a PC sends a packet, it compares the Destination IP to its own IP and Subnet Mask. This determines where the L2 Frame is sent.

Scenario A: Same Subnet

The destination is local. The PC checks its ARP Cache for the Destination PC's MAC address.

L2 Dst MAC: Target PC's MAC
Scenario B: Different Subnet

The destination is remote. The PC checks its ARP Cache for the Default Gateway's (Router's) MAC address.

L2 Dst MAC: Router's MAC
ARP Missing?
If the MAC isn't in the cache, the PC pauses the packet and broadcasts an ARP Request: "Who has IP x.x.x.x? Tell me your MAC!"
⚖️

The Golden Rule of Encapsulation

As a packet travels from PC1 (Source) through multiple Routers to PC2 (Destination), observe how the addresses behave:

Layer 3 (IP Addresses)

Are End-to-End.

The Source IP and Destination IP NEVER CHANGE throughout the entire journey. They identify the original sender and final receiver.

Layer 2 (MAC Addresses)

Are Hop-by-Hop.

The Source MAC and Destination MAC CHANGE AT EVERY ROUTER. They only identify the local jump across the current physical link.

📝

The Router's Internal Checklist

When a router receives a frame addressed to its MAC address, it performs these exact steps in order:

1. Verify the incoming Frame Check Sequence (FCS) for errors. If corrupt, drop it.
2. Verify the Destination MAC matches the router's receiving interface.
3. De-encapsulate (strip off) the Layer 2 Ethernet Header and Trailer.
4. Check the Routing Table for the Destination IP Address (Longest Prefix Match).
5. Decrement the TTL (Time To Live) by 1 in the IPv4 Header. (If TTL = 0, drop and send ICMP Time Exceeded).
6. Re-encapsulate into a NEW Layer 2 Frame.
New Src MAC = Router's Exit Interface MAC. New Dst MAC = Next-hop Router's MAC (via ARP).
7. Calculate a NEW FCS and forward out the exit interface.

🪝 Brain Hooks

The Envelope & The Truck

IP Addresses: The "To" and "From" written in ink on the letter itself. (Never changes).
MAC Addresses: The FedEx truck carrying the letter. The letter changes trucks at every distribution center (Router) until it reaches the final house.

Switches vs Routers

Switches do NOT change MAC addresses. They just forward the existing frame.
Routers DO rewrite MAC addresses because they are bridging entirely different L2 domains.

⌨️ Tracking the Packet (CLI)

C:\> arp -a Windows command: Views the PC's ARP cache (IP-to-MAC mappings). Crucial for checking if the PC knows the Default Gateway's MAC.
Router# show ip arp Cisco IOS command: Views the router's ARP cache. If a router needs to send a packet to a local PC but doesn't have its MAC here, it will send an ARP request.
Router# clear arp-cache Forces the router to delete all dynamic ARP entries and learn them again. Useful if a device's NIC was recently replaced.

🔧 Troubleshooting

Ping fails to remote subnet

Action 1: Check the PC's Default Gateway IP (ipconfig). Is it correct?

Action 2: Check the PC's ARP cache (arp -a). Did it successfully resolve the Gateway's MAC address? If not, check the local switch/VLAN.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. PC1 (10.0.0.5) sends a packet to PC2 (192.168.1.10). What Destination MAC address will PC1 place on the Layer 2 frame? ▼
Because PC2 is on a different subnet, PC1 must send the frame to its Default Gateway. Therefore, the Destination MAC will be the Router's Interface MAC address.
2. As a packet crosses a router from one LAN to another, which addresses change, and which stay the same? ▼
The Source and Destination IP addresses stay exactly the same (end-to-end). The Source and Destination MAC addresses change at every router hop.
3. What does a router do if it needs to forward a packet to a directly connected PC, but the PC's MAC address is missing from the router's ARP table? ▼
It holds the packet and sends out an ARP Request broadcast on that local interface, asking "Who has this IP address? Send me your MAC."
4. Inside the L3 IPv4 Header, what field does the router actively change before re-encapsulating and forwarding the packet? ▼
The router decrements the TTL (Time to Live) field by 1. (It also updates the Header Checksum to reflect the new TTL value).
