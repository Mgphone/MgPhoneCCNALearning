What is a VLAN?

A Virtual Local Area Network (VLAN) logically segments a single physical switch into multiple virtual switches.

Broadcast Domain Boundaries

By default, all ports on a switch are in the same broadcast domain. VLANs create separate broadcast domains at Layer 2. A broadcast sent in VLAN 10 will NEVER be forwarded out a port in VLAN 20.

Layer 3 Requirement

Because devices in different VLANs are essentially on completely different networks, they cannot communicate with each other without a Layer 3 device (a Router or a Layer 3 Switch) to route between them.

One VLAN = One Subnet

Golden rule of network design: Every VLAN should correspond to its own unique IPv4/IPv6 subnet (e.g., VLAN 10 = 10.1.10.0/24, VLAN 20 = 10.1.20.0/24).

✅

Why use VLANs?

  • 🛡️
    Security Isolate sensitive devices. If HR and Guest PCs are on the same switch, placing them in different VLANs prevents Guests from sniffing HR traffic.
  • 🚀
    Performance Reduces the size of broadcast domains. Fewer devices hearing useless ARP/DHCP broadcasts means less wasted CPU processing.
  • 🧩
    Flexibility & Design Group users by function (e.g., Sales, IT) rather than physical location. A Sales PC on Floor 1 and Floor 3 can both be in VLAN 20.
🔌

Access Ports & VLAN 1

VLAN 1 (The Default)

Out of the box, every port on a Cisco switch is assigned to VLAN 1. It cannot be deleted or renamed. Security best practice: Move unused ports to a different "dead" VLAN and shut them down.

Access Ports

An interface configured as an access port belongs to exactly ONE data VLAN. It connects directly to end devices (PCs, servers, printers). The PC has no idea VLANs exist; the switch strips the VLAN tag before sending the frame to the PC.

☎️

Voice VLANs (The Exception)

Generally, an access port belongs to ONE VLAN. However, IP Phones create a special scenario. To save wiring, a PC connects to the IP Phone, and the IP Phone connects to the switch.

How it Works

The switch port is configured as an access port, but we add a Voice VLAN command. The port now technically carries TWO VLANs:

  • The Data VLAN (untagged) for the PC.
  • The Voice VLAN (tagged with 802.1Q) for the phone.

CDP Magic

The Cisco switch uses CDP (Cisco Discovery Protocol) to tell the Cisco IP Phone which VLAN ID to use for its voice traffic. The phone then actively tags its own frames before sending them to the switch.

🪝 Brain Hooks

The Apartment Building

Think of a switch as an apartment building. A VLAN is a specific floor. People on Floor 10 (VLAN 10) can shout (broadcast) and hear each other, but Floor 20 can't hear them. To visit another floor, you must use the elevator (a Router).

Voice VLAN = The Backpack

The phone acts as a tiny 3-port switch. It carries the PC's traffic "in its backpack" (untagged) while labeling its own voice traffic (tagged) so the switch can prioritize it.

⌨️ VLAN Configuration CLI

Switch(config)# vlan 10 Switch(config-vlan)# name SALES Creates VLAN 10 and names it "SALES". (If you assign a port to a VLAN that doesn't exist, IOS will automatically create it with a default name, but manual creation is best practice).
Switch(config-if)# switchport mode access Hardcodes the interface to be an access port (turns off dynamic trunking).
Switch(config-if)# switchport access vlan 10 Assigns the access port to data VLAN 10.
Switch(config-if)# switchport voice vlan 20 Tells the connected IP phone to tag its voice traffic for VLAN 20.
Switch# show vlan brief The most important VLAN command! Lists all VLANs, their names, and exactly which active ports are assigned to them.

🔧 Troubleshooting

"PC can't ping default gateway"

Action: Run show vlan brief. Did you forget to assign the PC's switch port to the correct VLAN? If it's still in VLAN 1, it won't reach the router interface in VLAN 10.

Deleted VLAN blackhole

Warning: If you delete a VLAN globally, but ports are still assigned to it, those ports go "inactive" and pass zero traffic until reassigned.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. By placing a group of switch ports into VLAN 20, you are effectively creating a boundary for which type of domain? ▼
The Broadcast Domain. A broadcast sent by a device in VLAN 20 will only be flooded to other ports in VLAN 20.
2. If PC-A is in VLAN 10 and PC-B is in VLAN 20 on the same physical switch, what piece of equipment is strictly required for them to ping each other? ▼
A Layer 3 device (like a Router or a Layer 3 Switch). Because they are in different broadcast domains (different subnets), traffic must be routed.
3. Which command is the fastest way to verify which interfaces are assigned to which VLANs? ▼
show vlan brief
4. When configuring an access port to support both a PC and a Cisco IP Phone, how many VLANs are effectively traversing that single port? ▼
Two. The port carries an untagged Data VLAN for the PC, and a tagged Voice VLAN for the IP Phone.
