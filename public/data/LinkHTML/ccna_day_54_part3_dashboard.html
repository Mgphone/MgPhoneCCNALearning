Day 54: Virtualization (Pt 3)

Virtual Routing and Forwarding (VRF)

🔀

The Power of VRF

Virtual Routing and Forwarding (VRF) is a technology that allows multiple instances of a routing table to co-exist within the same router simultaneously.

Layer 3 Isolation

By default, a router has one Global Routing Table. Everything connects to it. VRF slices that router into multiple logical routers. Traffic inside "VRF RED" cannot be routed to "VRF BLUE" because they literally do not share a routing table.

VRF-Lite vs Full VRF

VRF-Lite is what we study in CCNA/CCNP. It is simply creating VRFs on a local router to segment traffic (e.g., isolating Guest Wi-Fi from Corporate). Full VRF is used by ISPs running MPLS and BGP to separate different customers' traffic across a massive WAN.

The Golden Rule of Isolation Without special configuration (called Route Leaking), it is mathematically impossible for a packet to jump from one VRF into another. This provides hardware-level security segmentation.
👯

Overlapping IP Addresses

Normally, a Cisco router will immediately reject your configuration if you try to assign the same IP subnet to two different interfaces (e.g., % 10.1.1.0 overlaps with GigabitEthernet0/1).

The ISP Scenario

Imagine you are an ISP. Customer A and Customer B both decide to use the default 192.168.1.0/24 subnet for their offices. They both connect to your PE (Provider Edge) router.

Without VRF, your router would crash trying to figure out which interface "192.168.1.5" belongs to.

The VRF Solution

By assigning Customer A's interface to VRF CUST_A and Customer B's interface to VRF CUST_B, the router maintains two completely separate logical realities.

You can now legally configure 192.168.1.1/24 on BOTH interfaces simultaneously without any overlap errors!

🛑 The Configuration Gotcha

There is a very specific behavior you must memorize for exams and real-world networking regarding VRF assignment.

When you assign a physical (or virtual) interface to a VRF using the ip vrf forwarding command, Cisco IOS will instantly delete the IPv4 and IPv6 addresses currently configured on that interface.

% Interface GigabitEthernet0/1 IPv4 disabled and address(es) removed due to enabling VRF RED

Why? The router is moving the interface from the Global Routing Table into the new VRF table. To prevent routing corruption during the move, it strips the IP. You must re-type your ip address command immediately afterward.

🔄 Legacy vs. Modern Syntax

Cisco IOS has two different ways to configure VRFs. You should recognize both, but understand why the modern way exists.

Legacy (IPv4 Only)

Router(config)# ip vrf RED Router(config-if)# ip vrf forwarding RED

The old `ip vrf` commands only created routing tables for IPv4. If you tried to route IPv6 through this, it wouldn't work.

Modern (Multi-Protocol)

Router(config)# vrf definition RED Router(config-vrf)# address-family ipv4 Router(config-vrf)# address-family ipv6 Router(config-if)# vrf forwarding RED

The modern `vrf definition` command creates a container that can hold isolated routing tables for BOTH IPv4 and IPv6 simultaneously.

🪝 Brain Hooks

VLAN vs. VRF

VLAN: Slices a Switch into multiple isolated Layer 2 Broadcast Domains.

VRF: Slices a Router into multiple isolated Layer 3 Routing Tables.

The "Blind" Router

If an interface is in VRF RED, and you just type `ping 10.1.1.1` in the console, the router will fail. The global console is "blind" to the VRF tables. You must specifically tell the ping command to look inside the VRF folder!

⌨️ VRF Verification & Operations

Router# show ip route vrf RED Shows the routing table SPECIFICALLY for VRF RED. If you just type `show ip route`, it will only show you the Global routing table, and the VRF routes will be completely hidden!
Router# show ip vrf interfaces Excellent summary command. Lists all configured VRFs and shows exactly which physical/virtual interfaces are assigned to them.
Router# ping vrf RED 192.168.1.10 Executes a ping sourced from the VRF RED routing table.
Applying Routing Protocols to a VRF Router(config)# router ospf 1 vrf RED You must launch a dedicated instance of your routing protocol specifically tied to the VRF. This ensures OSPF only advertises the routes inside the RED table to other routers in the RED VRF.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. How is it possible to configure the exact same IP address (e.g., 10.0.0.1/24) on two different interfaces on the same Cisco router without receiving an overlap error? ▼
By placing the two interfaces into different VRFs. Because the routing tables are 100% isolated, the router doesn't see them as overlapping.
2. What happens immediately to an interface's IP configuration the moment you type `ip vrf forwarding [name]` on it? ▼
Cisco IOS instantly deletes the IP address currently configured on that interface to prevent routing corruption as it moves from the Global table to the VRF table. You must re-add the IP address afterward.
3. You type `show ip route` and cannot find the routes for your newly configured Guest network. What command must you type to see them? ▼
show ip route vrf [name]. The standard command only shows the global routing table.
4. What is the primary advantage of using the modern `vrf definition [name]` command over the legacy `ip vrf [name]` command? ▼
The modern `vrf definition` command supports multiple address families, allowing you to route both IPv4 and IPv6 within the same VRF instance.
