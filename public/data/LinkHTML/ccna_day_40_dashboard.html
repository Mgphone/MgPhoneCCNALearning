Day 40: SNMP

Simple Network Management Protocol

👁️

SNMP Components

SNMP is an Application Layer protocol used to monitor, configure, and manage network devices remotely. It consists of four core components.

1. NMS (Network Management Station)

The centralized server/software (like SolarWinds or PRTG) that polls devices for information and displays dashboards.

2. SNMP Agent

The software module running on the managed device (the router or switch) that gathers data and answers the NMS's questions.

3. MIB (Management Information Base)

A virtual, hierarchical database residing on the agent. It stores the actual statistics and variables of the device.

4. OID (Object Identifier)

A specific numeric path pointing to a single variable in the MIB tree (e.g., 1.3.6.1.2.1.2.2.1.8 might point to an interface's operational status).

✉️

SNMP Message Types

How the NMS and Agent communicate with each other.

Polled (NMS -> Agent)

  • Get Request: The NMS asks the Agent for a specific variable value.
  • GetNext / GetBulk: The NMS asks for the next OID in the tree, or a large chunk of data at once (GetBulk added in v2c).
  • Set Request: The NMS tells the Agent to CHANGE a variable (e.g., shutting down a port).

Pushed (Agent -> NMS)

Highly Tested!
  • Trap: An unsolicited alert sent by the Agent when an event happens (e.g., a link goes down). It is unreliable—the Agent doesn't care if the NMS receives it.
  • Inform: Similar to a Trap, but it is reliable. The Agent expects an Acknowledgment from the NMS. If none is received, it resends the Inform. (Added in v2c).

🛡️ SNMP Versions

SNMP has evolved drastically, primarily to address security flaws.

SNMPv1

Legacy. Uses simple Community Strings (like a password) to authenticate. Extremely insecure because strings are sent in clear text.

SNMPv2c

Most common. Still uses clear text community strings, making it insecure. The "c" stands for Community-based. Added GetBulk and Informs.

SNMPv3

The modern, secure standard. Provides three key security features:

  • Authentication: Uses HMAC-MD5 or SHA to verify the user.
  • Encryption: Uses DES, 3DES, or AES to encrypt the payload.
  • Integrity: Ensures packets aren't altered in transit.

🔑 Ports & Community Strings

UDP Ports (161 vs 162)

SNMP utilizes two distinct UDP ports depending on who is initiating the communication.

  • UDP 161: The port the Agent (Router/Switch) listens on. The NMS sends Get/Set requests to this port.
  • UDP 162: The port the NMS Server listens on. The Agent sends Traps and Informs to this port.

Community Strings (v1 & v2c)

Community strings act as shared passwords between the NMS and Agent. There are two types:

  • Read-Only (RO): Allows the NMS to view variables (Get). Default is often mistakenly left as "public".
  • Read-Write (RW): Allows the NMS to change variables (Set). Highly dangerous if compromised. Default is often "private".

🪝 Brain Hooks

Traps vs Informs

Trap: Sending a text message. You assume they got it, but don't know for sure.

Inform: Sending a letter via Certified Mail. The recipient must sign for it (ACK), or you will send it again.

The MIB Menu

The MIB is like a restaurant menu, and the OID is the specific dish number (e.g., #42 is the CPU load). The NMS orders dish #42.

⌨️ Cisco SNMP Config

Basic SNMPv2c Setup: Router(config)# snmp-server community MYSECRET ro Sets the read-only community string to "MYSECRET".
Setting Contact Info (Optional but recommended): Router(config)# snmp-server location DataCenter_Rack2 Router(config)# snmp-server contact admin@company.com These values populate standard OIDs that the NMS can read.
Enabling Traps to an NMS: Router(config)# snmp-server enable traps Router(config)# snmp-server host 10.1.1.50 version 2c MYSECRET Turns on traps, and tells the router to send them to the NMS at 10.1.1.50 using v2c.

🔧 Troubleshooting

NMS cannot graph the router

Action 1: Verify the community string. It is case-sensitive! (e.g., public != Public).

Action 2: Check any ACLs applied to the router's interfaces. Ensure you are permitting UDP port 161 inbound from the NMS IP address.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. What is the fundamental difference between an SNMP Trap and an SNMP Inform? ▼
A Trap is unreliable; the agent sends it and forgets it. An Inform is reliable; the agent expects the NMS to acknowledge receipt, otherwise it resends it.
2. If a router (Agent) needs to send a Trap to the monitoring server (NMS), which UDP port will the packet be destined for? ▼
UDP Port 162. (The NMS listens on 162 for traps/informs. The Agent listens on 161 for gets/sets).
3. What major security features were introduced in SNMPv3 that were completely absent in SNMPv1 and SNMPv2c? ▼
SNMPv3 introduced Authentication (verifying user credentials via hashes like MD5/SHA) and Encryption (encrypting the payload via AES/DES), eliminating clear-text community strings.
4. What is the name of the hierarchical, virtual database residing on the managed device that stores the operational statistics and variables? ▼
The MIB (Management Information Base). Specific variables within the MIB are identified by an OID (Object Identifier).
