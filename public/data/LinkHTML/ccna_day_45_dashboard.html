Day 45: NAT (Pt 2)

Dynamic Network Address Translation

🔄

The Dynamic Shift

Static NAT is excellent for servers but terrible for standard office employees. You don't want to manually map 500 individual PCs. Dynamic NAT automates the translation process.

Pool-Based Allocation

Instead of hardcoding IPs, you configure a range (a "Pool") of public IP addresses on the router. When an internal PC wants to access the internet, the router pulls an available public IP from the pool and assigns it dynamically.

Temporary Bindings

Unlike Static NAT, the mapping in the NAT table is temporary. Once the host finishes its session and the idle timer expires, the router clears the binding and returns the public IP to the pool.

Outbound Only Dynamic NAT relies on the internal host initiating the connection. An external user on the internet cannot reach inside to a PC because there is no static mapping telling the router where to send the inbound traffic.
⚙️

The NAT Lifecycle

How does the router process a packet requesting dynamic translation?

1

Traffic Arrival & Matching

A host sends a packet to the internet. The router intercepts it and checks its configured access-list. If the host's Source IP matches the ACL, it proceeds to translation.

2

Pool Selection & Translation

The router finds an unused Public IP from the ip nat pool. It logs the entry in the NAT table (`Inside Local` → `Inside Global`) and forwards the translated packet.

🛑 The "Gotchas" (Exam Focus)

Dynamic NAT has a massive constraint that you must understand for the CCNA.

  • The 1:1 Constraint

    It is a huge misconception that Dynamic NAT shares a public IP. It does not! Dynamic NAT is still strictly a 1-to-1 mapping. If you have 11 public IPs in your pool, only 11 users can access the internet simultaneously.

  • Pool Exhaustion

    If your pool has 10 IPs and the 11th user tries to open a webpage, the router will drop their traffic and log a "%NAT: Pool exhausted" error.

    This exact limitation is why we must use NAT Overload (PAT) in modern networks.

🪝 Brain Hooks

The Bowling Shoe Rental

Dynamic NAT is like a bowling alley renting out shoes (Public IPs). If they have 50 pairs of shoes, only 50 people can bowl. If a 51st person shows up, they must wait until someone leaves and returns a pair.

Interest & Resources

To configure Dynamic NAT, you need two things: The Interest (The Access-List defining who is allowed) and the Resource (The NAT Pool defining what IPs they get).

⌨️ Dynamic NAT Configuration CLI

Step 1: Define the range of available Public IPs Router(config)# ip nat pool PUBLIC_POOL 203.0.113.5 203.0.113.15 netmask 255.255.255.240 Creates a pool named 'PUBLIC_POOL' containing 11 usable public IPs.
Step 2: Define who is allowed to use those IPs Router(config)# access-list 10 permit 192.168.10.0 0.0.0.255 Creates a standard ACL permitting the entire internal 192.168.10.x subnet.
Step 3: Link the ACL to the Pool Router(config)# ip nat inside source list 10 pool PUBLIC_POOL Tells the router: "If traffic matches list 10, translate it using an IP from PUBLIC_POOL."
Step 4: Assign Interfaces (Crucial!) Router(config-if)# ip nat inside (on the LAN port) Router(config-if)# ip nat outside (on the WAN port)

🔧 Verification CLI

show ip nat translations

Displays the active temporary mappings currently in the router's memory.

show ip nat statistics

Highly useful. Shows the total number of hits, misses, and how many IPs are currently in use within your pool.

clear ip nat translation *

Danger: Destroys all current active NAT mappings, effectively disconnecting all current users from the internet. Use with caution.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Does Dynamic NAT allow multiple internal hosts to share a single public IP address simultaneously? ▼
No. Dynamic NAT still enforces a strict 1-to-1 ratio. It simply automates the assignment from a pool. To share a single IP, you must use NAT Overload (PAT).
2. Which two specific configuration components must be linked together in the `ip nat inside source...` command to make Dynamic NAT work? ▼
An Access Control List (list) to define who is permitted, and a NAT Pool (pool) to define what public IPs they are assigned.
3. What condition occurs when 50 internal hosts attempt to access the internet through a Dynamic NAT pool that contains only 10 public IP addresses? ▼
Pool Exhaustion. The first 10 users will get online, and the remaining 40 users will have their traffic dropped until an IP is freed up.
4. Which CLI command completely flushes all active dynamic NAT mappings from the router's memory? ▼
clear ip nat translation *
