Day 28: OSPF (Pt 3)

LSA Types, ABRs, ASBRs & External Routes

🎭

OSPF Router Roles

To scale efficiently, OSPF divides networks into Areas. Depending on where a router sits, it is assigned a specific role.

Internal Router

A router whose interfaces all belong to the same single area. It only maintains a Link-State Database (LSDB) for its own area.

Backbone Router

Any router that has at least one interface in Area 0. (All areas must connect to Area 0).

ABR (Area Border Router)

A router that connects one or more areas to the backbone (Area 0). It maintains a separate LSDB for each area it connects to and translates routing info between them.

ASBR (Autonomous System Boundary Router)

A router that connects the OSPF network to an external routing domain (like the Internet, EIGRP, or a statically routed network) and redistributes those routes into OSPF.

🗂️

LSA Types 1 to 3 (Internal)

Routers communicate by flooding Link State Advertisements. These build the LSDB.

Type 1: Router LSA

Generated by EVERY OSPF router.

  • Identifies the router's active interfaces, IP prefixes, and link states.
  • Flooded ONLY within its own specific area. It never crosses an ABR.

Type 2: Network LSA

Generated ONLY by the DR (Designated Router) on broadcast networks.

  • Lists all routers connected to that specific multi-access network.
  • Flooded ONLY within its own area.

Type 3: Summary LSA

Generated by the ABR.

  • Takes the internal routes (Type 1 & 2) from one area and advertises them into another area.
  • This is how Inter-Area routing is achieved.
👽

LSA Types 4 & 5 (External)

When a router connects to the outside world (ASBR), it must inform the rest of the OSPF domain.

Type 5: AS External LSA

Generated by the ASBR.

Advertises routes from outside the OSPF domain (e.g., an injected default route to the internet, or routes redistributed from EIGRP). Type 5 LSAs are flooded throughout the entire OSPF autonomous system (across all normal areas).

Type 4: ASBR Summary LSA

Generated by the ABR.

Type 5 LSAs contain the external route, but routers in other areas don't know how to physically reach the ASBR! The ABR generates a Type 4 LSA to say: "Hey, if you need to reach the ASBR, come to me."

📖 OSPF Route Codes

How OSPF routes appear in show ip route:

  • O
    Intra-Area Route (Inside your own area. Built from LSA 1 & 2).
  • O IA
    Inter-Area Route (In a different OSPF area. Built from LSA 3).
  • O E2 / O E1
    External Route (Outside OSPF. Built from LSA 5).

⚖️ O E1 vs O E2 Metrics

When external routes are injected into OSPF, they are tagged as Type 2 (E2) by default.

E2 (External Type 2 - Default)

The metric (cost) of the route stays exactly the same as it travels through the OSPF domain. The internal cost to reach the ASBR is ignored.

E1 (External Type 1)

The metric increases as it travels through the network. It adds the internal OSPF link cost to the external cost. Used when you have multiple ASBRs and want routers to choose the closest one.

⌨️ LSDB & Default Route Config

Router(config-router)# default-information originate Turns the router into an ASBR. If the router has a default route (`ip route 0.0.0.0 0.0.0.0 ...`) in its routing table, it will inject it into OSPF as a Type 5 LSA (O E2) so all other routers can reach the internet.
Router(config-router)# default-information originate always Injects the default route into OSPF even if the router doesn't currently have a default route in its routing table.
Router# show ip ospf database Views the LSDB. Displays LSA Types 1-5. Look for "Router Link States" (LSA 1), "Net Link States" (LSA 2), "Summary Net Link States" (LSA 3), etc.
Router# show ip route ospf Filters the routing table to show only routes learned via OSPF (O, O IA, O E1, O E2).

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Which specific type of OSPF router generates Type 3 (Summary) LSAs to advertise routes between different areas? ▼
The ABR (Area Border Router).
2. You look at the routing table and see a route marked with "O IA". What does this stand for, and what type of LSA generated it? ▼
It stands for OSPF Inter-Area. It indicates the route is in a different OSPF area, and it was generated by a Type 3 LSA.
3. You have configured a static default route out to the internet on your edge router. What command tells OSPF to advertise this default route to all internal routers? ▼
default-information originate. This injects the default route as a Type 5 External LSA (O E2).
4. Which LSA type is generated ONLY by the Designated Router (DR) on a broadcast network segment? ▼
Type 2 (Network LSA). It acts as a pseudo-node representing the broadcast segment itself and lists all connected routers.
