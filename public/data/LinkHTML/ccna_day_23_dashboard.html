Day 23: EtherChannel

What is EtherChannel?

EtherChannel (Link Aggregation) groups multiple physical interfaces together into one single logical interface called a Port-Channel.

STP Loop Prevention

Normally, STP would block redundant links to prevent loops. Because EtherChannel makes multiple cables look like one logical link, STP leaves all grouped ports forwarding!

Increased Bandwidth

Bundling four 1 Gbps links creates a single 4 Gbps logical link, increasing total throughput.

Redundancy

If one physical cable in the bundle fails, traffic instantly fails over to the remaining active links without triggering an STP recalculation.

🔤

Negotiation Protocols

Switches can statically form a channel, or dynamically negotiate it using one of two protocols.

LACP (IEEE 802.3ad)

Industry Standard. Use this in production!

  • Active: Actively sends LACP packets to form a channel.
  • Passive: Waits to receive LACP packets.
  • Max Links: Up to 16 links (8 active, 8 standby).

PAgP (Cisco Proprietary)

Port Aggregation Protocol. Older, Cisco only.

  • Desirable: Actively sends PAgP packets.
  • Auto: Waits to receive PAgP packets.
  • Max Links: Up to 8 links.
Static (No Protocol): mode on Forces the channel. Danger: Can cause loops if misconfigured.

✅ Bundling Prerequisites

For a Port-Channel to form, all physical interfaces in the group MUST have identical configurations. If they mismatch, the port is suspended (err-disabled or suspended).

  • ✔ Speed & Duplex must match.
  • ✔ Switchport mode (Access or Trunk) must match.
  • ✔ If Access, the VLAN must match.
  • ✔ If Trunk, allowed VLANs & Native VLAN must match.
  • ✔ STP port costs must match.

⚖️ How Load Balancing Works

EtherChannel does NOT do true round-robin (sending packet 1 on link 1, packet 2 on link 2). That would cause out-of-order packets. It uses a hashing algorithm.

The Hash Algorithm

The switch reads specific packet details (like Source MAC and Destination MAC), runs a math equation, and outputs a number (0-7). That number determines which physical wire to use.

Result: A single conversation (e.g., a file transfer from PC A to Server B) will ALWAYS take the exact same physical link.

Load-Balancing Methods

You can configure what the hash looks at:

  • src-mac / dst-mac / src-dst-mac
  • src-ip / dst-ip / src-dst-ip (Best for modern networks)
  • src-port / dst-port / src-dst-port

🪝 Brain Hooks

The Protocol Modes

LACP (IEEE): Active/Passive. (LACP).
PAgP (Cisco): Desirable/Auto. (Same exact modes as Cisco DTP!).

Load Balancing = Lanes

EtherChannel doesn't split a car in half across two lanes. It forces one car to stay in the left lane, and another car to stay in the right lane, preventing pile-ups.

⌨️ EtherChannel Configuration

Switch(config)# interface range g0/1 - 2 Switch(config-if-range)# channel-group 1 mode active Assigns the physical ports to Port-Channel 1 and sets them to LACP Active. (Creates logical int automatically).
Switch(config)# interface port-channel 1 Switch(config-if)# switchport mode trunk CRITICAL RULE: Once the channel is formed, make ALL configuration changes (VLANs, trunks) on the logical `port-channel` interface, NOT the physical interfaces!
Switch# show etherchannel summary The master verification command. Look for SU (S = Layer2, U = In Use). If it says SD, the channel is Down.
Switch(config)# port-channel load-balance src-dst-ip Changes the global load-balancing algorithm for the switch.

🔧 Troubleshooting

Status flag shows "s"

Symptom: show etherchannel summary lists ports with an (s) flag, meaning "Suspended".

Fix: This means parameters don't match. Ensure speed, duplex, and allowed VLANs are identical on all physical interfaces.

Mode Mismatch

Warning: You cannot mix LACP, PAgP, and On modes. Active + Auto = Fail. On + Active = Fail.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Switch A's interface is set to `mode active`. Switch B's interface is set to `mode passive`. Will an EtherChannel successfully form? ▼
Yes. This is LACP. The Active side initiates negotiation, and the Passive side accepts it. (Passive + Passive would fail).
2. If you connect two switches with 4 cables, Spanning Tree will block 3 of them. How does EtherChannel bypass this STP blocking? ▼
EtherChannel bundles the physical interfaces into a single logical interface (Port-Channel). Because STP evaluates logical links, it sees only ONE path between the switches, so it leaves the entire bundle in a Forwarding state.
3. Which EtherChannel negotiation protocol uses the modes "Desirable" and "Auto"? ▼
PAgP (Cisco Proprietary).
4. When you run `show etherchannel summary`, you see the port-channel is listed with the flags (SU). What does SU indicate? ▼
S = Layer 2 (Switching). U = In-Use (The channel is up and operational).
