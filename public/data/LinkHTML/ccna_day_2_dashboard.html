Day 2: Interfaces & Cables

CCNA 200-301 Mastery Dashboard

🔌

Copper Cables (UTP)

Uses electrical signals. Unshielded Twisted Pair (UTP) is standard for LANs. Twists prevent crosstalk.

  • Connector: RJ-45 (8 pins).
  • Distance Limit: 100 meters max.
  • Cat5: 100 Mbps (FastEthernet).
  • Cat5e: 1 Gbps (GigabitEthernet).
  • Cat6: 10 Gbps (up to 55m only).
  • Cat6a: 10 Gbps (full 100m).
✨

Fiber Optic Cables

Uses light to transmit data. Immune to Electromagnetic Interference (EMI).

  • Single-Mode (SMF): Uses a laser. Small core. Designed for very long distances (kilometers). Usually yellow jacket.
  • Multi-Mode (MMF): Uses an LED. Larger core (light bounces). Designed for shorter distances (LAN backbone, up to ~500m). Usually orange/aqua jacket.
  • Connectors: LC (Lucent, small), SC (Subscriber, larger), ST.
🔀

Pinouts & Types

How pins are wired inside the RJ-45 connector (T568A vs T568B).

  • Straight-Through: Connects unlike devices (PC to Switch, Router to Switch). Pins 1&2 map to 1&2.
  • Crossover: Connects like devices (Switch to Switch, PC to PC, Router to PC). Pins 1&2 map to 3&6.
  • Auto-MDIX: A modern feature that automatically detects and swaps the transmit/receive pins, allowing you to use a straight-through cable for everything.

🧠 Brain Hooks

"Unlike likes Straight, Like likes Cross"
Unlike devices = Straight-through. Like devices (or Router+PC) = Crossover.
S for Single, S for Straight laser
Single-Mode Fiber (SMF) goes in a straight line for long distances.
Interface Status Rules
"Status" = Layer 1 (Physical Cable).
"Protocol" = Layer 2 (Data Link/Config).

⌨️ CLI Command Memorization

show ip interface brief The most important CCNA command! Shows interface IP, Status (L1), and Protocol (L2) states.
show interfaces Shows detailed MAC address, speed, duplex, and input/output errors.
interface g0/0 Enters interface configuration mode for GigabitEthernet 0/0.
no shutdown Turns on (enables) a disabled interface. (Changes state from 'administratively down' to 'up').

🔧 Common Troubleshooting

Status: Administratively Down / Protocol: Down

Fix: The interface was manually disabled. Enter config-if mode and type no shutdown.

Status: Down / Protocol: Down

Fix: Layer 1 issue. Check the cable, make sure the device on the other end is powered on, or check if the wrong cable type (straight vs cross) was used without Auto-MDIX.

Status: Up / Protocol: Down

Fix: Layer 2 issue. Cable is fine, but there is a mismatch (e.g., speed/duplex mismatch, or encapsulation mismatch on serial links).

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Without Auto-MDIX, what type of cable is required to connect two switches together? ▼
Crossover cable. Switches are "like" devices, so they require a crossover cable to connect their transmit (Tx) pins to the other switch's receive (Rx) pins.
2. What is the primary difference in use case between Single-Mode Fiber (SMF) and Multi-Mode Fiber (MMF)? ▼
Distance. Single-Mode Fiber uses a laser and is designed for very long distances (kilometers). Multi-Mode Fiber uses an LED and is designed for shorter LAN backbone distances (typically up to ~500m).
3. Which CLI command gives a quick summary of all interface IP addresses and their current Status/Protocol states? ▼
show ip interface brief
4. If an interface shows "administratively down", what does this mean and how do you fix it? ▼
It means an administrator manually disabled the port. You fix it by entering interface configuration mode and typing no shutdown.
