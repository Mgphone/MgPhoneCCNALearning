Why Subnet?

Subnetting is the process of borrowing host bits to create smaller, more efficient network divisions.

1. Reduces Broadcast Traffic

Large networks suffer from excessive broadcast traffic (ARP, DHCP). Subnetting breaks one massive broadcast domain into several smaller ones.

2. Conserves IP Addresses

Instead of wasting a full /24 (254 hosts) on a point-to-point link that only needs 2 IPs, you can use a /30 subnet and save the rest.

3. Improves Security

By dividing networks (e.g., HR, Sales, IT), you can place a Router or Firewall between them and apply Access Control Lists (ACLs).

🧮

The Golden Formulas

You must know these two formulas by heart to pass the CCNA.

N

Number of Subnets

2n

n = Number of network bits borrowed from the host portion.

H

Usable Hosts per Subnet

2h - 2

h = Number of remaining host bits (0s).
*(Subtract 2 for Network & Broadcast IDs)*

✨ The Magic Number

The fastest way to subnet on an exam is finding the Block Size (how much each subnet jumps by).

256 - Interesting Octet

  1. Find the "interesting octet" in the subnet mask (the one that is not 255 or 0).
  2. Subtract that value from 256.
  3. The result is your block size! Your subnets will increment by this number.

🛠️ Subnetting in Action: 192.168.1.0 /26

1. Find the Magic Number

A /26 mask is 255.255.255.192.

The interesting octet is the 4th (192).

256 - 192 = 64 (Block Size)

2. Build the Subnet Table

Start at 0, and add the Magic Number (64) to the 4th octet.

Network ID Range Broadcast
.0 .1 - .62 .63
.64 .65 - .126 .127
.128 .129 - .190 .191
.192 .193 - .254 .255

🪝 Brain Hooks

The Broadcast Trick

The Broadcast address is ALWAYS exactly 1 less than the NEXT Network ID.

If the next network starts at .64, the previous broadcast must be .63.

Powers of 2 to memorize:

22 = 4
23 = 8
24 = 16
25 = 32
26 = 64
27 = 128

⌨️ Verifying Subnets in CLI

Router# show ip interface brief Displays the exact IP address configured on the interface, but NOT the subnet mask.
Router# show ip route connected Look at the 'C' routes. The router automatically calculates the Network ID based on your configured IP/Mask and displays it here. E.g., `C 192.168.1.64/26 is directly connected...`
Router# show interfaces g0/0 Look near the top. It displays the IP address alongside its subnet mask in dotted decimal format (e.g., `Internet address is 192.168.1.65/26`).

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. What is the "Magic Number" (block size) for a subnet mask of 255.255.255.224? ▼
32. (Calculation: 256 - 224 = 32). This means the subnets will increment by 32 (0, 32, 64, 96...).
2. You have a /28 subnet mask. How many USABLE hosts are available in each subnet? ▼
14 usable hosts. A /28 leaves 4 bits for hosts (32 - 28 = 4). The formula is $2^4 - 2$. (16 - 2 = 14).
3. In the host formula ($2^h - 2$), exactly why do we subtract the number 2? ▼
Because two addresses in every subnet are reserved and cannot be assigned to devices: The Network Address (host bits all 0) and the Broadcast Address (host bits all 1).
4. If your current Network ID is 192.168.1.64 /26, what is the Broadcast Address for this subnet? ▼
192.168.1.127. A /26 has a block size of 64. The next network starts at 192.168.1.128 (64+64). The broadcast is always one less than the next network (128 - 1 = 127).
