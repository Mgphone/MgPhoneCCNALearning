MAC Address Anatomy

Media Access Control (MAC) addresses are physical, burned-in addresses (BIA) operating at Layer 2.

48 Bits 12 Hex Digits 6 Bytes
00:1A:2B:3C:4D:5E
  • First 24 Bits (OUI): Organizationally Unique Identifier. Assigned to the vendor (e.g., Cisco, Dell) by the IEEE.
  • Last 24 Bits (NIC): Network Interface Controller specific. Assigned by the vendor (like a serial number).
  • Cisco Format: Cisco routers/switches format MACs with periods instead of colons: 001a.2b3c.4d5e
🚦

Traffic Types

  • Unicast (One-to-One) Destined for a single, specific MAC address.
  • Broadcast (One-to-All) Destined for all devices on the local network. The broadcast MAC address is always FF:FF:FF:FF:FF:FF.
  • Multicast (One-to-Many) Destined for a specific group of devices listening to that multicast stream. (IPv4 Multicast MACs begin with 01:00:5E).

  • FCS (Trailer): Frame Check Sequence. Checks for errors/corruption using a CRC (Cyclic Redundancy Check).

🧠 How a Switch Works: Learning & Forwarding

When a switch receives a frame on a port, it performs two distinct, sequential operations:

Step 1: Learning (Source MAC)

The switch looks at the Source MAC Address of the incoming frame.

  • If the Source MAC is not in the MAC table, it adds it along with the port it came in on.
  • If the Source MAC is already in the table, it resets the aging timer (usually 300 seconds / 5 mins).

Step 2: Forwarding (Dest MAC)

The switch looks at the Destination MAC Address to decide where to send it.

  • If the Dest MAC is in the table, it forwards the frame ONLY out of that specific port.
  • If the Dest MAC is NOT in the table (Unknown Unicast), or if it is a Broadcast address, the switch FLOODS the frame out of ALL ports except the port it received it on.

🪝 Brain Hooks

"Learn from the Source, Forward to the Destination" Never mix these up. The MAC table is built entirely by looking at Source MACs. Routing/Forwarding is done entirely by looking at Dest MACs.
Flooding Rule "When in doubt, shout it out!" (If the destination is unknown, flood it out all ports except the origin).

⌨️ CLI Commands

Switch# show mac address-table Displays the MAC address table, showing learned MACs, their associated VLAN, and the interface they reside on. (Note: On older IOS, it's `show mac-address-table` with a hyphen).
Switch# clear mac address-table dynamic Flushes all automatically learned MAC addresses from the table. Good for troubleshooting or resetting lab environments.

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. Which address in the Ethernet frame does a switch use to BUILD its MAC address table? ▼
The Source MAC Address.
2. What is the Layer 2 Broadcast MAC address? ▼
FF:FF:FF:FF:FF:FF (All 48 bits are set to 1).
3. What action does a switch take when it receives an "Unknown Unicast" frame? ▼
It floods the frame out of all active ports, except the port it originally received the frame on.
4. How long is a MAC address in bits and bytes? ▼
48 bits, which equals 6 bytes.
