Open Shortest Path First Basics

🗺️

OSPF Core Concepts

OSPF is an open-standard, Link-State IGP. It uses Dijkstra's Shortest Path First (SPF) algorithm to calculate the best route to all destinations.

Link-State Database (LSDB)

Every OSPF router within an area builds an identical topology map of the network by exchanging Link-State Advertisements (LSAs).

Administrative Distance & Algorithm

  • AD: 110
  • Algorithm: Dijkstra's SPF
  • Protocol: IP Protocol 89 (Does NOT use TCP/UDP)

Areas & Area 0

OSPF uses a hierarchical design to scale. All areas MUST connect to the Backbone Area: Area 0. This prevents the LSDB from becoming too large and crashing router CPUs.

🆔

Router ID (RID) Election

Every OSPF router must have a unique 32-bit (IPv4 format) name to identify itself to neighbors. The router chooses its RID using a strict order of operations:

  1. Manual Configuration: If the `router-id` command is used, this wins immediately. (Best Practice!)
  2. Highest Active Loopback IP: If no manual RID exists, the router picks the highest IP address on any active Loopback interface. Loopbacks are preferred because they never physically go down.
  3. Highest Active Physical IP: If no loopbacks exist, it picks the highest IP on any active physical interface.
🧮

OSPF Metric: Cost

OSPF uses Cost as its metric. The path with the lowest cumulative cost to the destination wins. Cost is calculated automatically based on interface bandwidth.

Reference Bandwidth


Interface Bandwidth

The Reference Bandwidth Problem

The default Reference Bandwidth is 100 Mbps. OSPF calculates cost in whole numbers (no decimals).

  • 10 Mbps Ethernet: 100 / 10 = Cost 10
  • 100 Mbps FastEth: 100 / 100 = Cost 1
  • 1 Gbps Gigabit: 100 / 1000 = Cost 1 (Rounded up)
  • 10 Gbps 10-Gig: 100 / 10000 = Cost 1

The Fix: Modern networks MUST change the default reference bandwidth so OSPF can tell the difference between 1G and 10G links! (`auto-cost reference-bandwidth 100000`)

🃏 Wildcard Masks

OSPF uses wildcard masks in its `network` commands. A wildcard mask is usually the exact inverse of a subnet mask.

  • 0 = Must Match Exactly
  • 255 = Don't Care

How to calculate quickly:

Subtract the subnet mask from 255.255.255.255.

Ex: /24 (255.255.255.0) -> 0.0.0.255

Ex: /30 (255.255.255.252) -> 0.0.0.3

⌨️ Basic OSPF Configuration

Router(config)# router ospf 10 Enters OSPF configuration mode. The '10' is the Process ID (locally significant, doesn't need to match neighbors).
Router(config-router)# router-id 1.1.1.1 Manually sets the Router ID. If OSPF was already running, you must run `clear ip ospf process` for this to take effect!
Router(config-router)# network 192.168.1.0 0.0.0.255 area 0 Tells OSPF to look for interfaces matching this IP/Wildcard. It enables OSPF on those interfaces and puts them in Area 0.
Router# show ip ospf neighbor The most important verification command. Shows active OSPF neighbors and their state (look for 'FULL').

🔧 Troubleshooting Adjacencies

Neighbors not forming?

For two routers to become OSPF neighbors, several parameters MUST match exactly:

  • Area ID
  • Hello and Dead Timers (Default 10s/40s)
  • Subnet Mask
  • Authentication Passwords

⚡ Quick Quiz (Active Recall)

Click questions to reveal answers.

1. What is the default Administrative Distance (AD) of OSPF? ▼
110.
2. In a multi-area OSPF design, all standard areas must connect directly to which specific area? ▼
Area 0 (The Backbone Area).
3. Why does OSPF struggle to differentiate between a 1 Gbps link and a 10 Gbps link by default? ▼
Because OSPF's default Reference Bandwidth is 100 Mbps, and it does not use decimals. Therefore, 100 Mbps, 1 Gbps, and 10 Gbps links all calculate to a Cost of 1. (Requires the `auto-cost reference-bandwidth` command to fix).
4. If a router is not manually configured with a Router ID, what is the very next thing it checks to determine its RID? ▼
It selects the highest IP address configured on an active Loopback interface. (If no loopbacks exist, it falls back to the highest physical interface IP).
