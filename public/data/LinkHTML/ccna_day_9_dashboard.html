Bulk Config & Labels

Managing a 48-port switch port-by-port is inefficient. Use ranges and descriptions to stay organized.

Interface Range

Allows you to apply the same configuration to multiple interfaces simultaneously.

Switch(config)# interface range f0/1 - 12 Switch(config)# interface range f0/1 - 5, g0/1 - 2

⚠️ Syntax Alert: You MUST have spaces around the hyphen.

Descriptions

A local label for the port. It does not affect operation but is crucial for administration and troubleshooting.

Switch(config-if)# description UPLINK TO CORE

Viewed via show interfaces description.

🔀

Auto-MDIX

Automatic Medium-Dependent Interface Crossover.

  • The Problem: Historically, connecting Switch-to-Switch required a Crossover cable. Connecting Switch-to-PC required a Straight-through cable.
  • The Solution: Auto-MDIX detects the required cable connection type and configures the interface pins appropriately.
  • Result: You can use a straight-through cable for everything if Auto-MDIX is enabled (it usually is by default on modern Cisco gear).
⛔

Err-Disable State

When the switch detects a severe violation or error on a port, it automatically shuts it down to protect the network.

Common Causes: Port Security violations (plugging in an unauthorized MAC), BPDU Guard (STP loop protection), or severe flapping.

How to Recover (The "Bounce"):
  1. Fix the underlying issue first!
  2. Enter config-if mode.
  3. Type shutdown
  4. Type no shutdown

🪝 Brain Hooks

"Bouncing the Port" A common IT phrase. It means turning the port off and back on again (shut / no shut). Mandatory for err-disable recovery!
Auto-MDIX = The Universal Adapter Like a smart plug that automatically flips its positive/negative poles so you don't blow a fuse, regardless of what you plug into it.

⌨️ Interface Configuration CLI

Switch(config)# interface range f0/1 - 10 , g0/1 Enters interface range mode. Note the spaces around the hyphen and comma separation for different types.
Switch(config-if)# mdix auto Manually enables Auto-MDIX on the interface.
Switch# show interfaces status err-disabled Filters the status output to ONLY show ports currently in the err-disabled state and the reason why.
Switch# show interfaces description Displays a quick list of interfaces, their up/down status, and their configured descriptions.

🔧 Troubleshooting

Status: err-disabled

Action: Run show interfaces status err-disabled to find the reason. Fix the cause, then shut and no shut the interface.

Status: up / Protocol: down

Action: Layer 1 is fine (cable is plugged in), but Layer 2 is failing. Check for Speed/Duplex mismatches between the two switches.

1. A port suddenly changes to "err-disabled". What likely caused this, and how do you turn the port back on? ▼
It was caused by a security violation (like Port Security or BPDU Guard). After fixing the issue, you must bounce the port by entering shutdown followed by no shutdown.
2. Write the exact command to configure FastEthernet ports 1 through 24 at the same time. ▼
interface range f0/1 - 24 (Note the spaces around the hyphen!)
3. What feature allows a switch to automatically adjust its transmit and receive pins, forgiving you for using a straight-through cable instead of a crossover? ▼
Auto-MDIX (Automatic Medium-Dependent Interface Crossover).
4. Which command provides a clean list of all ports, their status, and the labels configured by the administrator? ▼
show interfaces description
